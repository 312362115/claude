# 命题驱动分析主流程

> **skill 的主入口**。用户触发一个命题，skill 抓数据+深度分析+输出决策卡。
> **手动触发**，不自动定时。
>
> 这是 finance-analysis skill 的核心工作流——**命题 → 分析 → 决策**。

---

## ⚡ 最高铁律：数据真实 + 有效 + 最新

> **数据是"1"，分析是"0"。"1"不到位，后面再多"0"都没意义。**

所有分析的基础前提是**数据质量**。违反任一条，**整个分析作废**，不是"打折"，是**失效**。

### 三条铁律（任一违反 = 重做）

#### 铁律 1：**真实**
- 数据必须来自原始披露源（SEC / HKEXnews / 巨潮 / 公司 IR / 官方 API）
- **不相信单源二手转引**（尤其是雪球/同花顺聚合的衍生数据）
- 关键数字至少 2 源交叉验证（若冲突，**以一手源为准**）

#### 铁律 2：**有效**
- 数字必须**精确到可计算**（不是"大约""左右""接近"）
- 缺字段必须**明确标注**（不硬编 / 不猜测 / 不用同业代入）
- 单位一致（市值 "亿" vs "亿美元" / 股价 "RMB" vs "HKD"）
- 口径一致（Trailing TTM / Forward / 本季度 / 同比 / 环比 必须明确）

#### 铁律 3：**最新**
- **股价必须是当天或前一交易日数据**
- **财报必须是最新发布的季报/年报**（不能用去年的数据分析今年）
- **宏观数据**（如 Capex 指引）必须是最新一季
- 每个数据点标 `【时点：YYYY-MM-DD HH:MM】`
- 跨时点数据**不可直接混用**（4 月的股价和 3 月的业绩不能简单除出 PE）

### 数据鲜度分级

| 数据类型 | 鲜度要求 | 超期处理 |
|---------|---------|---------|
| 股价 / 市值 | 当天（盘中）/ 前交易日（盘后）| 必须重抓，不降级 |
| 财报数据 | 最新发布的季报 / 年报 | 老财报不能代替新财报 |
| 行业动态 | 近 30 天 | 超期标"数据陈旧"警告 |
| 产业链数据（Capex等）| 近 3 月 | 超期标"数据陈旧"警告 |
| 历史对比数据 | 可用旧数据 | 但必须标时点 |

### 抓不到的处理（**穷尽多源再承认**）

**错误**：一次抓失败 → "数据不足不建议下注"（**太懒**）
**正确**：

```
源 1 失败 → 源 2 → 源 3 → 源 4 → ... → 穷尽 → 明确告诉用户"N 个源都试过，抓不到"
```

**A 股股价多源备选**（WebFetch 有效性实测，按优先级）：

| 源 | WebFetch 可抓 | 备注 |
|----|------------|------|
| **cn.investing.com/equities/** | ✅ **最可靠** | 数据完整 + 含 PE/PB/RSI/52周等 |
| investing.com（英文版） | ✅ | 同上 |
| 同花顺 basic.10jqka.com.cn/{code}/ | ⚠️ 部分 | F10 页可抓基本面 |
| 雪球 xueqiu.com/S/SZ{code} | 🔴 WAF 反爬 | blacklist |

### 穷尽之后仍不行才承认边界

写明：
- 具体试过哪些源
- 每个源为什么失败（反爬 / 付费 / 无数据 / 停服）
- 告诉用户**需要手工补什么数据**才能继续

**这是职业分析师标准** —— "做不到" 必须有清单。

## 🎯 Skill 的灵魂：两个核心价值

**这两点是 skill 存在的理由**。做不到这两点 = skill 没价值，还不如让用户自己 Google。

### 价值 1：**数据深度**（原始数据优先，不抄）

**铁律：一手 > 二手 > 三手**。每份分析必须优先抓**原始数据**，而不是别人的综述。

#### 数据源分级（T1-T5）

| 级别 | 类型 | 例子 | 使用优先级 |
|------|------|------|----------|
| **T1 一手原始** | 公司披露 / 政府原文 / 交易所原始数据 | SEC 10-K 原文 / 央行货币政策原文 / HKEXnews PDF | **必抓** |
| **T2 结构化 API** | 机读数据 | FRED API / SEC EDGAR XBRL / BIS SDMX | **必抓** |
| **T3 权威媒体** | 专业财经记者深度报道 | 彭博 / 路透 / WSJ / 财新 | 辅助 |
| **T4 聚合分析** | 卖方摘要 / 咨询报告 | Seeking Alpha / 券商报告 PDF | 辅助（需验证） |
| **T5 社区观点** | 论坛 / 社交 | X / Reddit / 雪球 | **仅作情绪指标** |

#### 数据深度 3 条铁律

1. **能读财报原文就不要读新闻摘要**：新浪财经写"群核科技业绩转正" 3 秒看完，真正的 Alpha 在招股书第 100 页"研发费用率从 72.9% 降到 37.5%"的小字里。
2. **能拿 API 机读就不要看 PDF 截图**：SEC EDGAR 有 JSON API 可以批量抓 XBRL 字段，比人工截图读 10-K 快 100 倍且准确。
3. **能直接抓交易所就不要信二手聚合**：A 股看巨潮，港股看 HKEXnews，美股看 SEC EDGAR。Tushare / Yahoo Finance 是二手，有延迟、有错误、可能腐烂。

#### 数据源必须多源交叉

单源容易被骗（文字可以写得天花乱坠，数据也可以漂亮）。每份分析至少 **4 个独立源**交叉：

```
原始披露（T1-T2）     ← 主
+ 产业链旁证（T1-T3）  ← 验真
+ 同业对标（T1-T2）    ← 定位
+ 反面证据（主动搜）   ← 压力测试
```

### 价值 2：**分析深度**（独立推演，不抄观点）

**铁律：自己算 > 自己推 > 自己评**。不做"观点聚合器"。

#### 3 类偷懒的反面教材（绝对避免）

❌ **"5 家卖方都说买入，所以买入"** → 这是抄共识，不是分析
❌ **"新闻说群核涨 144%，因为 AI 概念"** → 这是复读机，不是分析
❌ **"同花顺研报推荐这只股"** → 这是转引，不是分析

#### 独立分析的 4 个动作

**动作 1：自己算估值**
- 不用卖方目标价，自己做相对估值 / DCF
- 对标 3-5 家同业，自己算 PE/PS/EV 中位数
- 算历史估值分位（当前 vs 过去 5 年）
- **案例**：群核 37x PS 首日收盘，卖方说"AI 溢价合理"。自己算：Autodesk 7.1x PS / 广联达 3.9x PS / 中望软件 12.8x PS → 37x 已经 3 倍于国内 AI 软件锚 → 严重溢价，不是"合理"。

**动作 2：自己推产业链（≥ 2 跳）**
- 不看"分析师报告说 CPO 受益"，自己推：ChatGPT → NVDA → HBM → CPO
- 每跳问"为什么"：为什么 AI 推理需要光模块？因为 GPU 集群互联带宽是瓶颈
- **案例**：群核单看是 SaaS 公司，但推 2 跳："Tesla Optimus 量产 → 需要具身智能训练数据 → 谁有 3D 场景？→ 群核"。这不是抄观点，是独立推演得出 Layer 6+7 双卡位。

**动作 3：自己读财报（不看摘要）**
- 10-Q 至少读 MD&A（Management Discussion）全段 + 财务附注
- 关注小字段：Accounts Receivable / Inventory / CapEx / Deferred Revenue
- 每次财报做 YoY + QoQ + 二阶导（加速/减速）
- **案例**：NVDA 2023-Q1 财报，新闻头条"Beat 预期"。真正 Alpha 在电话会"Q2 guidance $11B（consensus $7.1B）" —— 读原文的人当日加仓，看头条的人追高。

**动作 4：自己压测反面**
- 不等别人写 bear case，主动问"最坏会怎样"
- 至少推 3 种失败路径 + 量化下行幅度
- **案例**：群核 bear case 不是"AI 泡沫破"这种空话，而是"中国房地产持续下行 → 85% 收入受冲击 → 2026-H2 营收 -20%"。

#### 独立分析的边界

**不是"重新发明"**，是 "**别人的结论只作输入，结论必须自己得出**"：
- 可以参考卖方目标价（作为市场共识的 anchor）
- 可以看新闻（作为事件起点）
- 可以读别人的 bear case（补充自己没想到的）
- **但最终判断必须自己做**，不当观点复读机

### 价值 3：**透过表象 + 预判未来**（最高阶的 Alpha 生产）

**铁律：分析现状只是起点，预判未来才是终点**。任何分析最后必须回答"未来 6-24 个月会怎样"，不回答 = 没价值。

#### "透过表象"3 层拆解

金融市场里，信息有 3 层：

| 层 | 特征 | 示例（NVDA 2023 Q1） | Alpha |
|----|------|---------------------|------|
| **表层** | 媒体标题、情绪、涨跌 | "NVDA Q1 Beat 预期 10%" | 0（所有人都看到） |
| **中层** | 财报数据、指引、公告 | "Data Center 营收 $4.28B YoY +14%" | 小（读财报的人能看到） |
| **里层** | 数据间的异常、趋势拐点、隐藏信号 | "Q2 Guidance $11B（consensus $7.1B），+55% 跳空 → HBM 供应已锁死" | **大（这才是 Alpha）** |

**透过表象 = 从表层跳到里层**。3 个方法：

1. **看数据之间的关系**（不看单点）
   - 营收 YoY 加速 + 应收账款同比更快增 = 压货信号（警告）
   - 库存下降 + 交付周期延长 = 供应紧张（涨价信号！）
   - Capex 跳空 + R&D 占比升 = 新产品周期开启
2. **看指引和实际的落差**（预期管理）
   - 公司给保守指引但财报超 beat = 沙袋策略（好）
   - 公司给激进指引但勉强达标 = 已在顶部（警告）
3. **看沉默的字段**（什么没说）
   - 财报电话会突然不提某产品线 = 问题信号
   - 8-K 披露的时间点（周五盘后 vs 周一早盘）= 隐含态度

#### "预判未来" 4 个视角

每份分析必须预判以下 4 个时间维度的未来：

**1. 下季度（T+3 月）**
- 下次财报怎么走？Beat/Miss/In-line？
- 关键信号：Capex 指引 / guidance / 订单能见度

**2. 下半年（T+6 月）**
- 产业链传导完成没？A 股间接标的启动没？
- 关键信号：HBM 价格 / 光模块出货 / 客户扩产

**3. 一年（T+12 月）**
- 业绩兑现程度？估值 re-rating 空间？
- 关键信号：营收增速是否保持 / 毛利率趋势 / 新产品兑现

**4. 周期拐点（T+24 月）**
- 主题进入共识期还是衰减期？
- 关键信号：媒体覆盖 / 散户 FOMO / 卖方一致目标价

#### 预判的可证伪性（决定分析价值）

**无价值的预判**："AI 是未来 10 年最大机会" （放之四海，无法证伪）
**有价值的预判**："2026-Q3 NVDA Data Center 营收 YoY > +40%，低于此预期 → 清仓" （可证伪 + 有明确时点）

**所有预判必须满足**：
- ✅ 具体（数字 / 时点）
- ✅ 可证伪（X 未发生 → 论点错）
- ✅ 有行动意义（结果触发什么动作）

#### 经典案例：透过表象的 Alpha

**案例 1：CPO 2023-Q4**
- 表层：NVDA 涨了（所有人看到）
- 中层：NVDA 财报 Data Center +200%（读财报看到）
- **里层**：NVDA 财报中"HBM 供应紧张限制出货" 1 行字 → 推 2 跳到光模块 CPO（**Alpha**）

**案例 2：群核 0068.HK**
- 表层：IPO 首日 +144%（新闻看到）
- 中层：市占 23% + 数据飞轮（读招股书）
- **里层**：SpatialLM 开源登 HF 榜 + 智元/银河通用战略合作 → 具身智能数据层卡位（**Alpha**）

**案例 3：A 股存储股 2024-03**
- 表层：美光财报 Beat
- 中层：HBM 收入 +500%
- **里层**：美光电话会"HBM 卖光到 2025" + 产线扩张跟不上 → A 股兆易 / 佰维 15 天内启动（**Alpha 窗口期**）

## 核心原则（在三大价值之上）

1. **命题为王**：用户给什么命题，分析围绕什么转。不扩散不越界。
2. **数据驱动**：每个结论都要有数据支撑（T1-T2 为主），WebSearch/WebFetch 必选。
3. **决策导向**：最后必须输出"买/卖/持 + 目标价 + 止损 + 仓位"，不做"情况分析"就结束。
4. **诚实边界**：数据不足 / 论点弱 / 预期差 ≤ 2 → 明确说"不建议下注"，不硬凑结论。

## 铁律 -2：两阶段工作流 —— 先搞懂，再深入

**最顶层流程原则**：任何个股/多业务标的分析，**必须先完成 Phase A 基础画像，才能进 Phase B 深入分析**。跳阶段 = 瞎分析。

### Phase A：先搞懂（基础画像，7 步）

**不做完 Phase A 不能进 Phase B**。每步产出记录，Phase B 所有判断基于 Phase A 的数据。

| 步骤 | 任务 | 产出 | 失败处理 |
|------|------|------|---------|
| **A.1 时点校准** | 抓当前股价 / 市值 / 交易时间 | 明确时点的数字 | 穷尽多源（见铁律 0）|
| **A.2 业务画像** | 分部营收 / 占比 / 增速 / 毛利率 | 分部表 | 至少 3 年历史 |
| **A.3 板块归类** | 属于哪些板块 / 产业链 Layer 位置 | 板块归属表 + Layer 图 | 主业 Layer + 新业务 Layer 都标 |
| **A.4 同业地图** | 主动发现 ≥ 5 家同业（见铁律 -1）| 同业候选 table | 筛 3-5 家做对标 |
| **A.5 历史估值区间** | 5 年 PE/PS/PB 分位数 | 当前 vs 历史分位 | 数据源：同花顺/东财 F10 |
| **A.6 股东结构（7 类资金拆分，必做）** | 见下方详细拆解 | 7 类资金力量对比表 | 用"机构"一个词不够，必须拆到资金性质 |
| **A.7 关键事件时间线** | 近 90 天股价异动 vs 新闻对应 | 事件-股价映射表 | 用于 Phase B 估值归因 |

### Phase B：再深入分析

```
1. 事件驱动分析（若有财报/政策/并购，套 earnings-deep-dive）
2. 业务归因拆解（估值重塑归因 5 步法，见铁律 -0.5）
3. 同业对标估值（至少 3-5 家，龙头 + 直接对手 + 上下游）
4. 7 问法深度论证（套 alpha-thesis-card）
5. 4 维预判 + 证伪条件
6. 决策卡输出
```

### 两阶段的逻辑关系

```
Phase A 产出 → 喂给 Phase B
  ├─ A.1 时点 → B.3 估值锚（对标用当前股价）
  ├─ A.2 业务画像 → B.2 业务归因（拆到分部）
  ├─ A.3 板块 → B.3 对标（按板块找对手）
  ├─ A.4 同业地图 → B.3 对标候选池
  ├─ A.5 历史估值 → B.5 目标价（历史分位是参考锚）
  ├─ A.6 股东结构 → B.5 顶部信号（机构出货？）
  └─ A.7 事件时间线 → B.2 估值归因 Step 4（股价事件映射）
```

Phase A 任一失败 → Phase B 对应环节也失败（数据链断裂）。

### 红线

- Phase A 完成度 < 80% → **不得进 Phase B**
- Phase A 某步骤数据穷尽多源仍抓不到 → 明确标注 "缺失" 不猜测
- Phase B 的每个判断必须引用 Phase A 的具体数据点

### A.6 股东结构 7 类资金拆解（必做，不拆视同未做）

**"机构"是笼统词，不同性质机构方向可能完全相反**。必须拆到资金性质：

| 类别 | 关键数据 | 解读 | 数据源 |
|------|---------|------|-------|
| **1. 公募基金 - 价值派大佬** | 睿远/中欧/易方达蓝筹 等重仓/减仓 | **最重要信号**（价值派判断基本面）| 基金季报 / 十大流通股东 |
| **2. 公募基金 - 主题派** | 新华/鹏华/永赢/万家/华夏 等重仓 | 跟主题不跟基本面 | 同上 |
| **3. 北向资金** | 陆股通持股量周度变化 | 外资长钱，相对理性 | 港交所披露易 / 东财北向数据 |
| **4. 主力资金（游资/席位）** | 单日净买入/卖出大 | 短期博弈，易在顶部出货 | 东财主力资金流 |
| **5. 社保/养老金/QFII** | 长期资金增减 | 极稳定长线资金 | 季度披露 |
| **6. 大股东/董监高** | 增持/减持公告 | 内部人信号 | 交易所 SDI 披露 |
| **7. 股东户数变化** | 集中度提升/分散 | 筹码结构 | 季度披露 |

### 关键：识别"机构内部分裂"

7 类资金方向可能**完全相反**：
- 价值派减仓 + 主题派加仓 = **最后一波行情**（顶部特征）
- 价值派加仓 + 主题派减仓 = **价值回归起点**（底部特征）
- 北向 + 社保加仓 = **长线看好**（最强信号）
- 游资撤 + 公募进 = **筹码转换，稳定正向**
- 内部人减持 + 外部机构追 = **警报信号**（内部人最懂）

**血泪案例**：东山精密 002384（2026-04 分析）
- 只看"4-14 主力净卖 10 亿" → 误判为"机构全面出货"
- 真实情况：傅鹏博（价值派）减，但主题派仍重仓（40+ 只基金）
- **三股资金博弈** → 结论应为"聪明钱已走，主题资金最后一波"，减仓 50-60% 不是全清

### 7 类资金数据抓取源

- **基金季报**：1/4/7/10 月披露，WebSearch "<标的> 一季报 基金重仓"
- **北向资金**：WebFetch hkexnews.hk CCASS 或 data.eastmoney.com/hsgt
- **主力资金**：stock.eastmoney.com/zhuli.html 或 WebSearch "<ticker> 主力资金 日期"
- **大股东/董监高**：交易所披露易 SDI
- **股东户数**：季度披露，东财 F10

### 快速自检

每次个股分析前先跑：

```
□ A.1 当前股价（含时点）
□ A.2 分部营收表（≥ 3 分部若多业务）
□ A.3 所属板块 / Layer
□ A.4 ≥ 5 家同业（候选）→ 3-5 家（对标）
□ A.5 5 年估值分位
□ A.6 7 类资金方向（不是一个"机构"词）
□ A.7 近 90 天股价 - 事件映射

全部 √ 才进 Phase B
```

## 铁律 -1：主动发现同板块标的（不依赖用户喂信息）

**任何个股命题，必须自动完成板块发现**，不能依赖用户告诉你"竞争对手是谁"。

### 5 种发现方法（至少用 2 种交叉）

| 方法 | 工具 | 产出 |
|-----|------|------|
| **WebSearch 板块** | "<标的> 同行业 竞品 / 板块 龙头 对比" | 媒体文章列出的对标 |
| **东财板块页面** | `q.10jqka.com.cn/gn/detail/code/<板块>` 或 `quote.eastmoney.com/center/boardlist.html` | 全板块成员 + 市值 PE 排名 |
| **自有 framework 地图** | `ai-thematic-investing.md` Layer 地图（若 AI 主题）| 自己沉淀的产业链成员 |
| **年报"竞争格局"章节** | 抓年报原文该章节 | 管理层亲口说的对手 |
| **专业研报对标表** | "<标的> 深度报告 / 研报 PDF" | 卖方列出的同业对比表 |

### 强制步骤（Step 0.5-0.7）

```
Step 0 时点校准（股价）
  ↓
Step 0.5 识别产业链位置
  - 主业是什么（PCB / 光模块 / 消费电子 / ...）
  - AI 产业链 Layer 定位（若适用）
  - 多 Layer 卡位检查（如东山：PCB + 光模块 + 苹果供应链 + 汽车）
  ↓
Step 0.6 **自动发现同板块**（5 种方法至少 2 种）
  - 拉 5-10 家候选
  - 要求：市值不能太离谱（±10 倍内）
  ↓
Step 0.7 筛 3-5 家做对标
  - 按"业务重合度 × 市值接近度 × 数据可得性"排序
  - 龙头（1 家）+ 直接对手（2-3 家）+ 上游或下游（1 家）
  - 东山精密案例：沪电（主业 PCB 对标）+ 鹏鼎（苹果供应链对标）+ 中际旭创（光模块对标）+ 生益科技（上游覆铜板）+ 永鼎股份（光模块二线）
  ↓
Step 1-7 正常分析 + 对标估值
```

### 发现不到同业怎么办

真正冷门或稀缺的标的（如港股 pure-play LLM MiniMax）同业少：
- 按**垂直替代**（该市场有哪些公司做类似事）
- 按**水平扩展**（美股同类 / 港股同类 / 全球同类）
- 至少保证 **≥ 3 家** 对标（否则估值对比失效）

### 红线

**未做同板块发现 = 违反铁律 -1 = 重做**。只对标 1 家 = 估值分析无意义（单点没锚）。

## 铁律 -0.5：多业务标的必须做"估值重塑归因"

**任何多业务标的（≥ 2 个业务线）的个股分析**，必须**拆解估值到各业务贡献**。

### 为什么是铁律

多业务标的常见陷阱：
- 市场把它当 "AI 股" 炒，但真正增速来自消费电子
- 叙事是"新业务转型"，但 90% 利润仍来自老业务
- 搞不清驱动块 → 无法判断估值合理性 → 无法识别核心风险点

**血泪案例**：东山精密 002384（2026-04 分析）
- 市场给 178 元估值（Forward PE 70-80x）
- 按中际旭创（全球光模块第一）逻辑估值
- 但索尔思光电实际规模 << 中际旭创
- **业务归因若缺失** → 只知道"贵"，不知道"贵在哪个业务兑现上"

### 归因 5 步法（必做）

#### Step 1：分部营收拆解
从年报"分部报告"章节抓：
- 每块业务营收占比 + 3 年变化
- 每块业务同比增速 + 环比趋势
- 每块业务毛利率

#### Step 2：边际毛利贡献拆解

```
各业务 YoY 营收增量 × 该业务毛利率 = 毛利增量贡献
贡献排序 → 真正驱动业绩的业务（不一定是营收最大的）
```

#### Step 3：叙事 vs 业绩对照（故事溢价识别）

| 匹配度 | 含义 | 风险 |
|-------|------|------|
| 一致 | 市场故事 = 实际贡献 | 低（估值有支撑）|
| 叙事 >> 业绩 | 炒作占上风 | **高（故事溢价，易崩）**|
| 业绩 >> 叙事 | 价值低估 | 中（可能 Alpha）|

#### Step 4：股价拉涨事件映射

每轮股价拉涨节点对应当时新闻：
- 哪块业务的消息最贴近涨停日 = 市场给估值的核心逻辑
- 如果股价涨了但没有该业务新闻 = 板块泛炒（Beta）

#### Step 5：板块联动交叉验证

```
个股涨幅 X%
PCB 板块龙头 A 涨幅 Y%
光模块板块龙头 B 涨幅 Z%
苹果供应链龙头 C 涨幅 W%

→ 和哪个最接近 = 市场归到哪块业务
```

### 归因产出格式（强制）

```markdown
## 估值重塑归因

| 业务 | 营收占比 | 同比增速 | 毛利率 | 估值贡献推演 | 依据 |
|------|---------|---------|-------|------------|------|
| 业务 A | X% | +Y% | Z% | **~N%** | ... |
| 业务 B | ... | ... | ... | ... | ... |

### 核心判断
- **主导驱动**：业务 X（贡献估值 N%）
- **叙事 vs 业绩匹配度**：一致 / 错配
- **故事溢价程度**：低 / 中 / 高

### 核心风险点
- 若业务 X 兑现不及预期 → 估值 -N%（对应股价回调 $X）
```

### 红线

多业务标的（≥ 2 业务）决策卡**缺少业务归因** = 违反铁律 -0.5 = 重做。

## 铁律 0：Step 0 必须抓当前时点股价（个股分析）

**任何涉及个股的命题，第一步必做**：

```
Step 0 时点校准（1 分钟，不做重来）
  ↓
  1. 抓今天的股价（WebSearch "<ticker> 今日股价 YYYY-MM-DD" 或直接 fetch 行情页）
  2. 记录数据时点：当前日期 + 股价 + 市值 + 当日涨跌
  3. 后续所有估值计算必须用这个时点的股价
  ↓
Step 1-N 才是正常分析流程
```

### 为什么这是铁律

**典型陷阱**：WebSearch 返回多篇文章，不同时点的股价同时出现：
- "某公司业绩发布时股价 X 元"（几天前）
- "某公司涨停收于 Y 元"（前天）
- "某公司最新 Z 元"（今天）

不校准时点 = **把历史股价当成当前股价算估值** = **所有 PE / 市值 / 对标全错**。

**血泪案例**：东山精密 002384（2026-04-22 分析）
- 误用 4-07 业绩快报时点股价 109 元
- 实际当日股价 178.80 元
- Trailing PE 真实值 236x，误算成 144x
- **结论方向虽对（溢价），但严重程度估错**
- 如果错向另一边（真实股价低于抓到的），可能得出完全错误的买入建议

### 数据时点校准规则

1. **所有股价数据必须标时点**（如：`股价 178.80 元（2026-04-22 14:30）`）
2. **跨时点数据必须换算**（如：4-07 的 PE ≠ 4-22 的 PE）
3. **市值 = 当前股价 × 总股本**，不用历史市值
4. **涉及盘中实时数据**时，声明延迟（WebFetch 的股价通常延迟 15 分钟+）

### 抓取源优先级（A 股为例）

| 优先级 | 源 | 备注 |
|-------|----|----|
| P1 | 东财 quote.eastmoney.com | 反爬弱，数据精确 |
| P1 | 新浪 finance.sina.com.cn/realstock | 稳定 |
| P2 | 雪球（仅查看快照页） | blacklist 但快照页可用 |
| P2 | Investing.com | 国际化源 |
| P3 | WebSearch + 最新日期关键词 | 兜底 |
| Tier 2 | 本地 PG 当日收盘价 | 零延迟且精确 |

**抓不到** = 明确告诉用户"当前股价时点缺失"，不硬编估值。

## 套路 A：个股分析

### 输入
- 股票名 / 代码
- 用户意图：要不要买 / 持仓/ 何时卖

### 数据抓取（5-10 分钟）

**必抓（按优先级）**：
1. **官方披露**：SEC EDGAR（美股）/ HKEXnews（港股）/ 巨潮（A 股）
   - 最近 10-K 或年报 → 业务结构 / 竞争描述 / 风险因素
   - 最近 10-Q 或半年报 → 当季业绩 + guidance
   - 最近 8-K → 重大事件
2. **IR 页面**：公司官网投资者关系
   - 最近一次业绩电话会文字稿
   - Investor Day PPT（若有）
3. **行情 + 估值**：
   - 当前股价 / 52 周高低 / 市值
   - PE / PS / PB / EV-EBITDA vs 同业 vs 历史
4. **同业对标**：找 3-5 家可比公司，抓同样指标
5. **卖方共识**：分析师一致预期（Yahoo Finance / Seeking Alpha 免费层）
6. **反面证据**：主动搜"<公司> bear case / 做空 / 风险 / 质疑"
7. **可选（rss 参考）**：`~/workspace/rss/daily/` 最近 7 天 brief，看社区近期讨论

**抓取工具组合**：
- WebSearch：关键词 + 最新日期限定
- WebFetch：直接读 SEC EDGAR / IR 页面 / 财报 PDF
- 结构化数据：Koyfin / TIKR / Stock Analysis 网页（若用户有订阅）

### 分析（按 alpha-thesis-card 7 问法）

1. 这是什么生意（business model + 财务轮廓）
2. 产业链位置（若 AI 主题标的，套 8 层地图）
3. 竞争格局 + 护城河
4. **预期差在哪**（核心 Alpha，最重要）
5. Bear case / 风险 / 压力测试
6. 估值锚 + 买卖触发价
7. 证伪条件 / kill criteria

### 输出

**填 `templates/alpha-thesis-card.md`**，存到 `docs/research/YYYY-MM-DD-<ticker>-thesis.md`。

**诚实边界**：
- 预期差 ≤ 2 → 明确告诉用户"当前无明显 Alpha，建议观望"
- Bear case 写不出 → 数据不足，不建议下注
- 估值已透支（PE 历史 90 分位）→ 减仓 / 不追高建议

## 套路 B：政策 / 事件驱动分析

### 输入
- 政策文件 / 监管事件 / 大会议题（CHIPS Act / 国务院 AI 行动纲要 / 美联储议息 / 中美谈判）
- 用户意图：找受益 / 受损标的

### 数据抓取

**必抓**：
1. **政策原文**（WebFetch 官方发布页）
   - 白宫 / 国务院 / 工信部 / FCC / 央行等
2. **权威解读**（2-3 篇深度文章）
   - 卖方大行报告（摘要）/ a16z 等 VC 博客
   - 避免 SEO 站（按 blacklist）
3. **历史类比**（该类政策过去如何影响股市）
   - 如 2022 CHIPS Act 对半导体股的冲击路径
4. **当前盘面反应**（已 price-in 多少？）
   - 主要受影响标的近 7 天涨跌

### 分析步骤

1. **政策核心诉求**：推什么 / 限什么 / 谁出钱 / 多大体量 / 时间表
2. **产业链影响地图**：
   - 直接受益（补贴对象 / 政策扶持对象）
   - 间接受益（下游 / 上游 / 配套）
   - 直接受损（被限制对象）
   - 间接受损（供应链切断 / 市场缩水）
3. **标的筛选**：每类至少 3-5 个标的，按市占 + 政策敏感度排序
4. **时点判断**：
   - 政策生效时间线
   - 盘面已 price-in 多少（看过去 7 天涨跌）
   - 苗头期 vs 共识期
5. **前 3 名深度分析**：对最受益的 3 个标的简版套路 A

### 输出

```markdown
# 政策分析：<政策名> YYYY-MM-DD

## 政策摘要（200 字内）

## 产业链影响地图
### 直接受益
| 标的 | 市占 | 受益路径 | 已 price-in | 建议 |
| ... | ... | ... | ... | 买/观望/不买 |

### 间接受益 / 直接受损 / 间接受损（同上结构）

## 前 3 名深度分析
（简版 alpha-thesis-card，每个标的 500 字内）

## 时点建议
- 抢跑窗口：... 天
- 进场价：...
- 止损：...
```

Layer 3: ... (T+30 传导)

- 最佳入场：T+X
- 预期兑现：T+Y

4. **估值水位对比**：
   - 同业中位 vs 历史分位
5. **卖方共识**：
   - 主题覆盖度（多少大行在推）
6. **情绪指标**：
   - Google Trends 关键词 YoY
   - 媒体覆盖密度

### 分析步骤

1. **主题当前阶段判断**（早期 / 中期 / 后期 / 晚期）：
   - 早期：论文 / 初创融资 → Alpha 最大但不确定性高
   - 中期：产业链成熟 + 龙头跑出来 → Alpha 中（当前 AI 所处）
   - 后期：应用落地 + 消费渗透 → Alpha 衰减
   - 晚期：平台固化 + 利润转移到头部 → 买 ETF 更稳
2. **Layer 逐层定位**：哪些 Layer 已 price-in，哪些 Alpha 未兑现
3. **标的排序**：按"精度 × 时点 × 仓位容量"三维
4. **建仓建议**：
   - 主题仓位总量（按用户风险偏好）
   - 单层 / 单股上限
   - 分批还是一次

## 诚实边界（不硬凑结论）

以下情况必须明确告诉用户 "**无法给出建议**"：

1. **数据不足**：关键一手数据抓不到（反爬 / 付费 / 未披露）
2. **预期差 ≤ 2**：分析后发现只是 Beta 不是 Alpha
3. **估值透支**：已到历史 95+ 分位，不值得追
4. **时间不对**：主题进入晚期 / FOMO 峰值
5. **风险超预期**：bear case 的最大亏损 > 用户风险承受

**不硬凑"买入" 或 "卖出" 结论**，观望也是决策。

## 执行检查单（每次分析前跑一遍）

### 基础流程
- [ ] 命题类型识别清楚（A/B/C/D 或组合）
- [ ] 按 framework 方法论执行（不偷懒）
- [ ] 最终输出决策卡（格式规范）
- [ ] 诚实 disclaimer（不构成投资建议）

### 数据深度（价值 1）
- [ ] 至少 **4 个独立源**（不是 4 篇同源转引）
- [ ] 至少 **2 个 T1-T2 原始源**（SEC / HKEXnews / IR / API）
- [ ] **反面证据**主动搜索并纳入
- [ ] 避开 blacklist（CSDN / 东财股吧 / 雪球 / 金色财经等）

### 分析深度（价值 2）
- [ ] 自己算估值（不抄卖方目标价）
- [ ] 自己推产业链 ≥ 2 跳
- [ ] 自己读财报（不看新闻摘要）
- [ ] 自己写 bear case（不等别人告诉你）
- [ ] **独立结论**（不做观点复读机）

### 预判未来（价值 3）
- [ ] 透过表象拆到里层（数据间关系 / 指引落差 / 沉默字段）
- [ ] 预判 T+3 / T+6 / T+12 / T+24 四个时间点
- [ ] 每个预判**可证伪**（具体 + 时点 + 触发动作）
- [ ] 预期差强度 ≥ 3（否则是 Beta 不是 Alpha）

### 红线（任一触发重做）
- [ ] 结论只来自卖方研报 → 重做（没独立分析）
- [ ] 未读任何原始披露 → 重做（没 T1 数据）
- [ ] 只讲现状不预判未来 → 重做（没前瞻价值）
- [ ] 未写 bear case → 重做（没压力测试）
