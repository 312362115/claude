# 命题驱动分析主流程

> **skill 的主入口**。用户触发一个命题，skill 抓数据+深度分析+输出决策卡。
> **手动触发**，不自动定时。
>
> 这是 finance-analysis skill 的核心工作流——**命题 → 分析 → 决策**。

---

## ⏳ 根本认知：时间维度 × 可预测性（短期不可测，长期反而可测）

**最反直觉的投资真相**：大多数人盯明天涨跌（预测不出），却不算半年后（反而能算）。

### 可预测性矩阵

| 时间尺度 | 主导因素 | 预测胜率 | 应做什么 |
|---------|---------|---------|----------|
| **1 天** | 噪音 + 游资 + 突发 | ~50%（抛硬币）| **放弃预测** |
| **1 周** | 情绪 + 资金流 | 50-55% | **放弃预测** |
| **1 月** | 板块轮动 | 55-60% | 关注催化剂不预测方向 |
| **3 月** | 财报 + 产业数据 | 60-65% | 可预判 Beat/Miss |
| **6 月** | 产业链传导 + 业绩 | **65-70%** | **可计算合理区间**|
| **1 年** | 预期兑现 | **70-75%** | **估值框架清晰** |
| **2 年** | 周期拐点 | **70-80%** | **产业逻辑锚定** |
| **5 年** | 均值回归 | **75-85%** | **最可预测**（但要拿得住）|

### 为什么短期不可测

```
明天股价 = 今天收盘
         + 今晚未知新闻
         + 明天游资情绪
         + 大盘情绪
         + 程序化交易
         + 对手方博弈
= 几乎随机
```

短期价格已反映所有公开信息，**唯一变量是未知噪音**，无法预测。

### 为什么半年反而可测

```
半年后股价 = 半年业绩预期 × 合理 PE × 兑现概率 × 主题溢价

每个变量都可算：
  - 业绩：财报指引 + 订单能见度 （~70% 确定）
  - PE：同业对标 + 历史（~80% 确定）
  - 催化剂：可列催化剂日历
  
→ 能给出区间（±15-20%）
```

**半年足够让业绩落地、订单披露、产业信号显性化**。

### Skill 纪律：只做中长期判断

**finance-analysis skill 的标准预判时点**：

| 维度 | 意义 | 强制输出 |
|------|------|---------|
| **T+3**（3 月） | 下季财报 Beat/Miss | 必做 |
| **T+6**（6 月） | 产业链传导 + 业绩 | 必做 |
| **T+12**（1 年） | 预期兑现度 | 必做 |
| **T+24**（2 年） | 周期拐点 | 必做 |
| ~~T+1 天~~ | 噪音 | **禁止**（除非是打新类确定性事件）|
| ~~T+1 周~~ | 情绪 | **禁止** |

**违反 = 冒充算命先生**。短期预测不是投资分析师的工作。

### 真正能做的 3 件事

1. **算"半年后合理区间"**（区间 ±15-20%）
2. **追踪"兑现进度"**（订单落地 / 财报达标 / 催化剂触发）
3. **识别"兑现路径偏差"**（实际 vs 预期差距）

**不是**：
- ❌ 预测明天涨跌
- ❌ 看 K线算技术位
- ❌ 猜主力下一步

### 操作频率 × 胜率 × 成本

```
频率越高 → 胜率越低 + 交易成本越高 = 长期必输
频率越低 → 胜率越高 + 成本越低 = 长期赢家

每天操作 ≈ 赌博
每月操作 ≈ 投机
每季操作 ≈ 投资
每年操作 ≈ 长期投资
持有 3-5 年 ≈ 价值投资
```

### 红线

- [ ] 任何分析**不预测明天涨跌 / 本周走势**
- [ ] 必给 T+3 / T+6 / T+12 / T+24 四个维度（每个可证伪）
- [ ] 用"**兑现进度追踪**"代替"价格预测"
- [ ] 催化剂日历 > K线分析

## 📈 根本认知：股市定价是预期驱动，不是当前业绩

**这是比所有铁律更根本的认知错误**。违反这条 → 所有分析都会向"当前业绩"对齐，完全错过股市运行逻辑。

### 核心公式

```
股价 ≠ 当前业绩 × PE
股价 = 未来 1-2 年预期业绩 × 合理 PE × 预期兑现概率 × 主题溢价
```

**股市永远在交易未来**，当前 PE 仅是参考，不是估值锚。

### 成长股典型生命周期

| 阶段 | 特征 | PE 变化 |
|------|------|---------|
| **第一阶段：预期打满** | 主题爆发，股价先涨 | Trailing PE 100x-500x+ |
| **第二阶段：业绩兑现** | 营收/净利快速追上 | PE 消化到 50-80x |
| **第三阶段：稳态增长** | 常规成长股 | PE 回到 30-40x |

**历史案例**：
- **NVIDIA 2022 Q4**：PE 150x "泡沫" → 2024 PE 40x（业绩涨 4 倍）
- **特斯拉 2020-2021**：PE 1000x+ → 2024 PE 60-80x（业绩消化）
- **宁德时代 2021**：PE 250x → 2023 PE 30x
- **亚马逊 1998-2000**：亏损但涨 20x → 10 年消化

**结论**：**高 PE ≠ 泡沫**。高 PE 是**预期提前定价**的正常现象，只有**预期无法兑现**才是真泡沫。

### 判断估值的正确方法：情景分析 + 概率加权

不做"静态 PE 对标"，做**预期驱动的情景分析**：

```
Step 1: 预判 2-3 年后的业绩（乐观/中性/悲观 三情景）
Step 2: 每个情景给合理 PE（龙头 / 成熟 / 新兴的 PE 锚）
Step 3: 算每个情景的合理市值 + 股价
Step 4: 每个情景给概率（主观估计，保守点）
Step 5: 概率加权得期望价
Step 6: 和当前股价对比 + 考虑主题溢价
```

### 估值映射模型

```
合理股价 = Σ(情景概率 × 情景股价) × 主题溢价因子

主题溢价因子：
  - 主线主题（如 2024-2026 AI）: 1.3-1.5x
  - 细分主题：1.1-1.3x
  - 无主题：1.0x
  - 逆主题：0.7-0.9x
```

### "隐含预期"分析（代替"泡沫"判断）

不说"股价泡沫 X%"，说"**股价隐含的是哪个情景的多少百分比**"：

| 隐含位置 | 含义 | 判断 |
|---------|------|------|
| 隐含**悲观情景 60%+** | 预期偏低 | **值得买入** |
| 隐含**中性情景 80-100%** | 预期合理 | 持有观望 |
| 隐含**乐观情景 70-100%** | 预期已打满 | **谨慎，等回调或更强催化剂** |
| 隐含**乐观情景 100%+ 主题溢价** | 情绪顶部 | **盈亏比差，减仓** |

### 血泪案例（东山精密 2026-04-22）

**错误分析**（静态 PE）：
> "PE 248x vs 中际 28x 溢价 800% → 泡沫 80-120% → 清仓"

**正确分析**（预期驱动）：
> **情景 1 乐观 30%**：2027 索尔思 200-300 亿营收 → 合理股价 90-140 元
> **情景 2 中性 50%**：Meta 到位 NVDA 部分 → 2027 100-150 亿 → 合理 50-80 元
> **情景 3 悲观 20%**：NVDA 没戏 + CPO 替代 → 合理 25-35 元
>
> 概率加权期望价 ≈ 73 元（基础）× 主题溢价 1.4 ≈ **100-140 元**
>
> 现价 186 隐含的是**乐观情景 + 主题溢价已打满 + 25% 额外透支**
>
> 盈亏比：上行 -25 to -50%，下行 -55 to -85%
>
> **判断**：不是"泡沫"，是"**预期已打满，盈亏比差，不值得追**"

### 盈亏比决策矩阵

```
| 上行空间 | 下行风险 | 盈亏比 | 决策 |
|--------|--------|-------|------|
| +100%   | -30%   | 3.3:1  | 强买 |
| +50%    | -30%   | 1.7:1  | 可买 |
| +30%    | -50%   | 0.6:1  | 观望 |
| -20%   | -70%   | 负    | 减仓 |
```

### 红线

**任何估值判断 必做**：
- [ ] 三情景分析（乐观/中性/悲观）+ 概率加权
- [ ] 考虑主题溢价因子
- [ ] 算"隐含预期位置"（不是"泡沫 %"）
- [ ] 算盈亏比（上行空间 vs 下行风险）

**违反**：用"泡沫 X%"单维判断 = 静态思维 = 重做。

## 🧠 最顶层思维方法：递归下钻（Iterative Deep Dive）

**在所有铁律之上的思维本质**：分析不是线性列 checklist，是**不断提问 → 回答 → 引出下一个问题**的递归过程。

### 为什么这是最顶层

- **铁律解决"做什么"**（Phase A/B、7 类资金、多源交叉）
- **递归下钻解决"怎么想"**（从表层答案追到里层事实）
- 没有递归下钻 → 分析停留在"复读新闻" 层面
- 有递归下钻 → 挖到数字事实 + 找到不可知变量

### 递归下钻的核心原则

```
任何答案都引出下一个"为什么"
直到触达以下 4 个终止条件之一：
  1. 数字事实（具体金额/产能/客户名单）→ 停,作为论据
  2. 不可知边界（如 NVDA 何时签单）→ 停,标记为"未知变量/关键催化剂"
  3. 枝叶无关项（如装修费）→ 停,避免过度下钻
  4. 既有知识（如"什么是 PCB"）→ 停,除非直接相关命题
```

### 下钻示范（东山精密 2026-04 案例）

```
Q0: 为什么涨 3 倍?
  └─ A: 索尔思子公司 + 光模块主题
       ├─ Q1: 光模块是什么?
       │   └─ Q1.1: AI 为什么需要光模块?
       │       └─ Q1.2: 带宽瓶颈多严重?
       │           └─ 【数字: 2026 年 1.6T 需求 2000+ 万只】
       ├─ Q2: 为什么机构扎堆?
       │   ├─ Q2.1: 哪些机构? 方向一致吗?
       │   │   └─ 【数字: 40+ 公募 vs 傅鹏博减仓 vs 游资 20 亿净卖】
       │   └─ Q2.2: 业绩真兑现吗?
       │       └─ 【数字: 光模块龙头净利 +40-108%】
       ├─ Q3: 索尔思具体干什么?
       │   └─ Q3.1: 和中际旭创差距?
       │       └─ 【数字: 规模 1/10-1/15】
       ├─ Q4: 真订单有吗? 客户是谁?
       │   ├─ Q4.1: Meta 订单明细?
       │   │   └─ 【数字: Q1 交付 30 万只 / Q2 月产 50 万只】
       │   ├─ Q4.2: NVDA 何时成单?
       │   │   └─ 【不可知: 关键催化剂未知】
       │   └─ Q4.3: 1.6T 订单哪家?
       │       └─ 【不可知: 北美头部云厂商未具名】
       ├─ Q5: 能支撑现价吗?
       │   └─ 【合理估值 85-140 元，现价 186 泡沫 30-120%】
       └─ Q6: 第一波 vs 第二波 驱动一样吗?
           └─ 【第一波主题，第二波业绩；第三波需 Q2 中报 + NVDA 订单】
```

### 下钻的 5 个典型方向

| 方向 | 从哪挖到哪 | 产出 |
|------|---------|------|
| **业务下钻** | 主业 → 分部 → 关键子公司 → 产品线 → 具体型号 | 子公司级尽调（见 A.2）|
| **行业下钻** | 大赛道 → 细分领域 → 产业链 → 关键环节 → 供需数字 | 产业链位置判断 |
| **客户下钻** | "大客户导入" → 客户名称 → 订单金额 → 交付进度 | 真订单 vs 故事 |
| **估值下钻** | PE 高/低 → 对标 → 折价系数 → 合理价位 | 估值锚 |
| **资金下钻** | "机构扎堆" → 哪类机构 → 方向一致吗 → 最新动向 | 7 类资金拆分 |

### 下钻的快速自检

每做一次分析，问自己：

- [ ] 有没有挖到**具体数字**（不是"大 / 多 / 显著"等形容词）?
- [ ] 有没有追到**关键子公司 / 业务块**（不停在集团层面）?
- [ ] 有没有识别**真订单 vs 洽谈中 vs 故事**?
- [ ] 有没有找到**不可知变量**（关键催化剂 / 未知风险）?
- [ ] 下钻到**足以给出买卖决策**的颗粒度了吗?

任何一项 ❌ → 继续下钻。

### 反模式（警惕）

❌ **停在表层**："机构看好 → 买"（没问哪些机构 / 方向一致吗 / 持仓变化）
❌ **引用而不深挖**："分析师目标价 92 元"（没问数据时效 / 聚合源 / 单家最新）
❌ **满足于新闻措辞**："索尔思导入大客户"（不挖具体客户 / 订单金额 / 交付进度）
❌ **过度下钻**：挖到与命题无关的枝叶（"索尔思办公楼在哪"）

### 好分析 vs 差分析

**差分析**（我之前犯的错）：
> "东山精密光模块业务爆发 + 公募扎堆买 → PE 248x 严重溢价 → 清仓"

**好分析**（递归下钻后）：
> "索尔思 Meta 真订单 $1 亿（已验证，Q1 交付 30 万只 = $X 营收）+ NVDA/AWS 还在送样（不可验证，标记为关键催化剂）→ 按业务分拆估值 85-140 元 → 186 元提前 price-in 2027 全部预期 → **减仓 40-50%**（锁利润，保留看 NVDA 订单落地）"

**差别**：
- 颗粒度（到美元金额 vs 笼统"爆发"）
- 真伪分辨（真订单 vs 送样）
- 确定性分层（已兑现 vs 关键催化剂）
- 决策可执行性（具体仓位比例 vs 一刀切）

## ⚡ 最高铁律：数据真实 + 有效 + 最新

> **数据是"1"，分析是"0"。"1"不到位，后面再多"0"都没意义。**

所有分析的基础前提是**数据质量**。违反任一条，**整个分析作废**，不是"打折"，是**失效**。

### 三条铁律（任一违反 = 重做）

#### 铁律 1：**真实**
- 数据必须来自原始披露源（SEC / HKEXnews / 巨潮 / 公司 IR / 官方 API）
- **不相信单源二手转引**（尤其是雪球/同花顺聚合的衍生数据）
- 关键数字至少 2 源交叉验证（若冲突，**以一手源为准**）

#### 铁律 2：**有效**
- 数字必须**精确到可计算**（不是"大约""左右""接近"）
- 缺字段必须**明确标注**（不硬编 / 不猜测 / 不用同业代入）
- 单位一致（市值 "亿" vs "亿美元" / 股价 "RMB" vs "HKD"）
- 口径一致（Trailing TTM / Forward / 本季度 / 同比 / 环比 必须明确）

#### 铁律 3：**最新**
- **股价必须是当天或前一交易日数据**
- **财报必须是最新发布的季报/年报**（不能用去年的数据分析今年）
- **宏观数据**（如 Capex 指引）必须是最新一季
- 每个数据点标 `【时点：YYYY-MM-DD HH:MM】`
- 跨时点数据**不可直接混用**（4 月的股价和 3 月的业绩不能简单除出 PE）

### 数据鲜度分级

| 数据类型 | 鲜度要求 | 超期处理 |
|---------|---------|---------|
| 股价 / 市值 | 当天（盘中）/ 前交易日（盘后）| 必须重抓，不降级 |
| 财报数据 | 最新发布的季报 / 年报 | 老财报不能代替新财报 |
| 行业动态 | 近 30 天 | 超期标"数据陈旧"警告 |
| 产业链数据（Capex等）| 近 3 月 | 超期标"数据陈旧"警告 |
| 历史对比数据 | 可用旧数据 | 但必须标时点 |

### 抓不到的处理（**穷尽多源再承认**）

**错误**：一次抓失败 → "数据不足不建议下注"（**太懒**）
**正确**：

```
源 1 失败 → 源 2 → 源 3 → 源 4 → ... → 穷尽 → 明确告诉用户"N 个源都试过，抓不到"
```

3. 专业报告（国金证券/中金公司 免费摘要）
4. 行业协会数据（信通院/艾瑞/TrendForce）

### 穷尽之后仍不行才承认边界

写明：
- 具体试过哪些源
- 每个源为什么失败（反爬 / 付费 / 无数据 / 停服）
- 告诉用户**需要手工补什么数据**才能继续

**这是职业分析师标准** —— "做不到" 必须有清单。

## 🎯 Skill 的灵魂：两个核心价值

**这两点是 skill 存在的理由**。做不到这两点 = skill 没价值，还不如让用户自己 Google。

### 价值 1：**数据深度**（原始数据优先，不抄）

**铁律：一手 > 二手 > 三手**。每份分析必须优先抓**原始数据**，而不是别人的综述。

#### 数据源分级（T1-T5）

| 级别 | 类型 | 例子 | 使用优先级 |
|------|------|------|----------|
| **T1 一手原始** | 公司披露 / 政府原文 / 交易所原始数据 | SEC 10-K 原文 / 央行货币政策原文 / HKEXnews PDF | **必抓** |
| **T2 结构化 API** | 机读数据 | FRED API / SEC EDGAR XBRL / BIS SDMX | **必抓** |
| **T3 权威媒体** | 专业财经记者深度报道 | 彭博 / 路透 / WSJ / 财新 | 辅助 |
| **T4 聚合分析** | 卖方摘要 / 咨询报告 | Seeking Alpha / 券商报告 PDF | 辅助（需验证） |
| **T5 社区观点** | 论坛 / 社交 | X / Reddit / 雪球 | **仅作情绪指标** |

#### 数据深度 3 条铁律

1. **能读财报原文就不要读新闻摘要**：新浪财经写"群核科技业绩转正" 3 秒看完，真正的 Alpha 在招股书第 100 页"研发费用率从 72.9% 降到 37.5%"的小字里。
2. **能拿 API 机读就不要看 PDF 截图**：SEC EDGAR 有 JSON API 可以批量抓 XBRL 字段，比人工截图读 10-K 快 100 倍且准确。
3. **能直接抓交易所就不要信二手聚合**：A 股看巨潮，港股看 HKEXnews，美股看 SEC EDGAR。Tushare / Yahoo Finance 是二手，有延迟、有错误、可能腐烂。

#### 数据源必须多源交叉

单源容易被骗（文字可以写得天花乱坠，数据也可以漂亮）。每份分析至少 **4 个独立源**交叉：

```
原始披露（T1-T2）     ← 主
+ 产业链旁证（T1-T3）  ← 验真
+ 同业对标（T1-T2）    ← 定位
+ 反面证据（主动搜）   ← 压力测试
```

### 价值 2：**分析深度**（独立推演，不抄观点）

**铁律：自己算 > 自己推 > 自己评**。不做"观点聚合器"。

#### 3 类偷懒的反面教材（绝对避免）

❌ **"5 家卖方都说买入，所以买入"** → 这是抄共识，不是分析
❌ **"新闻说群核涨 144%，因为 AI 概念"** → 这是复读机，不是分析
❌ **"同花顺研报推荐这只股"** → 这是转引，不是分析

#### 独立分析的 4 个动作

**动作 1：自己算估值**
- 不用卖方目标价，自己做相对估值 / DCF
- 对标 3-5 家同业，自己算 PE/PS/EV 中位数
- 算历史估值分位（当前 vs 过去 5 年）
- **案例**：群核 37x PS 首日收盘，卖方说"AI 溢价合理"。自己算：Autodesk 7.1x PS / 广联达 3.9x PS / 中望软件 12.8x PS → 37x 已经 3 倍于国内 AI 软件锚 → 严重溢价，不是"合理"。

**动作 2：自己推产业链（≥ 2 跳）**
- 不看"分析师报告说 CPO 受益"，自己推：ChatGPT → NVDA → HBM → CPO
- 每跳问"为什么"：为什么 AI 推理需要光模块？因为 GPU 集群互联带宽是瓶颈
- **案例**：群核单看是 SaaS 公司，但推 2 跳："Tesla Optimus 量产 → 需要具身智能训练数据 → 谁有 3D 场景？→ 群核"。这不是抄观点，是独立推演得出 Layer 6+7 双卡位。

**动作 3：自己读财报（不看摘要）**
- 10-Q 至少读 MD&A（Management Discussion）全段 + 财务附注
- 关注小字段：Accounts Receivable / Inventory / CapEx / Deferred Revenue
- 每次财报做 YoY + QoQ + 二阶导（加速/减速）
- **案例**：NVDA 2023-Q1 财报，新闻头条"Beat 预期"。真正 Alpha 在电话会"Q2 guidance $11B（consensus $7.1B）" —— 读原文的人当日加仓，看头条的人追高。

**动作 4：自己压测反面**
- 不等别人写 bear case，主动问"最坏会怎样"
- 至少推 3 种失败路径 + 量化下行幅度
- **案例**：群核 bear case 不是"AI 泡沫破"这种空话，而是"中国房地产持续下行 → 85% 收入受冲击 → 2026-H2 营收 -20%"。

#### 独立分析的边界

**不是"重新发明"**，是 "**别人的结论只作输入，结论必须自己得出**"：
- 可以参考卖方目标价（作为市场共识的 anchor）
- 可以看新闻（作为事件起点）
- 可以读别人的 bear case（补充自己没想到的）
- **但最终判断必须自己做**，不当观点复读机

### 价值 3：**透过表象 + 预判未来**（最高阶的 Alpha 生产）

**铁律：分析现状只是起点，预判未来才是终点**。任何分析最后必须回答"未来 6-24 个月会怎样"，不回答 = 没价值。

#### "透过表象"3 层拆解

金融市场里，信息有 3 层：

| 层 | 特征 | 示例（NVDA 2023 Q1） | Alpha |
|----|------|---------------------|------|
| **表层** | 媒体标题、情绪、涨跌 | "NVDA Q1 Beat 预期 10%" | 0（所有人都看到） |
| **中层** | 财报数据、指引、公告 | "Data Center 营收 $4.28B YoY +14%" | 小（读财报的人能看到） |
| **里层** | 数据间的异常、趋势拐点、隐藏信号 | "Q2 Guidance $11B（consensus $7.1B），+55% 跳空 → HBM 供应已锁死" | **大（这才是 Alpha）** |

**透过表象 = 从表层跳到里层**。3 个方法：

1. **看数据之间的关系**（不看单点）
   - 营收 YoY 加速 + 应收账款同比更快增 = 压货信号（警告）
   - 库存下降 + 交付周期延长 = 供应紧张（涨价信号！）
   - Capex 跳空 + R&D 占比升 = 新产品周期开启
2. **看指引和实际的落差**（预期管理）
   - 公司给保守指引但财报超 beat = 沙袋策略（好）
   - 公司给激进指引但勉强达标 = 已在顶部（警告）
3. **看沉默的字段**（什么没说）
   - 财报电话会突然不提某产品线 = 问题信号
   - 8-K 披露的时间点（周五盘后 vs 周一早盘）= 隐含态度

#### "预判未来" 4 个视角

每份分析必须预判以下 4 个时间维度的未来：

**1. 下季度（T+3 月）**
- 下次财报怎么走？Beat/Miss/In-line？
- 关键信号：Capex 指引 / guidance / 订单能见度

**2. 下半年（T+6 月）**
- 产业链传导完成没？A 股间接标的启动没？
- 关键信号：HBM 价格 / 光模块出货 / 客户扩产

**3. 一年（T+12 月）**
- 业绩兑现程度？估值 re-rating 空间？
- 关键信号：营收增速是否保持 / 毛利率趋势 / 新产品兑现

**4. 周期拐点（T+24 月）**
- 主题进入共识期还是衰减期？
- 关键信号：媒体覆盖 / 散户 FOMO / 卖方一致目标价

#### 预判的可证伪性（决定分析价值）

**无价值的预判**："AI 是未来 10 年最大机会" （放之四海，无法证伪）
**有价值的预判**："2026-Q3 NVDA Data Center 营收 YoY > +40%，低于此预期 → 清仓" （可证伪 + 有明确时点）

**所有预判必须满足**：
- ✅ 具体（数字 / 时点）
- ✅ 可证伪（X 未发生 → 论点错）
- ✅ 有行动意义（结果触发什么动作）

#### 经典案例：透过表象的 Alpha

**案例 1：CPO 2023-Q4**
- 表层：NVDA 涨了（所有人看到）
- 中层：NVDA 财报 Data Center +200%（读财报看到）
- **里层**：NVDA 财报中"HBM 供应紧张限制出货" 1 行字 → 推 2 跳到光模块 CPO（**Alpha**）

**案例 2：群核 0068.HK**
- 表层：IPO 首日 +144%（新闻看到）
- 中层：市占 23% + 数据飞轮（读招股书）
- **里层**：SpatialLM 开源登 HF 榜 + 智元/银河通用战略合作 → 具身智能数据层卡位（**Alpha**）

**案例 3：A 股存储股 2024-03**
- 表层：美光财报 Beat
- 中层：HBM 收入 +500%
- **里层**：美光电话会"HBM 卖光到 2025" + 产线扩张跟不上 → A 股兆易 / 佰维 15 天内启动（**Alpha 窗口期**）

## 核心原则（在三大价值之上）

1. **命题为王**：用户给什么命题，分析围绕什么转。不扩散不越界。
2. **数据驱动**：每个结论都要有数据支撑（T1-T2 为主），WebSearch/WebFetch 必选。
3. **决策导向**：最后必须输出"买/卖/持 + 目标价 + 止损 + 仓位"，不做"情况分析"就结束。
4. **诚实边界**：数据不足 / 论点弱 / 预期差 ≤ 2 → 明确说"不建议下注"，不硬凑结论。

## 铁律 -2：两阶段工作流 —— 先搞懂，再深入

**最顶层流程原则**：任何个股/多业务标的分析，**必须先完成 Phase A 基础画像，才能进 Phase B 深入分析**。跳阶段 = 瞎分析。

### Phase A：先搞懂（基础画像，7 步）

**不做完 Phase A 不能进 Phase B**。每步产出记录，Phase B 所有判断基于 Phase A 的数据。

| 步骤 | 任务 | 产出 | 失败处理 |
|------|------|------|---------|
| **A.1 时点校准** | 抓当前股价 / 市值 / 交易时间 | 明确时点的数字 | 穷尽多源（见铁律 0）|
| **A.2 业务画像（含关键子公司下探）** | 分部营收 + **驱动块的子公司级画像**（客户/订单/产能/竞争对手） | 分部表 + 子公司尽调 | 至少 3 年历史 + 关键子公司深挖 |
| **A.3 板块归类** | 属于哪些板块 / 产业链 Layer 位置 | 板块归属表 + Layer 图 | 主业 Layer + 新业务 Layer 都标 |
| **A.4 同业地图** | 主动发现 ≥ 5 家同业（见铁律 -1）| 同业候选 table | 筛 3-5 家做对标 |
| **A.5 历史估值区间** | 5 年 PE/PS/PB 分位数 | 当前 vs 历史分位 | 数据源：同花顺/东财 F10 |
| **A.6 股东结构（7 类资金拆分，必做）** | 见下方详细拆解 | 7 类资金力量对比表 | 用"机构"一个词不够，必须拆到资金性质 |
| **A.7 波次归因 + 数据时效再评估** | 识别过去 N 波上涨 + 每波驱动 + 判断历史数据是否失效 | 波次表 + 数据时效矩阵 | 短期大涨使历史数据失效 |

### Phase B：再深入分析

```
1. 事件驱动分析（若有财报/政策/并购，套 earnings-deep-dive）
2. 业务归因拆解（估值重塑归因 5 步法，见铁律 -0.5）
3. 同业对标估值（至少 3-5 家，龙头 + 直接对手 + 上下游）
4. 7 问法深度论证（套 alpha-thesis-card）
5. 4 维预判 + 证伪条件
6. 决策卡输出
```

### 两阶段的逻辑关系

```
Phase A 产出 → 喂给 Phase B
  ├─ A.1 时点 → B.3 估值锚（对标用当前股价）
  ├─ A.2 业务画像 → B.2 业务归因（拆到分部）
  ├─ A.3 板块 → B.3 对标（按板块找对手）
  ├─ A.4 同业地图 → B.3 对标候选池
  ├─ A.5 历史估值 → B.5 目标价（历史分位是参考锚）
  ├─ A.6 股东结构 → B.5 顶部信号（机构出货？）
  └─ A.7 事件时间线 → B.2 估值归因 Step 4（股价事件映射）
```

Phase A 任一失败 → Phase B 对应环节也失败（数据链断裂）。

### 红线

- Phase A 完成度 < 80% → **不得进 Phase B**
- Phase A 某步骤数据穷尽多源仍抓不到 → 明确标注 "缺失" 不猜测
- Phase B 的每个判断必须引用 Phase A 的具体数据点

### A.2 深化：驱动块的子公司级尽调（必做）

当标的的估值重塑来自**某个关键子公司/新业务**时，**必须下探到子公司做尽调**，不能停留在"分部营收 XX%"粗画像。

#### 必查 5 维度

| 维度 | 必须搞清楚 |
|-----|---------|
| **并购背景** | 什么时候收购、多少对价、并表时点（会影响会计数字）|
| **产品画像** | 具体产品规格 / 技术路线 / 护城河（如索尔思的"IDM 模式+自研芯片"）|
| **客户 & 订单** | 前 5 大客户？每个客户**真订单** vs **洽谈中**？金额？交付进度？|
| **产能 & 扩张** | 当前月产 / 计划月产 / 产能爬坡节奏 |
| **竞争对手** | 同产品的龙头是谁？本标的差距多少倍？差异化点？|

#### 区分"真订单" vs "故事"

**必须精确分辨**：
- ✅ **真订单**：已签合同 + 已交付数量 + 金额明确 → 可 price-in 估值
- ⚠️ **试单/试产**：小批量验证 + 未扩大 → 部分 price-in
- ❌ **送样/沟通**：无合同 + 无金额 → **不能 price-in**（市场常把这个当真订单炒作）

#### 血泪案例（东山精密索尔思 2026-04-22）

子公司下探拿到的真实信息：
- **真订单**：Meta 1 亿美元 800G（Q1 交付 30 万只，Q2 月产 50 万只）
- **洽谈中**：NVDA / AWS 送样沟通（**没订单**）
- **未披露**：1.6T 北美头部云厂商订单（有订单但规模不知）

→ 186 元股价已把"NVDA + AWS + 1.6T 全兑现"**提前 price-in**
→ 真实可确认的订单只支撑 1500-1900 亿市值（对应 85-105 元）
→ **泡沫 80-120%**

**如果未下探子公司**：只会得出"索尔思有大客户导入 + AI 光模块 = 买"的肤浅结论。

#### 数据来源

- 公司收购公告 / 并表公告（交易所披露）
- 子公司官网 / 客户 press release
- 行业媒体（讯石 / ofweek / LightCounting）
- 研报专章（"索尔思深度"类深度研报）
- 高管访谈 / 调研纪要

### A.6 股东结构 7 类资金拆解（必做，不拆视同未做）

**"机构"是笼统词，不同性质机构方向可能完全相反**。必须拆到资金性质：

| 类别 | 关键数据 | 解读 | 数据源 |
|------|---------|------|-------|
| **1. 公募基金 - 价值派大佬** | 睿远/中欧/易方达蓝筹 等重仓/减仓 | **最重要信号**（价值派判断基本面）| 基金季报 / 十大流通股东 |
| **2. 公募基金 - 主题派** | 新华/鹏华/永赢/万家/华夏 等重仓 | 跟主题不跟基本面 | 同上 |
| **3. 北向资金** | 陆股通持股量周度变化 | 外资长钱，相对理性 | 港交所披露易 / 东财北向数据 |
| **4. 主力资金（游资/席位）** | 单日净买入/卖出大 | 短期博弈，易在顶部出货 | 东财主力资金流 |
| **5. 社保/养老金/QFII** | 长期资金增减 | 极稳定长线资金 | 季度披露 |
| **6. 大股东/董监高** | 增持/减持公告 | 内部人信号 | 交易所 SDI 披露 |
| **7. 股东户数变化** | 集中度提升/分散 | 筹码结构 | 季度披露 |

### 关键：识别"机构内部分裂"

7 类资金方向可能**完全相反**：
- 价值派减仓 + 主题派加仓 = **最后一波行情**（顶部特征）
- 价值派加仓 + 主题派减仓 = **价值回归起点**（底部特征）
- 北向 + 社保加仓 = **长线看好**（最强信号）
- 游资撤 + 公募进 = **筹码转换，稳定正向**
- 内部人减持 + 外部机构追 = **警报信号**（内部人最懂）

**血泪案例**：东山精密 002384（2026-04 分析）
- 只看"4-14 主力净卖 10 亿" → 误判为"机构全面出货"
- 真实情况：傅鹏博（价值派）减，但主题派仍重仓（40+ 只基金）
- **三股资金博弈** → 结论应为"聪明钱已走，主题资金最后一波"，减仓 50-60% 不是全清

### 卖方共识识别（必做交叉验证）

**聚合目标价 ≠ 卖方共识**。任何卖方目标价数据必须**多维验证**：

#### 3 层数据校验

1. **区分"聚合均价" vs "最新单家研报"**
   - Investing.com / Yahoo Finance 的 "12-month average analyst target price" = **未来 12 个月目标价**（聚合过去 3-6 月研报均值，**不是 12 月的数据**）
   - 这种聚合被**所有历史研报**拖累
   - **高波动期间完全失效**——股价 3 个月翻倍时，聚合均价里还混着翻倍前的研报目标价
   - **任何"目标价 vs 现价差距 > 50%"的情况**，大概率是聚合滞后而非卖方看空
   - **判断阈值**：股价近 3 月涨幅 > +50% → 聚合均价禁用，**只看 30 天内单家研报**

2. **必查最新单家研报**
   - 搜索"<ticker> 券商 最新研报 2026"
   - 关注近 30 天有没有新研报出来
   - 最新研报 vs 聚合均价 的落差 = 市场分歧度

3. **区分国内 vs 国际视角**
   - 国内券商（中信/中金/华泰/国金/华创等）：懂 A 股炒作逻辑
   - 国际研究（Morgan Stanley/Goldman/UBS）：按财务模型估值
   - 同一标的两边目标价可能差 3 倍

#### 必问 3 个问题

1. **数据时点**：目标价是什么时候给的？（当前股价对应当时股价的多少倍？）
2. **覆盖券商**：聚合了哪几家？国内 vs 国际比例？
3. **趋势**：近 90 天目标价在上调还是下调？

#### 血泪案例（东山精密 2026-04-22）

- Investing.com 聚合均价 **92.17 元**（股价 186 的 -50%）
- 近 90 天机构均价 **62 元**（严重滞后）
- **华创证券 2026-04-17 最新目标价 199.87 元**（股价 186 的 +7%）

**直接用聚合 92 元作"共识下行 50%"论据 = 严重误导**。真实情况：国内最新研报仍强推，根本没有看空共识。

#### 红线

任何涉及"卖方目标价/共识"的论据，**未做 3 层验证 = 禁用**。

## 铁律 -0.3：对标必须深挖产业链位置（龙头溢价 vs 跟风溢价）

**对标不能只对标财务数字**，必须先搞清**每家在产业链的位置 + 护城河差异**。

### 3 类对标陷阱

1. **"龙头当次选"陷阱**：把次线公司按龙头估值给（如东山精密按中际旭创 PE 估值，实际规模差 10 倍）
2. **"同行业当同业务"陷阱**：都是 PCB 但 AI 服务器 PCB 和消费电子 PCB 估值差几倍
3. **"主题炒作 = 业绩成长"陷阱**：主题基金扎堆不等于真成长，要看订单+产能+客户

### 对标 4 维深挖

对每个对标标的查：

| 维度 | 问什么 |
|-----|-------|
| **产业链位置** | 龙头 / 次龙头 / 跟随者？市占率多少？ |
| **护城河** | 技术（光芯片自研？）/ 客户绑定（Apple？NVDA？）/ 规模？ |
| **业绩锚** | 最新季度营收 + 同比 + 毛利率 |
| **估值锚** | Forward PE + 相对历史 + 行业中位 |

### 估值映射公式

```
本标的合理估值 = f(产业链位置, 护城河深度, 业绩锚, 估值锚)

如果按龙头估值：
  合理股价 = 龙头 Forward PE × 本标的 EPS × 折价系数（0.5-0.8，按规模/护城河差距）

如果按次线估值：
  合理股价 = 次线 Forward PE × 本标的 EPS

血泪案例：东山精密按中际旭创估值
- 中际 Forward PE 28x，东山 Forward PE 75-85x
- 规模差 10 倍 → 折价系数应该 ~0.5-0.6
- 合理 PE = 28x × 0.5-0.6 = 14-17x ❌ 太低
- 或按"次线 + 差异化"给 PE 40x → 合理股价 95-120
- 现价 186 = 对应 PE 75x → 严重透支
```

### 红线

对标**未做产业链位置分析** = 违反铁律 -0.3 = 重做。只看财务数字对 PE 就下结论 = **典型的表层分析**。

## 铁律 -0.6：不熟悉标的必须先做"全业务线 × 主题映射矩阵"（枚举，不要靠直觉）

**估值归因的前提是业务线已经被完整枚举**。但"完整枚举"不能靠直觉——人脑（包括 LLM）天然会盯着"最近涨幅主因"、"最大营收来源"看，**系统性漏掉"稳定/承压"业务下隐藏的第二条 Alpha 线**。

### 为什么是铁律

**血泪案例**：东山精密 002384（2026-04-22 分析）
- 第一轮分析只盯住"索尔思光模块"这条 AI 线
- 把"电子电路产品（PCB/FPC）256.2 亿、+3.3%、毛利率下滑"一笔带过为"主业平淡"
- **完全漏掉 Multek 子公司是英伟达 GB200/H200 背板核心供应商**——第二条强 Alpha 线
- 错误结论：Forward PE 75-85x，透支 30-85%
- 用户提醒"PCB 升级 + 800G 光模块双重受益"后重新调研
- 修正结论：华创口径 Forward PE 42x，定价接近公允
- **偏差源于业务线枚举不完整，不是数据抓取不够**

**不熟悉标的的系统性风险**：如果分析师对公司熟悉度低（没有预先知道 Multek 是干嘛的），这条铁律不走就一定漏。

### 强制执行：5 步全业务线覆盖流程（Phase A Step 1 前置）

#### Step 1.1：业务线矩阵枚举（穷举，不是摘要）

从年报"分部报告"或业务介绍章节抓**全部业务线**，包括：
- 母公司分部
- 控股子公司（尤其是**近 5 年并购来的**——容易被当"小业务"忽视）
- 参股公司（如果披露）
- 历史并购资产（如 Multek 这种老并购，容易忘记它的独立身份）

格式（必填）：

| 业务/子公司 | 营收规模 | 同比 | 毛利率 | 主要产品 | 主要客户 | 并入时间 |
|-----------|---------|-----|-------|---------|---------|---------|
| 主业 A | ... | ... | ... | ... | ... | 自有 |
| 子公司 X | ... | ... | ... | ... | ... | YYYY 收购 |

**漏项 = 违反铁律 -0.6 = 重做**。

#### Step 1.2：每条业务线 × 当前市场主题映射

列出当前市场正在炒的主题（AI / 机器人 / 新能源 / 军工 / 国产替代 / ...），**每条业务线 × 每个主题**交叉检验：

| 业务 | AI | 机器人 | 新能源 | 国产替代 |
|------|----|-------|-------|---------|
| 业务 A | Y（具体受益点） | N | N | 待验 |
| 子公司 X | Y（具体受益点） | N | ? | N |

**默认假设每条业务线都可能有主题敞口**，逐条检验否定或确认，不要预设"只有新业务与主题相关"。

#### Step 1.3：子公司穿透三问（尤其对收购资产）

对每个子公司，**必答三问**：

1. **当前产品线是什么**？（不是收购时的产品——产品线会演进，如 Multek 从传统多层板升级到 78 层+ 高多层 AI 背板）
2. **当前客户清单是谁**？（尤其检查是否包含 NVDA / Google / AWS / Meta / Microsoft / xAI / OpenAI 等 AI 主线客户）
3. **所在行业近 2 年是否发生结构性变化**？（AI 爆发、Capex 暴涨、国产替代、政策驱动……）

三问至少有一个答"是"→ 该子公司可能是隐藏 Alpha 线，必须单独深挖。

#### Step 1.4：同业反向映射

列出同赛道（PCB / 光模块 / 半导体封测 / 汽车电子 / ...）已知受益主题的头部公司，**回头检验自家是否也有同样敞口**：

```
同赛道受益公司 → 他们的受益业务线 → 自家是否也有这条线 → Y/N/待查
```

示例（东山精密 PCB）：
- 沪电股份：AI 服务器 PCB（NVDA GPU 板）→ 东山 Multek：英伟达背板（Y，且是核心供应商）
- 胜宏科技：AI HDI 扩产 → 东山 Multek：78 层+ 高多层扩产（Y，珠海 100 亿）
- 鹏鼎控股：AI 服务器 HDI → 东山：硬板敞口待验

#### Step 1.5：券商预测分歧检测（> 1.5 倍必拆）

抓 3 家以上券商的次年净利预测，**最高值 / 最低值 > 1.5 倍时强制拆解**：

- 差异源于哪条业务线的不同假设？
- 差异源于哪个客户订单转化概率的不同？
- 差异源于 PE 倍数给得不同，还是业绩基数给得不同？

**分歧本身是 Alpha 线清单的信号**——券商 A 给 80 亿、券商 B 给 33 亿（2.4 倍差距），很可能是因为两家认的"业务线清单"不同。

示例（东山精密）：
- 华创 2026E 80 亿：算了 Multek AI PCB + 索尔思光模块双引擎
- 国联 2026E 33 亿：只算了传统 PCB 小幅增长 + 索尔思有限贡献
- **差距 2.4 倍 = 对 Multek AI 敞口的判断差 20-30 亿净利**

### 与铁律 -0.5 的关系

**铁律 -0.6（本条）**= 穷举业务线（先把表填完整）
**铁律 -0.5（估值归因）**= 对已枚举业务线做毛利贡献归因

顺序不能颠倒：没完成 -0.6，-0.5 的归因建立在残缺的业务清单上，结论必错。

### 红线

- 分析个股但没有完整业务线矩阵 = 违反铁律 -0.6 = 重做
- 业务线未做主题映射检验 = 违反铁律 -0.6 = 重做
- 子公司未做穿透三问 = 违反铁律 -0.6 = 重做
- 券商预测分歧 > 1.5 倍未拆解业务线根因 = 违反铁律 -0.6 = 重做

## 铁律 -0.5：多业务标的必须做"估值重塑归因"

**任何多业务标的（≥ 2 个业务线）的个股分析**，必须**拆解估值到各业务贡献**。

### 为什么是铁律

多业务标的常见陷阱：
- 市场把它当 "AI 股" 炒，但真正增速来自消费电子
- 叙事是"新业务转型"，但 90% 利润仍来自老业务
- 搞不清驱动块 → 无法判断估值合理性 → 无法识别核心风险点

**血泪案例**：东山精密 002384（2026-04 分析）
- 市场给 178 元估值（Forward PE 70-80x）
- 按中际旭创（全球光模块第一）逻辑估值
- 但索尔思光电实际规模 << 中际旭创
- **业务归因若缺失** → 只知道"贵"，不知道"贵在哪个业务兑现上"

### 归因 5 步法（必做）

#### Step 1：分部营收拆解
从年报"分部报告"章节抓：
- 每块业务营收占比 + 3 年变化
- 每块业务同比增速 + 环比趋势
- 每块业务毛利率

#### Step 2：边际毛利贡献拆解

```
各业务 YoY 营收增量 × 该业务毛利率 = 毛利增量贡献
贡献排序 → 真正驱动业绩的业务（不一定是营收最大的）
```

#### Step 3：叙事 vs 业绩对照（故事溢价识别）

| 匹配度 | 含义 | 风险 |
|-------|------|------|
| 一致 | 市场故事 = 实际贡献 | 低（估值有支撑）|
| 叙事 >> 业绩 | 炒作占上风 | **高（故事溢价，易崩）**|
| 业绩 >> 叙事 | 价值低估 | 中（可能 Alpha）|

#### Step 4：股价拉涨事件映射

每轮股价拉涨节点对应当时新闻：
- 哪块业务的消息最贴近涨停日 = 市场给估值的核心逻辑
- 如果股价涨了但没有该业务新闻 = 板块泛炒（Beta）

#### Step 5：板块联动交叉验证

```
个股涨幅 X%
PCB 板块龙头 A 涨幅 Y%
光模块板块龙头 B 涨幅 Z%
苹果供应链龙头 C 涨幅 W%

→ 和哪个最接近 = 市场归到哪块业务
```

### 归因产出格式（强制）

```markdown
## 估值重塑归因

| 业务 | 营收占比 | 同比增速 | 毛利率 | 估值贡献推演 | 依据 |
|------|---------|---------|-------|------------|------|
| 业务 A | X% | +Y% | Z% | **~N%** | ... |
| 业务 B | ... | ... | ... | ... | ... |

### 核心判断
- **主导驱动**：业务 X（贡献估值 N%）
- **叙事 vs 业绩匹配度**：一致 / 错配
- **故事溢价程度**：低 / 中 / 高

### 核心风险点
- 若业务 X 兑现不及预期 → 估值 -N%（对应股价回调 $X）
```

### 红线

多业务标的（≥ 2 业务）决策卡**缺少业务归因** = 违反铁律 -0.5 = 重做。

## 铁律 0：全量数据时点校准（不只是股价）

**任何涉及个股的命题，第一步必做全量数据时点校准**——股价、财务数据、分析师共识、订单/客户信息，**四类数据都要标注时点并使用最新可得版本**。下游所有估值、归因、决策环节都依赖上游数据，任何一项过时，全链路污染。

```
Step 0 全量时点校准（3-5 分钟，不做重来）
  ↓
  1. 股价时点：今日收盘 / 实时，标注时点
  2. 财务数据时点：最新季报 > 半年报 > 年报，绝不用隔年年报
  3. 分析师数据时点：最近 90 天单独研报 > 聚合均价，高波动期聚合必失效
  4. 订单/客户信息时点：最新调研纪要 > 年报 > 旧研报
  ↓
Step 1-N 才是正常分析流程
```

### 为什么这是铁律——三次血泪案例

**案例 1｜股价时点错**（东山精密 002384，2026-04-22）
- 误用 4-07 业绩快报时点股价 ¥109
- 实际当日股价 ¥186.34
- Trailing PE 真实值 236x，误算成 144x
- 结论方向虽对，严重程度估错

**案例 2｜聚合目标价失效**（东山精密，同日）
- 引用 Investing.com 聚合目标价 ¥92 作为"国际聚合看空"证据
- 实际该聚合来自股价 ¥60-100 区间的历史研报
- 最新单独研报（华创 2026-04-17）目标价 ¥199.87
- **误导为"国内看多 vs 国际看空"的假分歧**，真实是同向看多

**案例 3｜财务数据 + 分析师数据双错**（炬光科技 688167，2026-04-22）
- 业务分部数据用 **2024 年报**（营收 6.20 亿）——实际 **2025 前三季度已达 6.13 亿**，光通信分部 +109%、Q3 扭亏
- 分析师共识目标价用 Investing.com **¥65.40**（陈旧聚合）——实际申万宏源 2026-03-16 目标价 **¥410**（与现价 ¥393 基本一致）
- 错误结论"分析师 ¥65 vs 市场 ¥393 = 6 倍极端高估"——真相是市场与主流券商定价一致，**假分歧完全由数据过时制造**
- 真分歧在**券商之间** 2026E 净利 0.30-2.47 亿（8.2 倍），拆解业务线根因才是正题

**核心教训**：聚合数据在高波动期会系统性失效（聚合均价来自 3-6 个月甚至 12 个月的研报平均，股价快速波动后必然沦为"历史化石"）。

### 核心原则：每个数据引用都必须先确认时点（铁律 0 心脏）

**每次引用任何具体数字（股价 / 财务 / 目标价 / 订单规模 / 市场份额 / 客户份额 ...）之前，必须先确认这个数字对应的具体日期**。

```
不确认日期 → 禁用该数字
查不出日期 → 禁用该数字
```

**WebSearch 返回的研报 / 新闻 / 文章片段里的数字**（"沪电当前股价 ¥81.18"、"市值 330 亿"、"份额 70%" ...）：

- 片段本身不带日期 → **一律视为"日期未知"，禁止作为当前数据使用**
- 片段带日期但 > 7 天 → 股价 / 分析师目标价 / 订单数据禁用；仅财务数据可用但必须标"X 日期数据"
- 片段带日期且 ≤ 7 天 → 可用，但必须在引用处标出日期

**落地铁律**：
1. **股价：永远走实时行情接口抓**（见下方"股价时点"），不允许从研报片段中提取股价
2. **财务数据：必须确认是哪一期（年报 / 季报 / 预告）**，且必须校验"还有没有更新的一期"
3. **目标价：必须确认是哪家券商 / 哪一天的研报**，聚合价高波动期禁用
4. **订单/客户数据：必须确认是哪份调研纪要 / 哪次业绩会**

**沪电血泪案例**（2026-04-24）：
- AI 板块扫描时把 WebSearch 返回的"沪电股份 ¥81.18"片段当当前价使用
- 实际该片段是 2-3 月的研报引用价，**真实 4-23 收盘价 ¥106.62**
- 偏差 31%，导致 Forward PE 误算为 30x（真实 40x），错误结论"Top 1 建仓推荐"
- 根因：没有对每个数据做"这是哪一天的"独立校验，直接引用了 WebSearch 片段
- 修复后：股价数据统一走 `scripts/quote.sh`（新浪行情接口）直抓实时价

### 四类数据的时点校准规则

#### 1. 股价时点

| 规则 | 说明 |
|------|------|
| 必标时点 | `股价 ¥186.34（2026-04-22 14:30）` |
| 跨时点必换算 | 4-07 的 PE ≠ 4-22 的 PE |
| 市值算法 | 当前股价 × 总股本，不用历史市值 |
| 实时数据延迟 | WebFetch 行情页通常延迟 15 分钟+，须声明 |
| **禁止二手转引** | **禁止从 WebSearch 研报 / 新闻片段提取股价**——片段里的价都是写稿日期的历史价 |
| **强制走接口** | 每只标的必须通过实时行情接口单独抓，见下方 P1 源 |

**抓取源优先级**：

| 优先级 | 源 | 用法 |
|-------|----|----|
| **P0** | **本 skill `scripts/quote.sh`**（封装新浪 `hq.sinajs.cn`）| **默认首选，批量抓取带时点戳，A/H/美股通用** |
| P1 | 东财 quote.eastmoney.com / 新浪财经行情页 | 快速 WebFetch 个股行情页 |
| P1 | 同花顺 basic.10jqka.com.cn | 含盈利预测 |
| P2 | Investing.com / Yahoo Finance | 海外标的 |
| P3 | WebSearch 行情快讯 | 仅作交叉验证，不作首选 |
| ❌ | **研报 / 新闻 / 深度分析文章里引用的股价** | **历史价，禁止当当前价用** |
| Tier 2 | 本地 PG 量化栈 | 批量历史 K 线场景 |

**脚本使用**（见 `scripts/quote.sh`）：

```bash
# 批量抓取 A 股 + 港股实时行情（含日期时间戳）
bash skills/finance-analysis/scripts/quote.sh sz002463 sh601985 rt_hk00068
```

返回字段固定（新浪格式）：名称 / 昨开 / 昨收 / **当前价** / 今高 / 今低 / ... / **日期** / **时间**。读取时必须**校验日期戳是否为当日或上一交易日**，否则数据陈旧。

#### 2. 财务数据时点（新增）

**铁律**：**宁可用最新季报的不完整数据，也不用过时年报的完整数据**。

| 优先级 | 数据源 | 适用 |
|-------|-------|------|
| P1 | 最新季报（年初至报告期末）| **默认首选** |
| P1 | 最新业绩快报 / 预告 | 尚未披露季报时 |
| P2 | 最新半年报 | 季报缺失时 |
| P3 | 最新年报 | 只有跨年度对比才用 |
| ❌ | 隔年年报 | **禁用**（如 2026-04 分析用 2024 年报） |

**关键原则**：
- 分析 2026-04 的个股，优先级应为：**2026 Q1 预告 > 2025 年报 > 2025 Q3 报 > 2025 半年报 > 2024 年报**
- 如 2025 年报未发但 2025 Q3 已披露，**必须用 2025 Q3** 而非 2024 年报
- 并购标的：**并表时点**要单独标注（如炬光瑞士炬光 2024-01 并购，2024 年报只并了 11 个月）
- 高增长分部（如光通信 +109%）**必须用最新季报**，年报滞后半年以上即失真

**校验动作**：每次引用财务数据前，必问"**这是哪一期？有没有更新的？**"

#### 3. 分析师数据时点（新增，关键）

**铁律**：**高波动期（股价 3 个月内 ±30%+）禁用聚合目标价**——聚合是历史平均，股价快速波动时必然失效。

| 类型 | 时点要求 | 使用方式 |
|------|---------|---------|
| 单独券商研报目标价 | **最近 90 天内**（高波动期 30 天内）| 作为核心锚 |
| 聚合平台目标价（Investing.com / 雪球聚合 / 同花顺聚合）| 股价 3 个月内波动 < 20% 才可用 | **高波动期必失效** |
| 盈利预测均值 | 同花顺 F10 / 东财 F10 最新 | 不用 Investing 聚合 |
| 盈利预测分歧 | 拉**最高值 / 最低值 / 均值**三档 | 用于铁律 -0.6 Step 1.5 |

**5 秒判断聚合是否失效**：
- 聚合目标价相对现价偏离 > 50% → **大概率失效**
- 股价近 3 个月涨幅 > 30% 或跌幅 > 20% → **聚合一定失效**
- 聚合里最新研报日期 > 6 个月前 → **作废**

**抓取源优先级**：
| 优先级 | 源 | 说明 |
|-------|----|----|
| P1 | 同花顺 F10（basic.10jqka.com.cn/{code}/worth.html）| 中文聚合，时点清晰 |
| P1 | 东财 F10 盈利预测 | 时点精细 |
| P1 | 发现报告（fxbaogao.com）| 单独研报全文 |
| P2 | 各券商官网最新研报 PDF | 单独研报 |
| ❌ | Investing.com 聚合目标价 | **高波动期禁用** |

#### 4. 订单/客户信息时点（新增）

**铁律**：客户结构和订单状态快速变化，**年报披露的客户信息过时风险极高**。

| 优先级 | 数据源 | 说明 |
|-------|-------|------|
| P1 | 最新投资者调研纪要 | 含 NVDA/Google/AWS 等具体动态 |
| P1 | 最近 3 个月券商深度研报 | 含具体订单进展 |
| P2 | 最新业绩说明会披露 | 口径官方 |
| P3 | 最新季报 MD&A 章节 | 滞后 1-2 个月 |
| ❌ | 上年年报"前五大客户" | 基本失效（客户结构年变）|

**典型陷阱**：
- 年报"激光雷达定点 50 万套"→ 实际 2026-04-03 已取消（炬光案例）
- 年报"主要客户 X"→ 最新调研纪要里客户已换

### 全量校准产出格式（强制）

分析报告首段必须包含时点块：

```markdown
## 数据时点校准（Step 0 输出）

- **股价**：¥393.00（2026-04-22 14:59，+4.63%，市值 330.80 亿）
- **财务数据**：2025 年三季报（2025-10-30 披露，前三季度营收 6.13 亿）
- **业绩预告**：2026-01-29 披露 2025 全年预亏 3200-4200 万
- **分析师数据**：盈利预测均值 2026E 1.01 亿 / 最高 2.47 亿 / 最低 0.30 亿（同花顺 F10，2026-04）；单独目标价最新 ¥410（申万宏源 2026-03-16）
- **订单信息**：2025-09 投资者调研纪要（CPO V 型槽样件交付中、N×N 透镜阵列小批量出货）
- **失效数据**：Investing.com 聚合目标价 ¥65.40 ❌（高波动期失效，不使用）
```

### 红线

- 分析 2026 年用 2024 年报 = 违反铁律 0 = 重做
- 高波动期（股价 3 月内 ±30%+）用聚合目标价 = 违反铁律 0 = 重做
- 引用财务数据未标时点 = 违反铁律 0 = 重做
- 任何数据无时点标注 = 违反铁律 0 = 重做
- **引用数字前未确认"这是哪一天的" = 违反铁律 0 = 重做**（沪电血泪）
- **从 WebSearch 研报 / 新闻片段里提取股价当当前价用 = 违反铁律 0 = 重做**（必须走 `scripts/quote.sh`）
- **抓不到最新** = 明确告诉用户"某某数据时点缺失"，不硬编估值

## 套路 A：个股分析

### 输入
- 股票名 / 代码
- 用户意图：要不要买 / 持仓/ 何时卖

### 数据抓取（5-10 分钟）

**必抓（按优先级）**：
1. **官方披露**：SEC EDGAR（美股）/ HKEXnews（港股）/ 巨潮（A 股）
   - 最近 10-K 或年报 → 业务结构 / 竞争描述 / 风险因素
   - 最近 10-Q 或半年报 → 当季业绩 + guidance
   - 最近 8-K → 重大事件
2. **IR 页面**：公司官网投资者关系
   - 最近一次业绩电话会文字稿
   - Investor Day PPT（若有）
3. **行情 + 估值**：
   - 当前股价 / 52 周高低 / 市值
   - PE / PS / PB / EV-EBITDA vs 同业 vs 历史
4. **同业对标**：找 3-5 家可比公司，抓同样指标
5. **卖方共识**：分析师一致预期（Yahoo Finance / Seeking Alpha 免费层）
6. **反面证据**：主动搜"<公司> bear case / 做空 / 风险 / 质疑"
7. **可选（rss 参考）**：`~/workspace/rss/daily/` 最近 7 天 brief，看社区近期讨论

**抓取工具组合**：
- WebSearch：关键词 + 最新日期限定
- WebFetch：直接读 SEC EDGAR / IR 页面 / 财报 PDF
- 结构化数据：Koyfin / TIKR / Stock Analysis 网页（若用户有订阅）

### 分析（按 alpha-thesis-card 7 问法）

1. 这是什么生意（business model + 财务轮廓）
2. 产业链位置（若 AI 主题标的，套 8 层地图）
3. 竞争格局 + 护城河
4. **预期差在哪**（核心 Alpha，最重要）
5. Bear case / 风险 / 压力测试
6. 估值锚 + 买卖触发价
7. 证伪条件 / kill criteria

### 输出

**填 `templates/alpha-thesis-card.md`**，存到 `docs/research/YYYY-MM-DD-<ticker>-thesis.md`。

**诚实边界**：
- 预期差 ≤ 2 → 明确告诉用户"当前无明显 Alpha，建议观望"
- Bear case 写不出 → 数据不足，不建议下注
- 估值已透支（PE 历史 90 分位）→ 减仓 / 不追高建议

## 套路 B：政策 / 事件驱动分析

### 输入
- 政策文件 / 监管事件 / 大会议题（CHIPS Act / 国务院 AI 行动纲要 / 美联储议息 / 中美谈判）
- 用户意图：找受益 / 受损标的

### 数据抓取

**必抓**：
1. **政策原文**（WebFetch 官方发布页）
   - 白宫 / 国务院 / 工信部 / FCC / 央行等
2. **权威解读**（2-3 篇深度文章）
   - 卖方大行报告（摘要）/ a16z 等 VC 博客
   - 避免 SEO 站（按 blacklist）
3. **历史类比**（该类政策过去如何影响股市）
   - 如 2022 CHIPS Act 对半导体股的冲击路径
4. **当前盘面反应**（已 price-in 多少？）
   - 主要受影响标的近 7 天涨跌

### 分析步骤

1. **政策核心诉求**：推什么 / 限什么 / 谁出钱 / 多大体量 / 时间表
2. **产业链影响地图**：
   - 直接受益（补贴对象 / 政策扶持对象）
   - 间接受益（下游 / 上游 / 配套）
   - 直接受损（被限制对象）
   - 间接受损（供应链切断 / 市场缩水）
3. **标的筛选**：每类至少 3-5 个标的，按市占 + 政策敏感度排序
4. **时点判断**：
   - 政策生效时间线
   - 盘面已 price-in 多少（看过去 7 天涨跌）
   - 苗头期 vs 共识期
5. **前 3 名深度分析**：对最受益的 3 个标的简版套路 A

### 输出

```markdown
# 政策分析：<政策名> YYYY-MM-DD

## 政策摘要（200 字内）

## 产业链影响地图
### 直接受益
| 标的 | 市占 | 受益路径 | 已 price-in | 建议 |
| ... | ... | ... | ... | 买/观望/不买 |

### 间接受益 / 直接受损 / 间接受损（同上结构）

## 前 3 名深度分析
（简版 alpha-thesis-card，每个标的 500 字内）

## 时点建议
- 抢跑窗口：... 天
- 进场价：...
- 止损：...
```

## 套路 C：公司事件驱动分析

### 输入
- 财报 / 业绩预告 / 并购 / 发布会 / 新产品 / 诉讼
- 典型：NVDA Q3 Beat、Meta 新 AI 产品、Figure C 轮融资

### 数据抓取

1. **事件原始信息**：
   - 财报：SEC 10-Q / 电话会文字稿 / 财报 PPT
   - 发布会：公司 blog / 视频原始
   - 并购：8-K 披露 / 双方公告
2. **市场反应**：
   - 盘后 / 当日股价
   - 同业联动（上游 / 下游怎么反应）
3. **卖方反应**：
   - 分析师评级变化
   - 目标价调整
4. **社区情绪**（辅助）：
   - HN / FinTwit 热议
   - rss 最近 24 小时相关条目

### 分析步骤

1. **事件本体解读**（核心数据）
   - 财报：营收 / EPS / 毛利率 / Capex / Guidance vs 预期
   - 并购：对价 / 估值倍数 / 协同效应 / 监管风险
2. **关键数据 Beat/Miss**：对照共识预期，二阶导（加速/减速）
3. **产业链传导**（≥ 2 跳推演）：
   - NVDA Beat → HBM 需求 ↑ → 美光 / SK 海力士 → A 股兆易/佰维
4. **传导速度**（苗头期窗口）：
   - 美股反应 T+0
   - A 股间接标的 T+7 到 T+30（传导延迟）
5. **跟投清单**：
   - 优先级 1：直接受益（上游卡脖子环节）
   - 优先级 2：间接受益（下游配套）
   - 避免：已 price-in 的 Layer 1 龙头（追高低胜率）

### 输出

```markdown
# 事件驱动：<公司 + 事件> YYYY-MM-DD

## 事件核心（100 字）

## 关键数据 Beat/Miss
| 指标 | 预期 | 实际 | 差 | 信号 |

## 产业链传导推演（≥ 2 跳）
Layer 1: ... (已反应)
Layer 2: ... (T+7 传导)
Layer 3: ... (T+30 传导)

## 跟投清单
| 优先级 | 标的 | 传导延迟 | 目标 | 止损 |

## 时间窗口
- 最佳入场：T+X
- 预期兑现：T+Y
```

2. **产业链地图**：
   - 套 AI 8 层框架（若 AI 主题）
   - 或自建（非 AI 主题）
3. **赛道龙头 + 二线 + 新贵**：
   - 每层找 3-5 个标的
4. **估值水位对比**：
   - 同业中位 vs 历史分位
5. **卖方共识**：
   - 主题覆盖度（多少大行在推）
6. **情绪指标**：
   - Google Trends 关键词 YoY
   - 媒体覆盖密度

### 分析步骤

1. **主题当前阶段判断**（早期 / 中期 / 后期 / 晚期）：
   - 早期：论文 / 初创融资 → Alpha 最大但不确定性高
   - 中期：产业链成熟 + 龙头跑出来 → Alpha 中（当前 AI 所处）
   - 后期：应用落地 + 消费渗透 → Alpha 衰减
   - 晚期：平台固化 + 利润转移到头部 → 买 ETF 更稳
2. **Layer 逐层定位**：哪些 Layer 已 price-in，哪些 Alpha 未兑现
3. **标的排序**：按"精度 × 时点 × 仓位容量"三维
4. **建仓建议**：
   - 主题仓位总量（按用户风险偏好）
   - 单层 / 单股上限
   - 分批还是一次

### 输出

```markdown
# 主题分析：<主题名> YYYY-MM-DD

## 主题阶段判断
- 当前：中期
- Alpha 剩余窗口：预计 X 个月

## Layer 定位（AI 8 层对照）
| Layer | 状态 | 代表标的 | Alpha | 仓位建议 |

| L8 | Alpha | CEG/金盘 | 高 | 加仓 5% |

## 推荐建仓路径
（结合用户风险偏好出仓位矩阵）

## 风险 / 证伪
```

## 诚实边界（不硬凑结论）

以下情况必须明确告诉用户 "**无法给出建议**"：

1. **数据不足**：关键一手数据抓不到（反爬 / 付费 / 未披露）
2. **预期差 ≤ 2**：分析后发现只是 Beta 不是 Alpha
3. **估值透支**：已到历史 95+ 分位，不值得追
4. **时间不对**：主题进入晚期 / FOMO 峰值
5. **风险超预期**：bear case 的最大亏损 > 用户风险承受

**不硬凑"买入" 或 "卖出" 结论**，观望也是决策。

## 铁律 +1：报告完成前必须自检 + 改进（写完 ≠ 完成）

**报告写完字只是草稿，自检 + 改进才是完成。** 任何报告在输出给用户之前，必须先自己当一次质疑者，跑一遍自检清单，修掉致命问题、标注次要局限。**不走这步 = 报告未完成**。

### 为什么是铁律

LLM 写报告的天然倾向是"完成任务"——结构齐、字数够、看起来像报告就交卷。但金融决策报告的价值不在"结构完整"，在"每个数字、每个对标、每个结论都经得起挑战"。

**核心问题**：
- 方法论跑到位了不等于数据深度够
- 数据抓到了不等于估值工具选对
- 逻辑自洽不等于没有内部矛盾
- 结构规范不等于时机正确（年报前 1 周出报告本身就是错误）

**血泪案例**：炬光科技 688167（2026-04-22 首版报告）
- 方法论（铁律 0 + -0.6）执行到位
- 但用户反馈后自评发现：
  - 2025 年报马上披露，报告时机错
  - 用 PE 估值但公司今年才微利，应该用 PS / 期权型估值
  - V 型槽、N×N 阵列、ASML 匀化器的核心数字（单价、月产、份额）全没挖到
  - 没做同业 PS 硬对标（太辰光 / 天孚通信 / 奥普光电）
  - 逻辑矛盾：乐观区间 ¥330–395 / 概率加权 ¥240 / 申万目标 ¥410 三锚未统一
  - 模板思维残留（硬套东山"两波上涨"叙事，不贴炬光真实走势）
- **第一版用户反馈后自评才发现这些** → 说明"自检"应该在输出前就做

### 自检清单（8 维度 × 严重度分档）

#### 维度 1：方法论执行（红线）
- [ ] 铁律 0（全量时点校准）：股价 / 财务 / 分析师 / 订单 四类时点都标了吗
- [ ] 铁律 -0.6（业务线矩阵）：矩阵穷举 / 主题映射 / 子公司三问 / 同业反向 / 券商分歧 五步都跑了吗
- [ ] 铁律 -0.5（估值归因）：各业务毛利贡献有硬拆吗

#### 维度 2：数据深度（致命）
- [ ] **核心业务的关键数字有硬数据吗**（不是"样件交付中"这种定性）
  - 订单规模 / 客户身份 / 单价 / 毛利 / 供应份额 / 月产能
- [ ] **单位产品价值量测算**（下游每卖 1 个 X，上游自家产品值多少钱）
- [ ] 引用的每个数字是否可追溯（哪个报告哪一页）

#### 维度 3：估值工具匹配（致命）
- [ ] **公司生命周期与估值工具匹配吗**
  - 成熟盈利期 → PE 合理
  - 成长期 / 微利期 → 必用 PS
  - 亏损转正期 → PS + DCF + 期权型估值
  - 主题卡位型 → 期权组合估值（多条赛道 × 各自期权价值加总）
- [ ] 同业 PS / PE 硬对标是否有（不只是定性"稀缺"）

#### 维度 4：逻辑一致性（重要）
- [ ] 三情景 / 概率加权 / 券商目标价 三个锚是否自洽
- [ ] 各章节结论是否相互支持（不是前后矛盾）
- [ ] 操作建议的价位是否和估值结论对应

#### 维度 5：时机判断（致命）
- [ ] **重大披露是否即将发生**（年报 / 季报 / 重大订单 / 调研纪要）
  - 距今 ≤ 2 周的重大披露 → 建议等披露后再出报告，或**在报告顶部警示时机风险**
- [ ] 报告有效期是否合理设置（高波动期 ≤ 5 日）

#### 维度 6：技术面 + 筹码面（重要）
- [ ] K 线结构 / 支撑阻力 / 量能 有没有
- [ ] 股东户数 / 龙虎榜 / 两融余额 / 北向持仓 至少 2 个有没有
- [ ] 止损位设置是否有技术性依据（不只是心理关口）

#### 维度 7：反面证据（红线）
- [ ] 主动搜了 bear case / 做空 / 质疑 / 风险 没
- [ ] 写了反方声音段落没（不是只列风险清单）
- [ ] 压力测试（极端情景下损失上限）有没有

#### 维度 8：模板思维残留（重要）
- [ ] 叙事结构是否贴**本标的**真实走势（不是照搬上一份）
- [ ] 章节编排是否适配本标的特殊性
- [ ] 用词是否原创（不只是换了个股票名）

### 自检 → 问题分档 → 修复

每个发现的问题分三档：

| 严重度 | 定义 | 处理 |
|-------|------|------|
| **致命** | 结论可能错 / 估值工具错 / 时机错 | **必须修复**，不修掉 = 不完成 |
| **重要** | 说服力不足 / 数据薄弱 / 有漏 | 修复或明确标注缺口 |
| **次要** | 细节打磨 / 润色 | 在"数据局限"段落标注 |

### 自检是内部动作，不对外

**重要：自检评分卡 / 维度打分 / "补上次缺失"/ 版本标注 / 方法论打分——这些都是内部质量检查，不出现在对外报告里**。对外只有一个终版，没有迭代痕迹。

```
内部动作（不出现在报告里）            对外内容（出现在报告里）
───────────────────            ───────────────────
自检评分卡（8 维度打分）            → 数据局限章节（把"未修复项"以事实形式列出）
问题分档（致命/重要/次要）          → 关键风险章节（把"反面证据"以结构化形式呈现）
"v1 / v2 / v3 / 基于 xx 改进"     → 无版本号，无迭代痕迹
"补第一版缺失 / 修正版 / 血泪案例"  → 无内部视角，直接陈述结论
"方法论跑了铁律 X / Y / Z"         → 无内部方法论标签
```

### 对外报告的唯一强制输出

每份报告末尾**只需要**包含：

```markdown
**关键数据局限**：
- 未披露的数据（等年报 / 等调研）
- 未核实的数据（客户身份 / 单价 / 份额）
- 外推需谨慎的数据（低基数 / 波动大）

**报告有效期**：X 天（或"至某重大披露日"）。

**免责声明**：不构成投资建议。
```

这不是"评分卡"，是**事实陈述**：读者看到的是"哪些信息报告里缺失"，而不是"作者给自己的质量打多少分"。

### 红线

- **写完不自检 = 违反铁律 +1 = 报告未完成**
- 自检后不分档 = 违反铁律 +1 = 重做
- 有致命问题未修复就交付 = 违反铁律 +1 = 重做
- **把自检评分 / 打分 / 方法论标签 / 版本号 / "补上次缺失" 暴露给用户 = 违反铁律 +1 = 重做**（对外只有终版，没有内部视角）
- 不在"数据局限"披露未修复缺口 = 违反铁律 +1 = 重做（事实局限要对用户诚实）

## 执行检查单（每次分析前跑一遍）

### 基础流程
- [ ] 命题类型识别清楚（A/B/C/D 或组合）
- [ ] 按 framework 方法论执行（不偷懒）
- [ ] 最终输出决策卡（格式规范）
- [ ] 诚实 disclaimer（不构成投资建议）

### 数据深度（价值 1）
- [ ] 至少 **4 个独立源**（不是 4 篇同源转引）
- [ ] 至少 **2 个 T1-T2 原始源**（SEC / HKEXnews / IR / API）
- [ ] **反面证据**主动搜索并纳入
- [ ] 避开 blacklist（CSDN / 东财股吧 / 雪球 / 金色财经等）

### 分析深度（价值 2）
- [ ] 自己算估值（不抄卖方目标价）
- [ ] 自己推产业链 ≥ 2 跳
- [ ] 自己读财报（不看新闻摘要）
- [ ] 自己写 bear case（不等别人告诉你）
- [ ] **独立结论**（不做观点复读机）

### 预判未来（价值 3）
- [ ] 透过表象拆到里层（数据间关系 / 指引落差 / 沉默字段）
- [ ] 预判 T+3 / T+6 / T+12 / T+24 四个时间点
- [ ] 每个预判**可证伪**（具体 + 时点 + 触发动作）
- [ ] 预期差强度 ≥ 3（否则是 Beta 不是 Alpha）

### 红线（任一触发重做）
- [ ] 结论只来自卖方研报 → 重做（没独立分析）
- [ ] 未读任何原始披露 → 重做（没 T1 数据）
- [ ] 只讲现状不预判未来 → 重做（没前瞻价值）
- [ ] 未写 bear case → 重做（没压力测试）
