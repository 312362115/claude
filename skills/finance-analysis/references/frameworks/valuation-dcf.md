# DCF 估值（Valuation - DCF）

> **只讲三情景 DCF，不讲单点 DCF**。单点 DCF 输入假设过敏感，没有决策意义。
>
> 核心：**乐观/中性/悲观 三情景 × 显式预测 5Y × 终值 × WACC × 概率加权 + 敏感性 ±10%**。
>
> **认知基础**：`command-driven-analysis.md` §"股市是预期驱动" — DCF 本质是"用现金流语言翻译未来预期"。
> **姐妹篇**：`valuation-relative.md`（相对估值）。本 framework 关注**成长期 / 低可比 / 主题失真**场景。

---

## 1. 核心定位

### 适用场景

- **成长期**公司（高增速 + 转正现金流 + 未进入稳态）
- 可比公司少、相对估值失真（如独家商业模型、主题情绪极端期）
- 未来 1-2 年业绩与过去差异极大（如 NVDA 2023 爆发前后）
- 需要回答："XX 股现在 PE 100x 到底合理不合理"（DCF 给客观锚）
- 并购 / 分拆 / 业务剥离场景，需要估单一业务价值

### 不适用场景

- **极早期亏损期**（现金流为负且模式未定型） → 走期权估值 / Venture 方法
- **稳态低增速公司**（成熟现金牛）→ 相对估值（PE/PB）已足够
- **重资产强周期行业**（银行 / 保险 / 资源）→ PB / RNAV / NAV 更准
- **模式未经历完整周期**（SaaS 2 年期） → 假设成分 > 事实成分，置信度低
- **财报严重不可信** → 所有估值方法都失效，优先风险规避

## 2. 核心方法论：三段式 + 敏感性

```
Step A  关键假设（三情景：营收 CAGR / 利润率 / CapEx / WACC / 终值）
Step B  三情景 NPV × 概率加权 → 合理股价
Step C  敏感性分析（每参数 ±10% → 股价变化）→ 置信度判断
```

### 核心原则

1. **假设透明化** — 每个输入（营收、利润率、WACC、g）都必须标注"依据 + 范围 + 证据"
2. **三情景强制** — 单点 DCF 不算 DCF
3. **敏感性必选** — 任何假设 ±10% → 股价 > ±30%，降低 DCF 权重
4. **和相对估值交叉验证** — 两者差异 > 30% → 至少一个假设错了
5. **不算到小数点 3 位** — 决策导向，给区间不给精确值

## 3. 决策树

```
命题输入：XX 成长股现在合理价？
  │
  ├─ Step 1: 适用性判断
  │   ├─ [✓] 成长期（YoY > 30% 或转正期）
  │   ├─ [✓] FCF 为正或未来 3 年内转正
  │   ├─ [✓] 模式经历 ≥ 1 个完整周期 或 T+24 可预测性充足
  │   └─ 任一 [✗] → 退相对估值 / 期权估值 / 标注"DCF 不适用"
  │
  ├─ Step 2: 三情景营收假设（5Y 显式 + 5-10Y 延伸 + 终值）
  │   ├─ 乐观：主题全兑现 + 市占率继续扩张
  │   ├─ 中性：主题部分兑现 + 市占率稳定
  │   └─ 悲观：主题退潮 + 竞争冲击
  │
  ├─ Step 3: 利润率演进路径
  │   ├─ 毛利率阶梯（初期 → 规模化 → 稳态）
  │   ├─ 经营杠杆（营收增 1% → 利润增 X%）
  │   └─ 稳态利润率锚（对标行业龙头 ±5%）
  │
  ├─ Step 4: 自由现金流计算
  │   └─ FCF = EBIT × (1 - tax) + D&A - CapEx - ΔWC
  │
  ├─ Step 5: WACC 确定
  │   ├─ 无风险利率（10Y Treasury / 国债）
  │   ├─ 股权溢价（5-7%）
  │   ├─ Beta（5Y 月度回归 vs 指数）
  │   └─ WACC = Rd × (1-t) × D/V + Re × E/V
  │
  ├─ Step 6: 终值（Gordon 或出口倍数）
  │   ├─ Gordon: TV = FCF_n × (1+g) / (WACC - g)
  │   ├─ 出口 EV/EBITDA: TV = EBITDA_n × 倍数
  │   └─ 交叉验证：两方法偏差 < 20%
  │
  ├─ Step 7: 三情景 NPV
  │   └─ NPV = Σ FCF_i / (1+WACC)^i + TV / (1+WACC)^n
  │
  ├─ Step 8: 概率加权期望股价
  │   └─ 乐观 20-40% / 中性 40-60% / 悲观 10-30%（反面强制 ≥ 20%）
  │
  ├─ Step 9: 敏感性分析（必做）
  │   ├─ 营收 CAGR ±10% → 股价 ±X%
  │   ├─ 毛利率 ±10% → 股价 ±X%
  │   ├─ WACC ±10% → 股价 ±X%（反比）
  │   └─ 终值倍数 ±10% → 股价 ±X%
  │
  └─ Step 10: 决策输出
      ├─ 交叉验证 valuation-relative 期望价（差 > 30% → 回查假设）
      ├─ 隐含预期位置 + 盈亏比
      └─ 决策卡（`templates/valuation-card.md` DCF 版）
```

## 4. 量化信号

| 节点 | 可观测信号 | 通过阈值 | 未通过 |
|------|-----------|---------|--------|
| Step 1 适用 | 成长期 / FCF 路径 / 模式稳定 | 全 ✓ | 退相对估值或标注不适用 |
| Step 2 营收 | 卖方 consensus ≥ 3 家 + 最新 guidance | YES | 自建分部预测 + 降置信度 |
| Step 3 利润率 | 利润率路径 有明确阶梯依据 | YES | 标注"假设强依赖稳态锚" |
| Step 5 WACC | Beta ∈ [0.6, 1.8] | YES | 异常值 → 用行业中位 Beta |
| Step 6 终值 | 两方法交叉差 < 20% | YES | 差 > 20% → 揭示假设冲突 |
| Step 8 概率 | 悲观 ≥ 20% + 反面证据 ≥ 3 | YES | 补反面搜索 |
| Step 9 敏感 | 任一 ±10% → 股价 < ±30% | YES | 超过 → 假设过敏感，降 DCF 权重 |

## 5. 核心公式

### 5.1 自由现金流

```
FCF = EBIT × (1 - tax) + D&A - CapEx - ΔWC

简化（稳态近似）：
FCF ≈ 净利 × (1 + D&A率 - CapEx率 - ΔWC率)
```

### 5.2 NPV

```
NPV = Σ[i=1..n] FCF_i / (1+WACC)^i + TV_n / (1+WACC)^n

企业价值 EV = NPV
股权价值 = EV - 净债务
每股合理价 = 股权价值 / 总股本
```

### 5.3 WACC

```
WACC = (E/V) × Re + (D/V) × Rd × (1 - tax)

Re（股权成本）= Rf + β × ERP
  Rf: 10Y 国债 / Treasury
  β: 5Y 月度回归
  ERP: 5-7%

典型范围：
  美股大盘成长：8-11%
  A 股龙头：9-12%
  港股中资：10-13%
  高风险新兴：12-18%
```

### 5.4 终值

**Gordon 增长模型**：
```
TV = FCF_n × (1+g) / (WACC - g)
g: 稳态增长（= 长期通胀 + 长期 GDP 增速, 通常 2-4%）
```

**出口倍数**：
```
TV = EBITDA_n × 出口倍数（对标成熟同业当前 EV/EBITDA）
```

**交叉验证**：两方法偏差 < 20% 才可用；否则回查假设。

### 5.5 概率加权

```
合理股价 = Σ(情景概率_i × 情景每股合理价_i)
```

### 5.6 敏感性表（必做）

```
| 参数变动 ±10% | 股价变化 |
|-------------|---------|
| 营收 CAGR   | ±X%     |
| 毛利率稳态   | ±X%     |
| WACC         | ∓X%（反比）|
| 终值倍数 / g | ±X%     |
```

## 6. Worked Example：NVDA 2023 成长期 DCF

### 6.1 背景

- 2023-05 财报季前后，NVDA 股价 $270-300 区间
- Trailing PE ~100x，forward PE ~50x
- 市场分歧巨大：多头认为 AI 革命刚起步；空头认为 PE 过高 + 历史周期股特征
- **命题**：2023 年 $300 估值合理吗？未来 2 年目标价？

### 6.2 Step 1：适用性

- ✓ 成长期（2023 Q1 营收 YoY +18%，Q2 indication +100%+）
- ✓ FCF 正且大幅增长
- ✓ GPU 商业模式经历 4 个周期（游戏 → 加密 → AI）
- **适用 DCF**

### 6.3 Step 2：三情景营收假设（5Y 显式）

| 情景 | 概率 | 2023 | 2024 | 2025 | 2026 | 2027 | CAGR（5Y） |
|------|------|------|------|------|------|------|-----------|
| 乐观 | 30% | 55B | 110B | 170B | 220B | 260B | **+47%** |
| 中性 | 50% | 48B | 85B | 115B | 140B | 160B | **+35%** |
| 悲观 | 20% | 42B | 65B | 80B | 90B | 95B | **+21%** |

**依据**：
- 乐观：Data Center 继续 3x 增速 + 新产品周期（B100/Rubin）顺畅
- 中性：HBM 供应瓶颈限制出货，但仍保持 2x 节奏
- 悲观：云厂商自研芯片蚕食 + 训练需求触顶

### 6.4 Step 3：利润率演进

| 情景 | 稳态毛利率 | 稳态经营利润率 | 税率 |
|------|----------|--------------|------|
| 乐观 | 75% | 55% | 15% |
| 中性 | 70% | 48% | 18% |
| 悲观 | 65% | 40% | 20% |

**锚**：2023 Q1 毛利率 65% → 2023 Q4 已升至 75%，AI 产品毛利高是主因。

### 6.5 Step 5：WACC

- Rf = 4.2%（2023 年 10Y Treasury）
- β = 1.6（5Y 月度回归）
- ERP = 6%
- Re = 4.2% + 1.6 × 6% = **13.8%**
- 几乎无净债务 → **WACC ≈ 13.5%**

### 6.6 Step 6：终值

**Gordon** (g = 3%)：
- 乐观：FCF_2027 = 130B × 1.03 / (0.135 - 0.03) = **$1275B**
- 中性：FCF_2027 = 75B × 1.03 / 0.105 = **$735B**
- 悲观：FCF_2027 = 35B × 1.03 / 0.105 = **$343B**

**出口 EV/EBITDA** (对标 MSFT ~18x)：
- 乐观：EBITDA_2027 = 150B × 18x = **$2700B** ⚠️ 偏差 > 20%，降档 15x → $2250B
- 中性：EBITDA_2027 = 85B × 15x = **$1275B** ✓ 和 Gordon 偏差 40% → 取中值 **$1000B**
- 悲观：EBITDA_2027 = 40B × 13x = **$520B** ✓ 取 $432B

### 6.7 Step 7：三情景 NPV

```
乐观 NPV ≈ Σ(2023-2027 FCF 折现) + TV 折现
         ≈ $180B + $1100B ≈ $1280B
         ÷ 2.46B 股 = $520/股

中性 NPV ≈ $110B + $530B ≈ $640B
         ÷ 2.46B 股 = $260/股

悲观 NPV ≈ $60B + $230B ≈ $290B
         ÷ 2.46B 股 = $118/股
```

### 6.8 Step 8：概率加权期望股价

```
合理股价 = 0.30 × 520 + 0.50 × 260 + 0.20 × 118
        = 156 + 130 + 23.6
        ≈ $310/股
```

### 6.9 Step 9：敏感性分析

| 参数变动 ±10% | 乐观股价变化 | 中性股价变化 |
|-------------|------------|------------|
| 营收 CAGR  | ±18% | ±15% |
| 稳态毛利率 | ±25% | ±20% |
| WACC         | ∓22% | ∓18% |
| 终值倍数    | ±20% | ±17% |

**敏感性红旗**：毛利率 ±10% 对应 ±25%，WACC ±10% 对应 ∓22% → 略接近 30% 红线，但未突破，置信度 = 中。

### 6.10 Step 10：决策输出

- **DCF 期望股价**：$310
- **相对估值交叉验证**（用 forward PE 50x × 2025E EPS ~$6）：**$300**，差 3% ✓ 两方法收敛
- **现价 $270-300 对应隐含预期**：中性情景 90-100%，合理
- **盈亏比**：(520 - 285) / (285 - 118) = **1.4:1**，偏低但仍正
- **决策**：买入（小仓 2-3%），等乐观情景催化剂（B100 发布 / 新客户 / Q2 财报超预期）加仓
- **止损**：$200（悲观情景 +70% 缓冲）

**触发动作**：
- 若 2023 Q2 营收 YoY > +100% → 乐观概率升至 40%，目标价提升至 $380
- 若 Q3 云厂商自研大规模替代 → 悲观概率升至 30%，减仓至 1%

### 6.11 事后回溯（教训留档）

**实际发生**（2024-2025）：
- NVDA 2024 营收达 $130B（超过乐观情景的 $110B）
- 2025 股价涨至 $1200+ → split 后约 $120（复权约等于 $1200）
- 乐观情景显然低估了

**教训**：
- 2023 年的"乐观情景 CAGR +47%"仍然低估了 AI 爆发力度
- 情景假设要定期 refresh，不是一次算就用 5 年
- 但 DCF 框架仍然有效：它告诉你 $300 合理、$500 需要更强催化剂，这个判断在当时是对的
- 2025 年 $1200 隐含的是"远超乐观情景 + 主题溢价 2.0x"，属于情绪顶部

**沉淀到 self-improvement-loop**：DCF 的价值在提供锚和决策纪律，不在精准预测。

## 7. 执行纪律

### 红线（违反 = 重做）

1. **单点 DCF** → 违反"三情景必须"原则
2. **未做敏感性** → 无法判断假设质量
3. **WACC 拍脑袋** → 必须从 Rf / β / ERP 逐层推
4. **终值单方法** → Gordon 和出口倍数必须交叉验证
5. **乐观概率 > 50%** → 反面证据不足
6. **DCF 结果和相对估值差 > 30% 无解释** → 至少一个假设错
7. **隐含预期位置 > 100% 乐观 还建议买入** → 逻辑错位
8. **用过去 3 年 CAGR 线性外推** → 忽视增速衰减规律

### 反模式

❌ **"小数点精度"幻觉**：股价算到 $327.56（假设误差 > 50%，精度是假象）
❌ **敏感性跳过**：只算结果不算假设敏感度（= 不知道结果多不准）
❌ **终值占比 > 80%**：说明显式期太短或 g 过大（降 g 或延长显式期）
❌ **WACC 选择性低估**：用 8% 给高风险标的（得出很美的 NPV）
❌ **g > 长期经济增速**：g > 4% 基本是假设爆炸
❌ **情景假设无依据**："我觉得会 +50%"（没有市占率/产能/客户锚）
❌ **忽视 SBC**：算 FCF 不扣股权激励（美股科技股常见陷阱）

### 核查清单

- [ ] 适用性判断通过（成长期 / FCF / 模式成熟）
- [ ] 三情景营收有具体数字 + 市占率 / 产能 / 客户锚
- [ ] 利润率演进路径有稳态锚
- [ ] WACC 从 Rf / β / ERP 推导，不是拍脑袋
- [ ] 终值两方法交叉验证 < 20% 偏差
- [ ] 终值占 NPV 比例 < 80%
- [ ] 悲观概率 ≥ 20% + 反面证据 ≥ 3 条
- [ ] 敏感性 4 参数都做了
- [ ] 任一参数 ±10% → 股价变化 < ±30%（否则降权重）
- [ ] 和 valuation-relative 交叉验证
- [ ] 决策 + 止损 + 触发动作都写清楚

## 8. 失效场景

### 8.1 FCF 长期为负且模式未验证

**场景**：初创期 SaaS、Biotech、加密协议 Layer。

**退路**：
1. DCF 结果 100% 来自终值，本质是"赌"
2. 改用期权估值（Real Options）或 Venture 方法（市场规模 × 预期市占率 × 退出倍数）
3. 标注"DCF 不适用，估值为定性"

### 8.2 利润率路径不可锚

**场景**：新商业模型（如 LLM API）无成熟对标，稳态利润率是猜的。

**退路**：
1. 用"利润率区间 [X%, Y%]"扩大敏感性范围
2. 接受"DCF 结果置信度低"标签
3. 更多依赖相对估值的对标池（若有）

### 8.3 WACC 随时间剧变

**场景**：加息周期，Rf 从 0.5% → 5%，估值理论上应重新计算。

**退路**：
1. 每半年 refresh WACC
2. 敏感性必做 WACC ±20%（不是 ±10%），暴露利率风险
3. 决策中显式说"利率假设依赖"

### 8.4 监管 / 地缘政治冲击

**场景**：芯片出口管制、反垄断拆分、外资限制（如中概股）。

**退路**：
1. 不适合普通 DCF，需"政策折价系数"（0.5-0.8 作情景权重）
2. 或退化为"政策情景加权"单独处理

### 8.5 财报造假 / 商誉堆积

**场景**：同 `valuation-relative.md` §8.4 — DCF 输入污染。

**退路**：估值不适用，走信用 / 风险规避。

## 9. 和 valuation-relative 的交叉验证

**Golden rule**：任何成长股估值，两方法都做一遍，收敛才有置信度。

```
如果 DCF 期望价 和 相对估值加权价 差 < 15%  → 高置信度
如果 差 15-30%                              → 中置信度（挑假设冲突）
如果 差 > 30%                               → 至少一个假设错，回查
```

**差 > 30% 的常见原因**：
1. DCF 的 WACC / 终值过于激进或保守
2. 相对估值的对标池 PE 已严重失真（情绪顶部）
3. DCF 营收假设和相对估值的 forward 业绩不一致
4. 主题溢价因子在相对估值里吃了太多，DCF 里没体现

## 11. Disclaimer

- DCF 的价值是提供**决策锚**，不是精准预测
- 所有假设（营收、利润率、WACC、g）含主观判断
- 敏感性分析提示假设脆弱度，不保证未来实现
- 2026 年之后使用本 framework，请 refresh 利率 / Beta / 对标倍数
- 黑天鹅事件（监管 / 技术变革 / 地缘）可能使所有情景失效
- 最终决策需结合个人风险偏好、税务、流动性约束

**"所有模型都是错的，但有些是有用的"（George Box）** — DCF 属于"有用的错"，前提是承认它错。
