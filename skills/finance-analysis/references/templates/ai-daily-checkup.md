# AI 主题日度巡检 Template

> **每天 5-15 分钟**，对接 `~/workspace/rss/daily/YYYY/MM/DD.md`。
> 关联 framework：`frameworks/ai-thematic-investing.md`
>
> **这是主频**。月度巡检是系统性 review，日度才是真正的趋势启动识别主力。

---

## 执行节奏（多源交叉，不只 rss）

**核心原则**：rss 是主源但不唯一。rss 有 4 类盲区必须多源兜底。

| 时点 | 动作 | 耗时 | 源 |
|------|------|------|---|
| 早上 | 读 rss 昨日 brief（金融+AI 区块） | 3 分钟 | rss-daily |
| 早上 | **Watchlist 股价扫描**（20 只超阈值） | 3 分钟 | 行情软件（富途/东财） |
| 早上 | **公告直查**（SEC/HKEX/巨潮） | 3 分钟 | 对应官网 |
| 早上 | 扫 X FinTwit / Bloomberg 免费头条 | 3 分钟 | X / bloomberg.com |
| 早上 | 跑警报 checklist（上述信号综合判断） | 3 分钟 | - |
| 有触发 | 出决策卡 / 下单 | 15-30 分钟 | - |
| 无触发 | 记录 + 累积"无触发计数" | 1 分钟 | - |

**rss 盲区（必须认清）**：
1. **反爬漏抓**：部分源返回 412 / 加密 / JS 渲染（如中证指数公司）
2. **LLM 过滤漏**：信号被当噪音过滤掉
3. **小盘股覆盖不全**：Tier-2 标的可能没被主流媒体 cover（尤其 A 股中小盘）
4. **盘中突发**：rss 每日 1-3 次，盘中事件（财报时段 / 闪崩 / 突发公告）抓不到
5. **跨源关联**：rss 可能分别报 NVDA 财报和 A 股兆易异动，不会帮你连起来

**触发才决策，不触发就跳过**。不强制每天出决策，但**必须每天多源扫一遍**。

- [ ] Watchlist 公告（10-K/10-Q/8-K/港股公告）—— 源 1 或 3
- [ ] Watchlist 股价异动（±5%+ 或涨跌停）—— 源 2

### 📊 观察级（记录，不一定动）

- 不落实到 watchlist 的泛 AI 新闻

## 趋势启动 Checklist（对警报触发的标的跑，3 分钟）

三条同时满足 = **触发加仓决策**：

| 条件 | 判断 |
|------|------|
| 突破 200 日均线 +5% | ✅/❌ |
| 成交量 > 50 日均值 1.5x | ✅/❌ |
| 财报 Beat / 产业链信号 / 大厂订单 | ✅/❌ |

**结论**：
- 3/3 → 出决策卡加仓
- 2/3 → 进入"观察级"，加 watchlist（明天继续跟）
- ≤ 1/3 → 噪音

## FOMO 顶部 Checklist（对持仓标的跑）

任一触发 = **减仓 1/3**：
- [ ] 财经头条连续 5 次提及
- [ ] 打车师傅 / 亲戚问这只股
- [ ] 同概念股全部涨停
- [ ] 卖方一致目标价到离谱（共识率 > 90%）
- [ ] 散户资金流入 > 2 年均值 3x

## 今日决策

- [ ] 无动作（记录即可）
- [ ] 出决策卡：标的 XX，动作 XX（详见 docs/research/YYYY-MM-DD-XX.md）
- [ ] 加 watchlist：标的 XX
- [ ] 移出 watchlist：标的 XX（理由：论点崩塌 / 触顶）

## 决策

{具体决策内容 or "今日无动作"}

## 反思

- 昨日决策复盘：...
- 本周累计：X 次加仓 / X 次减仓 / X 次观察

## 日度 KPI

- **响应速度**：警报出现到决策卡出稿 < 30 分钟
- **执行速度**：决策卡出稿到下单 < 1 小时（盘中）或次日开盘（盘后）
- **命中率**：3/3 触发的加仓，60 天胜率 > 60%
- **纪律**：每月最多 5 次"临时起意决策"（超过 = 纪律失控）

## 与月度巡检的关系

| 层级 | 处理什么 | 不处理什么 |
|------|---------|----------|
| **日度（本 template）** | 事件响应 + 警报触发 + 单点决策 | 系统性平衡 |
| **周度（TODO）** | 7 天聚合 + 新浮现主题 | 深度产业扫描 |
| **月度** | 8 层系统扫描 + 宏观温度计 + 仓位重平衡 | 日常事件 |
| **季度（TODO）** | Framework 反思 + 3 月 KPI 兜底 | 个股决策 |

**日度跑得好，月度只是补漏**。日度是主频。

## 跳过规则（周末 / 节假日）

- 港股 / A 股周末可跳过
- 美股周六日盘休，但可读 rss 日报 + X/FinTwit（海外新闻不停）
- 节假日周一需补跑
- **连续跳过 > 3 天** = 风险信号（可能错过重要事件）

## rss 空转日的处理（常见但不危险）

"空转日"指 rss daily brief 无金融/AI 警报条目。原因多样：
1. 真的没事（周末 / 节假日常见）
2. rss 反爬漏抓（源 2-4 补）
3. LLM 评分过滤误伤（源 2-4 补）
4. 盘中突发未进昨日 brief（当天盘中再扫）

**空转日 ≠ 跳过检查**。必须跑源 2（watchlist 股价扫描），这是**最终兜底**——股价异动不会骗人。

**连续 5 个空转日**：检查 rss 本身是否出故障（launchd 异常 / 源批量失效）。

## 多源冗余的价值

极端 case：某天 rss 源全挂了 / GPT-5 发布 / 大盘暴跌 —— 此时：
- 源 1 rss：空
- 源 2 股价：NVDA 暴跌 -8%（警报触发）
- 源 3 公告：Meta 8-K 披露 AI 战略收缩
- 源 4 X：全网炸锅

**任何单源都可能漏，4 源同时漏的概率极低**。这是多源交叉的核心价值。
