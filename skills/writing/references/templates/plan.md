# 开发计划模板（plan）

> 存放路径：`docs/plans/YYYY-MM-DD-<模块名>-plan.md`
> plan 是 spec 的执行层：spec 定义"做什么、怎么做"，plan 定义"拆成几步、做到哪了"。

---

## 关联

- 方案：`docs/specs/YYYY-MM-DD-xxx.md`

## 子任务

- [ ] 1. <子任务名称>
  - **做什么**：具体实现内容
  - **涉及**：`src/xxx.ts`（新建/改造）
  - **验收**：怎么验证这一步做完了

- [ ] 2. <子任务名称>
  - **做什么**：具体实现内容
  - **涉及**：`src/xxx.ts`（新建/改造）
  - **验收**：怎么验证这一步做完了

- [ ] 3. 回归测试
  - **做什么**：编写并运行自动化测试
  - **涉及**：`docs/tests/xxx.md`、`tests/xxx.test.ts`
  - **验收**：全部用例通过

## 注意事项

- 每完成一个子任务后更新 checklist 状态
- 发现方案需要调整时，同步更新关联的 spec 文件
- 子任务之间有依赖关系的标注清楚
