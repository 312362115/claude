# 战略决策分析框架

> 调研涉及战略方向、市场进入、组织调整等命题时，叠加以下框架。

## 核心框架

### 波特五力（互联网适配版）

- **适用场景**：行业吸引力评估、竞争强度判断
- **操作步骤**：逐一分析五种力量，标注互联网特殊考量

| 力量 | 分析要点 | 互联网特殊性 |
|------|---------|------------|
| 现有竞争 | 竞争者数量、差异化程度 | 赢家通吃 vs 多寡头格局 |
| 新进入威胁 | 资本/技术壁垒 | 进入壁垒低，但网络效应可形成高壁垒 |
| 替代品威胁 | 替代方案性价比 | "不做"也是替代品 |
| 供应商议价 | 关键供应商集中度 | 云服务商的定价权 |
| 买方议价 | 客户切换成本 | C 端低，B 端较高 |

- **产出格式**：五力评估表 + 行业吸引力结论

### McKinsey 7S 模型

- **适用场景**：战略执行评估、组织变革诊断
- **操作步骤**：
  1. 评估硬件三要素：Strategy（战略）、Structure（组织）、Systems（流程制度）
  2. 评估软件四要素：Shared Values（文化）、Style（管理风格）、Staff（人才）、Skills（能力）
  3. 找出硬件与软件之间的错配（最常见的执行失败根因）
- **产出格式**：7S 诊断表 + 硬件/软件匹配度评估 + 优先改进项
- **常见误用**：只分析硬件不看软件；做完诊断不给改进路径

### Ansoff 矩阵

- **适用场景**：增长方向选择
- **操作步骤**：
  1. 将增长选项放入四象限
  2. 评估每个选项的风险和回报
  3. 按风险承受能力选择路径

| | 现有产品 | 新产品 |
|---|---------|--------|
| **现有市场** | 市场渗透（最低风险） | 产品开发（中等风险） |
| **新市场** | 市场开发（中等风险） | 多元化（最高风险） |

**互联网实例**：渗透=Netflix 提价 | 产品开发=微信→支付 | 市场开发=Spotify→播客 | 多元化=字节→TikTok

- **产出格式**：四象限填充 + 优先级排序 + 风险评估

### BCG 增长份额矩阵

- **适用场景**：多业务组合管理、资源分配决策
- **操作步骤**：
  1. 将各业务线按市场增长率（纵轴）和相对市场份额（横轴）定位
  2. 匹配对应策略

| 象限 | 特征 | 策略 |
|------|------|------|
| 明星 | 高增长+高份额 | 大力投入 |
| 现金牛 | 低增长+高份额 | 收割现金流 |
| 问号 | 高增长+低份额 | 选择性投入或放弃 |
| 瘦狗 | 低增长+低份额 | 收缩/剥离 |

- **产出格式**：BCG 矩阵图 + 各业务策略建议

## 典型报告骨架

```
1. 战略回顾（上周期复盘 + 关键指标达成率）
2. 外部环境（PESTEL + 五力 + 机会/威胁）
3. 内部能力（核心竞争力 + 资源缺口 + 7S 诊断）
4. 战略选项评估（Ansoff 象限 + NPV/ROI/风险评估）
5. 实施路线图（里程碑 + 资源需求 + 组织调整）
6. 监控指标（KPI Dashboard）
```

## 数据采集指南

### 核心数据源
- 行业报告：Gartner Magic Quadrant
- 竞品情报：SimilarWeb/SEMrush（流量）
- 专家访谈：GLG/AlphaSights（专家网络）
- 内部数据：经营数据、组织能力评估

### 分析方法
- **McKinsey 假设驱动法**：Day 1 Answer → Issue Tree（MECE）→ 数据收集 → 验证/证伪 → Synthesis
- **金字塔原理**：结论先行，论据支撑，同层 MECE，纵向总分
- **Pre-mortem**：假设项目已失败，倒推可能原因

## 常见反模式

- **分析瘫痪**：过度分析不做决策 → Bezos 原则：可逆决策快速做
- **战略漂移**：执行中不断偏离初始方向 → 定期回顾 7S 匹配度
- **群体思维**：大家都觉得方向对 → Pre-mortem + 匿名投票 + Red Team
- **只看硬件**：改了战略/组织/流程但文化/人才没跟上
