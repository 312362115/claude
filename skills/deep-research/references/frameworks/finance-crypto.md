# 加密货币分析框架

> 调研涉及加密货币投资分析时使用。

## 核心分析框架

### 链上分析（On-Chain Analysis）

- **适用场景**：BTC/ETH 等主链资产的市场周期判断
- **核心指标**：

| 指标 | 含义 | 信号判断 |
|------|------|---------|
| MVRV Ratio | 市值 / 已实现市值 | > 3.5 过热（卖出），< 1 极度低估（买入） |
| NVT Ratio | 网络价值 / 链上交易量 | > 100 高估，< 50 低估 |
| SOPR | 已花费输出利润率 | > 1 整体盈利，< 1 整体亏损 |
| NUPL | 净未实现利润/亏损 | > 0.75 极度贪婪，< 0 投降 |
| Exchange Balance | 交易所持币量 | 持续流出 = 长期持有信号 |

### 叙事周期（Narrative Cycle）

- **适用场景**：板块轮动、新币/新赛道投资
- **四阶段**：创新叙事 → 资金涌入（VC → 代币发行 → 散户 FOMO）→ 泡沫膨胀 → 叙事破灭
- **判断工具**：社交媒体热度(Twitter/CT)、Google Trends、VC 投资数据、TVL 变化

### 代币经济学（Tokenomics）

- **适用场景**：新项目/代币投资评估

| 维度 | 关键问题 | 红旗信号 |
|------|---------|---------|
| 供给机制 | 总量固定还是通胀？解锁计划？ | 大量代币即将悬崖解锁 |
| 分配结构 | 团队/投资者/社区各占比？ | 团队+VC > 50% |
| 效用价值 | 代币在生态中的实际用途？ | 纯治理代币无现金流 |
| 价值捕获 | 协议收入如何回流持有者？ | 无回购/销毁/分红机制 |
| 流动性 | 交易深度和上线交易所？ | 仅 DEX，深度极浅 |

### 宏观-加密相关性

- **核心**：BTC 与全球流动性高度相关
- 美联储扩表利好、DXY 走弱利好、负实际利率推动配置、全球 M2 扩张 → 风险偏好↑

## 估值方法

| 方法 | 适用 | 局限 |
|------|------|------|
| NVT Ratio | BTC/ETH | 忽略储值功能 |
| MVRV | PoW 主链 | PoW 与 PoS 不可比 |
| Stock-to-Flow | BTC | 纯供给侧，忽略需求 |
| 协议 P/S | DeFi | 收入可被激励扭曲 |
| TVL 倍数 | DeFi | TVL 可被杠杆虚增 |
| Metcalfe's Law | 网络型资产 | 定义"活跃"标准不一 |

## 典型研报骨架

```
一、项目概述（定位/团队/融资/技术架构）
二、市场分析（赛道TAM/竞品对比/叙事周期定位）
三、代币经济学（供给/分配/效用/价值捕获/通胀模型）
四、链上数据（活跃地址/TVL/协议收入/持币集中度）
五、估值（NVT/MVRV/可比协议P/S/解锁影响）
六、风险（智能合约/监管/竞争/解锁抛压）
七、投资建议
```

## 数据来源

| 平台 | 功能 | 定位 |
|------|------|------|
| Glassnode | 链上指标（MVRV/SOPR/NUPL） | 机构级链上分析 |
| Dune Analytics | 自定义 SQL 查询链上数据 | 分析师/研究员 |
| DeFiLlama | TVL 聚合、协议对比、收益率 | DeFi 研究必备 |
| Token Terminal | 协议收入、P/S、P/F | 将协议当公司分析 |
| Nansen | 钱包标签、Smart Money 追踪 | 机构资金追踪 |
| CoinGecko | 价格、市值、交易量 | 基础数据 |

## 风险分析要点

| 风险类型 | 评估方法 |
|---------|---------|
| 智能合约风险 | 审计报告数量+质量、TVL/审计覆盖率 |
| 监管风险 | SEC 诉讼/MiCA 实施动态 |
| 流动性风险 | 滑点测试、24h 交易量/市值比 |
| 代币解锁风险 | 解锁日历（TokenUnlocks.app） |
| 桥接/跨链风险 | 桥接资金量、审计状态 |
| 叙事风险 | 社交媒体热度趋势 |
