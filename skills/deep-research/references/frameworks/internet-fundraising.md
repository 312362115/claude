# 融资与估值框架

> 调研涉及公司估值、融资策略、BP 撰写等命题时，叠加以下框架。

## 核心框架

### VC 估值法

- **适用场景**：Seed 到 B 轮的创业公司估值
- **操作步骤**：
  1. 预估 5-7 年后退出收入
  2. 应用退出倍数（SaaS 通常 10-20x ARR）
  3. 退出价值 = 退出收入 × 倍数
  4. 按 VC 要求回报折现（种子 50-100x，A 轮 10-30x，B 轮 5-15x）
  5. Post-money = 退出价值 / 要求回报倍数
  6. Pre-money = Post-money - 本轮融资额
- **产出格式**：退出假设 + 估值推导表 + Pre/Post-money
- **常见误用**：退出倍数假设过于乐观；不做敏感性分析

### 可比公司分析法（Comps）

- **适用场景**：有同类已融资/上市公司时的估值锚定
- **操作步骤**：
  1. 选取 5-10 家可比公司（同行业/相近规模/相似增速）
  2. 计算关键倍数（ARR 倍数/P/S/EV/EBITDA）
  3. 按增速和效率调整倍数

**2025-2026 SaaS 估值倍数参考**：

| 增速 | ARR 倍数 | 条件 |
|------|---------|------|
| >100% | 15-50x | Rule of 60 以上 |
| 50-100% | 10-20x | NDR > 120% |
| 20-50% | 5-12x | Burn Multiple < 1.5 |
| <20% | 3-8x | 需盈利或接近盈利 |

- **产出格式**：可比公司表（公司 × 指标 × 倍数）+ 目标公司估值区间

### Berkus 方法（pre-revenue）

- **适用场景**：无收入的种子期项目
- **操作步骤**：逐项评估，每项最高 $500K，总上限 $2.5M

| 因素 | 评估内容 |
|------|---------|
| Sound Idea | 商业概念吸引力 |
| Prototype | 可演示的产品 |
| Quality Team | 团队执行力 |
| Strategic Relationships | 关键合作/渠道 |
| Product Rollout | 初步销售 |

### 记分卡法

- **适用场景**：早期项目与同类已融资项目对标
- **操作步骤**：按维度加权对比同类已融资公司，调整估值

| 维度 | 权重 |
|------|------|
| 团队实力 | 30% |
| 市场规模 | 25% |
| 产品/技术 | 15% |
| 竞争环境 | 10% |
| 营销/渠道 | 10% |
| 其他 | 10% |

### 多方法交叉验证

```
Berkus/记分卡（Seed）→ VC 法（Seed-B 轮）→ 可比公司（A 轮+）→ DCF（B 轮+）
```

CCA 作为"地板"，DCF 作为"天花板"，两者结合最稳健。

## 典型报告骨架

### BP / 融资文档
```
1. 执行摘要（使命+问题+方案+市场+牵引力+融资需求）
2. 市场机会（TAM/SAM/SOM + 驱动因素）
3. 产品与技术（演示+架构+路线图）
4. 商业模式（收入模型+单元经济学+定价）
5. Traction（关键指标曲线+标杆客户）
6. 竞争分析（矩阵+差异化）
7. 团队（核心成员+招聘计划）
8. 财务预测（3-5 年+关键假设）
9. 融资需求（金额+用途+里程碑+估值逻辑）
```

## 数据采集指南

### 核心数据源
- 估值数据：PitchBook/Carta Data/Bessemer Cloud Index
- 可比公司：Crunchbase（融资轮次）/公开财报
- SaaS 基准：Koyfin/SaaS Capital Annual Survey
- 国内数据：IT桔子/烯牛数据/清科数据

### 关键参考

**2025 基准**：Median Series A pre-money $49.3M（Carta 数据）

**2026 趋势**：投资人更注重效率而非纯增长，Burn Multiple 和 NDR 成为估值调整核心因子。

## 常见反模式

- **估值与效率脱节**：高估值但 Burn Multiple > 3，不可持续
- **只做一种估值**：应至少用两种方法交叉验证
- **忽略稀释**：不考虑未来轮次稀释对创始人持股的影响
- **对标错位**：拿增速 100% 公司的倍数套自己增速 20% 的业务
