# 产品职能分析框架

> 调研涉及产品功能、需求优先级、产品路线图等命题时，叠加以下框架。

## 核心框架

### KANO 模型（需求分类与优先级）

- **适用场景**：新产品规划、功能优先级排序、MVP 范围界定
- **操作步骤**：
  1. 针对每个功能设计正反向问卷（"如果有 X / 没有 X 你感觉如何"）
  2. 收集样本量 >= 100 的用户回答
  3. 用归类矩阵将功能分为：基本型（必须满足）、期望型（按 ROI 排）、兴奋型（差异化来源）、无差异（不投入）、反向（坚决不做）
  4. 计算 Better-Worse 系数确定优先级
- **产出格式**：KANO 四象限图 + 功能分类表 + 优先级排序
- **常见误用**：样本量不足；问卷设计引导性过强

### RICE 评分模型（功能优先级量化）

- **适用场景**：已有产品迭代排期、Sprint 规划
- **操作步骤**：
  1. 列出候选功能列表
  2. 产品+工程共同评估 Reach（季度受影响用户数）、Impact（3/2/1/0.5/0.25）、Confidence（100%/80%/50%）、Effort（总人月）
  3. 计算 `RICE = (R × I × C) / E`，按分数排序
- **产出格式**：RICE 评分表（功能 | R | I | C | E | Score）
- **常见误用**：Impact 评分全凭拍脑袋；不区分信心高低

### 北极星指标框架

- **适用场景**：产品方向对齐、团队目标统一
- **操作步骤**：
  1. 选择唯一核心度量（反映产品为用户创造的核心价值）
  2. 拆解 3-5 个可操作的输入指标
  3. 设定护栏指标（防止为追求北极星伤害其他维度）
- **产出格式**：北极星指标 + 输入指标树 + 护栏指标

### 用户故事地图

- **适用场景**：产品全景梳理、版本规划
- **操作步骤**：
  1. 识别目标用户角色
  2. 梳理用户完整活动流（横轴 = 时间/流程）
  3. 每个活动下展开具体任务（纵轴 = 优先级递减）
  4. 画水平线切分版本（每条线 = 一个可发布版本）
- **产出格式**：二维故事地图 + 版本切分线

### 选择逻辑

| 场景 | 推荐框架 |
|------|---------|
| 0→1 MVP 范围 | KANO + MoSCoW（Must/Should/Could/Won't） |
| 功能优先级排序 | RICE（数据充分）/ KANO（需用户调研） |
| 产品全景规划 | 用户故事地图 |
| 方向对齐/目标设定 | 北极星指标 |

## 典型报告骨架

```
1. 概述（背景 + 目标用户画像 + 核心问题定义）
2. 市场与竞品（规模 + 功能矩阵对比 + 差异化机会）
3. 需求分析（KANO 分类 + RICE 评分表 + 用户故事地图）
4. 产品方案（核心功能 + 信息架构 + 核心流程图 + MVP 范围）
5. 度量指标（北极星 + 输入指标 + 埋点方案）
6. 风险与假设 + 路线图
```

## 数据采集指南

### 内部数据（优先）
- 埋点数据：行为分析、漏斗分析（神策/GrowingIO/Mixpanel）
- A/B 测试：功能验证、方案对比（火山引擎 DataTester）
- 用户访谈/问卷：需求探索、KANO 调研（问卷星/Typeform）

### 外部数据
- 行业报告：市场规模、趋势（艾瑞/QuestMobile）
- 应用商店：竞品评分、评论（七麦/App Annie）
- 搜索指数：需求热度趋势（百度指数/Google Trends）

## 常见反模式

- **HiPPO 决策**：老板拍脑袋代替数据分析
- **功能堆砌**：不做减法，每个版本塞入过多功能
- **伪数据驱动**：先有结论再找数据支撑
- **需求 ≠ 解决方案**：用户说"我要更快的马"，需求是"更快到达目的地"
