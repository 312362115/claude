# 创业机会评估框架

> 调研涉及赛道分析、创业立项、机会评估等命题时，叠加以下框架。

## 核心框架

### TAM-SAM-SOM 市场规模评估

- **适用场景**：评估市场天花板，判断机会量级
- **操作步骤**：
  1. 定义目标客户画像（ICP），明确"谁会付费"
  2. 自上而下算 TAM：行业报告 + 政府统计
  3. 自下而上算 SAM：目标客户数 × 单客户年付费额
  4. 基于竞争格局和自身能力估算 SOM
  5. 交叉验证：两方向估算误差不超 3-5 倍
- **产出格式**：TAM/SAM/SOM 三层漏斗 + 测算假设
- **常见误用**：gaming TAM（很多最好的公司起初 TAM 看起来很小）

**VC 视角**：TAM > $10B 才值得关注（红杉原则）。a16z 强调看 TAM expansion 而非静态数字。

### Peter Thiel 七问检验

- **适用场景**：从根本上判断创业方向的颠覆性优势
- **操作步骤**：逐一回答七个问题，不需全部满足，但 0 个满足几乎不值得做

| 问题 | 本质 | 判断标准 |
|------|------|---------|
| 工程问题 | 能否 10x 改进？ | 数量级提升而非渐进改良 |
| 时机问题 | 为什么是现在？ | 技术/基础设施/用户习惯拐点 |
| 垄断问题 | 能否先垄断小市场？ | 小池塘里的大鱼策略 |
| 团队问题 | 合适的人？ | Founder-Market Fit |
| 分销问题 | 有效的获客方式？ | 不只造好产品还能卖出去 |
| 持久问题 | 10 年后守得住？ | 护城河/网络效应/规模效应 |
| 秘密问题 | 你看到了什么？ | 独特洞察（The Secret） |

- **产出格式**：七问回答表 + 综合评估（满足 5-6 个 = 优秀机会）

### PMF 测试（精益验证）

- **适用场景**：验证产品市场匹配度
- **操作步骤**：
  1. 明确最关键的待验证假设（按风险排序）
  2. 用最少资源构建 MVP / Landing Page
  3. 定义可操作指标（非虚荣指标）：转化率、留存率、NPS
  4. **Sean Ellis 测试**：问用户"如果不能用这产品你多失望？" → 40%+ 回答"非常失望" = 强 PMF
  5. 基于数据决定 Pivot 或 Persevere
- **产出格式**：假设清单 + MVP 方案 + 测试数据 + Go/No-Go 决策
- **常见误用**：只问活跃用户（幸存者偏差）；把虚荣指标当验证

### 选择逻辑

| 场景 | 推荐框架 |
|------|---------|
| 市场够不够大 | TAM-SAM-SOM |
| 方向够不够好 | Thiel 七问 |
| 产品够不够对 | PMF 测试 |
| 全面评估 | 三个都用 |

## 典型报告骨架

### Pitch Deck（红杉模板 12 页）
```
使命(1句话) → 问题 → 方案 → Why Now → 市场(TAM/SAM/SOM)
→ 竞争 → 产品 → 商业模式 → 团队 → Traction → 财务预测 → 融资需求
```

### 内部立项报告
```
1. 机会概述（1 页摘要）
2. 市场分析（TAM/SAM/SOM + 痛点 + 驱动力）
3. 竞争分析（定位图 + 功能矩阵 + 差异化空间）
4. 可行性评估（技术/商业/团队匹配度）
5. GTM 策略（切入点 + 获客 + 里程碑）
6. 资源需求与投入产出
7. 决策建议（Go / No-Go / 需进一步验证）
```

## 常见反模式

- **确认偏误**：只看支持自己的数据 → 主动寻找反面证据
- **幸存者偏差**：只看成功案例 → 系统研究同赛道失败案例
- **过度自信**：高估判断准确度 → 设定宽置信区间
- **沉没成本**："已投了这么多" → 归零思考：从头来过还会做吗？
