# 竞争战略分析框架

> 调研涉及竞争优势、壁垒评估、竞品策略等命题时，叠加以下框架。

## 核心框架

### 护城河分析

- **适用场景**：评估可持续竞争优势
- **操作步骤**：
  1. 逐一评估七类护城河是否存在
  2. 评估防御力强度和可持续性
  3. 综合判断护城河宽度

| 护城河类型 | 互联网典型案例 | 防御力 |
|-----------|--------------|--------|
| 网络效应 | 微信、淘宝、Uber | 极强 |
| 规模经济 | AWS、美团配送 | 强 |
| 转换成本 | Salesforce、企业微信 | 强 |
| 品牌 | Apple、茅台 | 中强 |
| 数据壁垒 | Google 搜索、字节推荐算法 | 强 |
| 专利/技术 | 高通、ASML | 中（互联网较弱） |
| 监管壁垒 | 支付牌照、游戏版号 | 中 |

- **产出格式**：护城河评估表（类型 × 存在/不存在 × 强度 × 可持续性）
- **常见误用**：把先发优势当护城河（先发不等于壁垒）

### 蓝海战略（四步行动框架）

- **适用场景**：寻找差异化竞争空间
- **操作步骤**：
  1. 列出行业竞争的关键要素
  2. 对每个要素做四步决策

| 行动 | 目的 | 示例 |
|------|------|------|
| 消除 | 去掉行业理所当然但无效的因素 | 任天堂 Wii 消除高性能竞争 |
| 减少 | 降到标准以下以降低成本 | 拼多多减少品牌溢价 |
| 提升 | 提升到标准以上创造差异化 | — |
| 创造 | 开辟行业从未提供的新因素 | Wii 创造体感游戏 |

- **产出格式**：价值曲线图（行业平均 vs 你的方案）+ 四步行动表

### 颠覆式创新

- **适用场景**：评估被颠覆风险 / 识别颠覆机会
- **操作步骤**：
  1. 判断现有巨头是否过度服务客户
  2. 识别两条颠覆路径：**低端颠覆**（小米手机：够用+便宜）/ **新市场颠覆**（iPhone：全新场景）
  3. 评估巨头是否会"理性地"选择不回应
- **产出格式**：颠覆潜力评估 + 路径选择 + 巨头回应分析
- **常见误用**：把"更好的技术"等同于颠覆（颠覆是用"刚好够用"服务被忽视的客户）

### 飞轮效应

- **适用场景**：设计可自我强化的增长引擎
- **操作步骤**：
  1. 识别 3-5 个关键成功要素
  2. 找到因果强化关系（A→B→C→A 形成闭环）
  3. 确认每环节有可量化指标
  4. 投资最能加速飞轮的环节

**亚马逊飞轮**：更低价格→更多消费者→更多卖家→更多选择→更好体验→增长→更低成本→更低价格

- **产出格式**：飞轮图 + 各环节指标 + 加速杠杆点

### 竞争战略选择

| 战略 | 适用条件 | 风险 |
|------|---------|------|
| 成本领先 | 市场标准化、价格敏感 | 技术变革消除成本优势 |
| 差异化 | 用户愿为差异付费 | 差异被模仿 |
| 聚焦（利基） | 大玩家忽视的细分市场 | 天花板低 |
| 平台垄断 | 强网络效应市场 | 监管风险 |

## 典型报告骨架

```
1. 竞争格局全景（市场份额 + 玩家定位图）
2. 竞品深度分析（Top3 竞品的护城河 + 策略 + 弱点）
3. 竞争优势评估（七类护城河逐一评估）
4. 威胁评估（颠覆风险 + 新进入者 + 替代品）
5. 竞争战略建议（选择路径 + 飞轮设计）
6. 监控机制（竞品动态追踪指标和频率）
```

## 常见反模式

- **竞争执念**：把大量精力放在盯竞品上 → Thiel 原则：最好让竞争不再是关键变量
- **护城河幻觉**：声称有护城河但经不起推敲 → 每类护城河都需要具体证据
- **忽略非直接竞品**：只看同类产品不看替代方案
- **静态分析**：只看当前格局不考虑演变趋势
