# 分析框架参考

> 调研第三步（分析与结论形成）时查阅本文件，选择合适的分析框架。
> 框架的详细操作方法见项目 `docs/specs/deep-research-frameworks/` 目录下的 4 份调研报告。

## 一、通用底座框架（任何行业适用）

### 框架速查表

| 分析需求 | 推荐框架 | 核心产出 | 常见误用 |
|---------|---------|---------|---------|
| 宏观环境扫描 | PESTEL | 关键驱动因素 + 机会/威胁清单 | 沦为列清单，未落到对行业的具体影响 |
| 行业结构评估 | 波特五力 | 行业吸引力评分 + 利润分配判断 | 忽略互补品和平台效应（数字经济下五力模型需修正） |
| 行业阶段判断 | 行业生命周期 | 所处阶段 + 对应策略建议 | 仅凭增长率判断，忽略结构性变化信号 |
| 产业链理解 | 价值链分析 | 产业链图 + 利润池分布 | 过于抽象，不落到具体环节和参与者 |
| 市场规模测算 | TAM/SAM/SOM | 市场天花板 + 可获取空间 | 自上而下拍 TAM，不做自下而上验证 |
| 竞争格局分析 | CR5/HHI + 战略组图 | 竞争态势 + 玩家定位 | 只看市场份额不看变化趋势 |
| 数据缺失时估算 | 费米估算 | 量级判断 + 不确定性范围 | 假设不透明，未标注关键假设的敏感性 |

### 标准分析流程

```
PESTEL → 行业生命周期 → 价值链 → 波特五力 → TAM/SAM/SOM → 竞争格局 → 趋势预判
```

不必每次走完全流程，根据调研深度选择：

| 深度 | 使用的框架 | 适用场景 |
|------|----------|---------|
| **快速摸底** | PESTEL(简) + 竞争格局 + 趋势 | 初步了解、快速判断 |
| **标准调研** | 全流程 | 技术选型、竞品分析 |
| **深度研究** | 全流程 + 每个框架深度展开 | 行业进入决策、投资决策 |

### 各框架操作要点

**PESTEL**：6 维度（政治/经济/社会/技术/环境/法律）各列 2-3 个关键因素，重点标注对本行业的具体影响方向（利好/利空/不确定）。不要泛泛而谈。

**波特五力**：逐一评估供应商议价力、买方议价力、新进入威胁、替代品威胁、行业内竞争。每个力量给出 高/中/低 定性判断 + 1-2 条关键依据。数字经济下需额外考虑平台效应和网络效应。

**行业生命周期**：判断行业处于 萌芽/成长/成熟/衰退 哪个阶段。关键判断指标：增长率、新进入者数量、技术变化速度、利润率趋势。不同阶段对应不同的竞争策略。

**价值链分析**：画出从原材料到终端用户的产业链，标注每个环节的参与者、利润率、议价能力。识别利润池最厚的环节和最薄弱的环节。

**TAM/SAM/SOM**：
- TAM（总可用市场）：自上而下估算 + 自下而上验证，取交叉区间
- SAM（可服务市场）：TAM × 地域/客群/产品线过滤
- SOM（可获得市场）：SAM × 市场份额假设（需说明假设依据）

**竞争格局**：CR5（前 5 名市场份额之和）判断集中度。HHI 指数 <1000 分散、1000-2500 中等集中、>2500 高度集中。画战略组图：X 轴价格/定位，Y 轴功能覆盖度。

**费米估算**：拆解为可估算的子问题，每个子问题给出乐观/基准/悲观三个值，最终结果给出范围而非单一数字。标注关键假设及其敏感性。

## 二、互联网行业特化框架

### 2.1 执行层（6 个职能）

在通用底座上，根据调研涉及的具体职能叠加专属框架：

| 职能 | 核心框架 | 何时用 | 核心产出 | 关键数据源 |
|------|---------|-------|---------|-----------|
| **产品** | KANO 模型、RICE 评分、北极星指标、用户故事地图 | 产品功能/需求相关调研 | 需求优先级、产品路线图 | 埋点数据(内部)、用户调研、竞品功能 |
| **技术** | ATAM、C4 模型、技术选型矩阵、ADR、USE 方法 | 架构/技术栈相关调研 | 架构决策记录、选型建议 | 压测数据(内部)、监控数据(内部)、官方文档 |
| **设计** | HEART 框架、Nielsen 十大启发式、Design Sprint | UX/设计相关调研 | UX 审计报告、设计改进建议 | 热力图(内部)、可用性测试、竞品截图 |
| **测试** | 测试金字塔、敏捷测试象限、质量度量指标体系 | 质量/测试相关调研 | 质量现状评估、改进路径 | CI/CD 数据(内部)、缺陷系统(内部)、覆盖率 |
| **运营** | AARRR、RFM 模型、用户生命周期、北极星+OSM | 增长/运营相关调研 | 运营分析报告、增长策略 | 埋点数据(内部)、BI 报表(内部)、行业基准 |
| **市场** | STP、4P、PESO 媒体模型、品牌健康度追踪 | 营销/市场相关调研 | 市场策略建议、品牌评估 | 投放数据(内部)、品牌调研、社交媒体监测 |

### 2.2 决策层（6 个场景）

当调研服务于公司/业务层面的战略决策时，叠加以下框架：

| 场景 | 核心框架 | 触发词 | 核心产出 | 关键数据源 |
|------|---------|-------|---------|-----------|
| **创业机会评估** | TAM-SAM-SOM、Thiel 七问、Lean Validation、PMF 测试 | 赛道、创业、立项 | 机会评估报告、Pitch Deck | 行业报告、投融资数据、竞品数据 |
| **商业模式分析** | 商业模式画布(9要素)、单元经济学(LTV/CAC)、收入模型类型学 | 商业模式、盈利、变现 | 商业模式拆解、盈利路径 | 财务数据(内部)、对标公司财报 |
| **战略决策** | 波特五力、McKinsey 7S、Ansoff 矩阵、BCG 矩阵 | 战略、怎么赢、方向 | 战略选择建议、实施路径 | 经营数据(内部)、竞品情报、专家访谈 |
| **业务健康度** | a16z 16 指标、Sacks SaaS 指标体系、AARRR、Cohort 分析 | 健康度、诊断、体检 | 业务健康度仪表盘 | BI 系统(内部)、产品数据(内部) |
| **融资与估值** | VC 估值法、可比公司分析、Berkus 方法、记分卡法 | 估值、融资、BP | 估值区间、融资策略 | 融资数据库、可比公司财报 |
| **竞争战略** | 护城河分析(网络效应/规模/转换成本/数据)、蓝海战略、颠覆式创新、飞轮效应 | 竞争、壁垒、护城河 | 竞争战略建议、壁垒评估 | 竞品数据、用户调研、专利数据 |

## 三、金融投资特化框架

当调研涉及投资分析时，根据投资领域叠加以下框架：

| 领域 | 核心分析框架 | 核心估值方法 | 关键数据平台 |
|------|------------|------------|------------|
| **A 股** | 自上而下宏观驱动、景气度投资、行业轮动 | PE/PB/PEG/DCF（按行业选） | Wind/Choice/东方财富 |
| **美股** | 基本面分析、DCF、可比公司分析(Comps) | DCF + Comps（Football Field） | Bloomberg/SEC EDGAR/Finviz |
| **港股** | AH 溢价分析、流动性双驱动、老千股识别 | 通用方法 + NAV 折让 + 股息率 | 港交所披露易/AAStocks |
| **加密货币** | 链上分析(MVRV/NVT)、叙事周期判断、代币经济学评估 | NVT/MVRV/S2F/协议 P/S | Glassnode/Dune/DeFiLlama |
| **固收/债券** | 利率分析、信用分析五维度、久期-凸性管理 | YTM/信用利差/OAS | 中债信息网/Wind |
| **基金** | Brinson 归因、因子归因、晨星风格箱、T-M 择时模型 | Sharpe/IR/Calmar/最大回撤 | 天天基金/晨星/Wind |

### 金融研报通用结构

**卖方深度研报**：封面(评级+目标价) → 核心摘要 → 公司/行业概况 → 核心逻辑 → 盈利预测(三表) → 估值(多方法交叉) → 风险提示

**买方投资备忘录**：投资论点 → Bull/Base/Bear 三情景 → 催化剂 → 风险 → 仓位建议 → 退出条件

## 四、分析视角选择

调研 skill 应根据用户角色和目的自动选择分析视角：

| 视角 | 触发词 | 核心问题 | 时间尺度 | 产出重点 |
|------|-------|---------|---------|---------|
| **券商/分析师** | 估值、值不值得买、目标价 | "值多少钱？" | 6-12 个月 | 估值 + 目标价 + 投资评级 |
| **咨询/战略** | 怎么做、战略、进入、优化 | "怎么赢？" | 3-5 年 | 战略选择 + 实施路径 |
| **VC/创业** | 赛道、创业、融资、机会 | "能不能 10x？" | 5-10 年 | 市场规模 + 壁垒 + 增长飞轮 |

**默认**：不确定时用咨询视角（最通用）。

## 五、框架自动选择规则

**输入**：调研类型 + 行业关键词 + 用户角色/目的

**选择逻辑**：
1. 根据调研类型，从通用底座中选择 2-3 个最相关的框架
2. 识别行业关键词 → 如果命中互联网/金融，叠加特化框架
3. 根据调研深度（快速/标准/深度）决定框架展开程度
4. 根据分析视角调整产出重点

### 调研类型 → 通用框架映射

| 调研类型 | 推荐的通用底座框架 |
|---------|----------------|
| 技术选型 | 价值链（理解技术在业务中的位置）+ 费米估算（成本估算） |
| 行业分析 | 全套：PESTEL → 生命周期 → 价值链 → 五力 → TAM → 竞争格局 |
| 概念研究 | 一般不需要通用底座，用概念研究专属框架 |
| 性能评测 | 一般不需要通用底座，用评测专属框架 |
| 竞品分析 | 竞争格局 + 价值链（理解竞品在产业链中的位置） |
| 可行性探索 | TAM/SAM/SOM（市场空间）+ 费米估算（成本/资源估算） |

### 选择示例

- **"Redis vs Dragonfly 技术选型"** → 通用：无需 + 互联网技术：选型矩阵 + ATAM → 咨询视角
- **"AI Agent 赛道分析"** → 通用：PESTEL + 竞争格局 + TAM + 互联网决策：创业机会框架 → VC 视角
- **"比亚迪 A 股深度研究"** → 通用：价值链 + 五力 + 金融：A 股框架 → 券商视角
- **"我们的测试体系做得怎么样"** → 通用：无需 + 互联网技术/测试：测试金字塔 + 质量度量 → 咨询视角（内部评估）
- **"SaaS 创业还有机会吗"** → 通用：PESTEL + 生命周期 + TAM + 竞争格局 + 互联网决策：创业机会 + 商业模式 → VC 视角
