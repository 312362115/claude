# Deep Research Skill 升级实施方案

## Context

当前 deep-research skill 是"会搜、会排版"的信息搜集工具，缺少"会想"的方法论。核心升级原则（已与用户对齐）：**评估不是独立类型，而是所有调研的底层逻辑——先摸清内部现状，再对比外部水位，找到差距，指导行动。**

关联 backlog：`docs/backlog/2026-03-29-deep-research-upgrade.md`
关联 spec（框架调研）：`docs/specs/2026-03-29-deep-research-analysis-frameworks.md`

## 子任务

### 任务 1：SKILL.md 主流程升级（核心，其他任务依赖此）

**Step 1 改造：「理解命题，摸清现状」**

- 1.1 命题解析：新增 2 个确认项（内部数据源、评估基准）
- 1.2 调研类型表：保留 6 类不变，加注释说明评估逻辑内嵌于所有类型
- **新增 1.4 形成初始假设**：搜索前拆解研究问题，提出 2-4 个可证伪假设
- **新增 1.5 现状摸底**：所有调研必做的内部数据采集前置步骤
  - 数据来源：本地代码库(Read/Grep) + 内部文档 + BI数据 + 用户口述
  - 按 6 种调研类型分别列出摸底重点
  - 无内部数据时标注"基于公开数据"
- 原 1.4 与用户对齐 → 改为 1.6，输出模板增加假设和内部数据源字段

**Step 2 改造：搜索策略升级**

- 2.1 搜索策略：第 2 层新增「反面证据搜索」环节
  - 关键词模板：`"<主题> limitations/problems/criticism/failure/risk"`
- 2.4 信息记录 → 改为「信息记录与来源分级」
  - 来源三级：一手[T1] / 二手[T2] / 三手[T3]
  - 数据属性：范围、年份、统计口径
  - 规则：核心结论须基于 T1 或 2个独立 T2
- 2.5 验证原则：新增假设对照检查

**Step 3 改造：分析框架升级**

- 3.2 分析框架：从薄表格改为「框架选择流程 + 引用 analysis-frameworks.md」
- 3.3 形成判断：扩充为「假设回顾 + 结论要求 + 反面观点」三部分

**Step 5 改造：报告输出升级**

- 5.2 报告结构：三层输出（Executive Summary ≤500字 + 正文 + 可选数据附录）
  - 正文新增：假设验证结果表、反面观点与回应章节
- 5.3 引用标注：脚注附带来源层级 [T1]/[T2]/[T3]
  - 参考资料按来源层级分组

**质量检查：从 8 项扩充为 4 类 18 项**

- 结论质量（可证伪性、假设验证、来源层级）
- 数据一致性（ES vs 正文 vs 图表数字一致）
- 逻辑完整性（推理链、反面观点）
- 来源与格式（引用、层级标记、时效性）
- 搜索覆盖自评

### 任务 3：新建 analysis-frameworks.md（可与任务 2 并行）

从 `docs/specs/deep-research-frameworks/` 4 份调研报告（共 ~183KB）**精炼**为运行时查询表（~300-400行）。

结构：
1. **通用底座框架**：7 个框架速查表 + 标准分析流程 + 各框架操作要点（精简版）
2. **互联网特化**：执行层 6 职能 + 决策层 6 场景（表格驱动）
3. **金融投资特化**：6 个领域（表格驱动）
4. **分析视角选择**：券商/咨询/VC 三种视角 + 触发词
5. **框架自动选择规则**：调研类型 × 行业 × 深度 → 框架组合

设计原则：**路由表，不是百科全书**。详细操作方法指向 specs 目录。

## 验证方案

### 结构完整性检查
- SKILL.md 5 步流程是否连贯可读
- 所有交叉引用路径正确（SKILL.md ↔ report-templates.md ↔ analysis-frameworks.md）
- 章节编号一致（1.1-1.6, 2.1-2.5, 3.1-3.3, 4.1-4.4, 5.1-5.4）

### 需求覆盖追踪

| 需求 | 落地位置 |
|------|---------|
| P0-1: 现状摸底（替代独立评估型） | SKILL.md 1.5 + templates "现状基线" |
| P0-2: 假设驱动 | SKILL.md 1.4 + 2.5 + 3.3 + 5.2(假设表) |
| P0-3: 反面证据 | SKILL.md 2.1(L2) + 3.3 + templates "反面观点" |
| P1-5: 来源分级 | SKILL.md 2.4(三级表) + 5.3(引用格式) |
| P1-6: 数据标准化 | SKILL.md 2.4(数据属性表) |
| P1-7: 多层次输出 | SKILL.md 5.2(ES + 正文 + 附录) |
| P2: 分析框架 | analysis-frameworks.md 全文 + SKILL.md 3.2 引用 |
| 质量检查升级 | SKILL.md 质量检查(4类18项) |

### 行为测试（dry run）
用"帮我调研是否应该从 Redis 迁移到 Dragonfly"走一遍升级后流程：
- 1.5 是否触发读内部代码库？
- 1.4 是否产出可检验假设？
- 2.1 L2 是否搜索反面证据？
- 3.3 是否产出假设验证表？
- 5.2 是否输出 ES + 假设表 + 反面观点？
