# Skill 瘦身 — 适配 Opus 4.7 字面化指令遵循

Claude Code 已升级到 Opus 4.7，指令遵循更字面化、长任务稳定性增强。老 skill 里为"保险"加的重复强调、防御性叮嘱（"必须/务必/切记"）在 4.7 下反而让模型困惑"到底听哪条"。扫描三份最长 skill 发现约 60 行可删（3.8%）。

## 扫描结果

| skill | 当前行数 | 可删行数 | 主要冗余类型 |
|-------|---------|---------|-------------|
| deep-research | 820 | ~20 | "必须/务必"强调词、数据采集方式重复、抓取纪律显而易见项 |
| task-execute | 384 | ~28 | "安全退出协议"重复叙述、道德化说教、评估规模表重复 |
| writing | 375 | ~13 | "必须/不能省"强调词、边界划分冗长 |

详细清单见 2026-04-17 的会话记录（扫描 agent 已产出逐行精简建议）。

## 执行原则

- **只去冗余，不动逻辑**：分析框架、决策树、示例模板、核心流程全部保留
- **优先级**：task-execute（日常用得最频繁，收益最大）→ deep-research → writing
- **每改一个 skill 做一次回归**：确保行为未退化再改下一个
- **后续扫描范围**：task-manager (364)、diagram (329)、task-start (306) 等 300+ 行 skill

## 预期收益

- 每次会话少吃 token（新 tokenizer 消耗 1~1.35x，skill 文件越长越贵）
- skill 更清爽，未来维护成本下降
- 适配 4.7 指令遵循风格，减少"重复叮嘱反而困惑模型"的隐性成本
