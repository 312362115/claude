# Deep Research Skill 升级：从信息搜集到专业研究引擎

## 背景

当前 deep-research skill 的流程骨架和工程化程度不错（5 步流程、6 套报告模板、5 套视觉主题、config-driven 图表生成），但在研究方法论深度上偏浅——更像是"系统化的信息搜集 + 整理工具"，而不是"能产出有深度判断的研究引擎"。核心差距：**会搜、会排版，但缺少"会想"的方法论指导**。

## 目标

让 skill 产出的报告从"信息罗列"升级为"有深度判断、经得起推敲的专业研究报告"。

## 关键任务

### P0 — 研究方法论补强（核心短板）

1. **新增"评估型调研"类型**
   - 当前 6 种类型都是"向外看"（调研外部技术/行业/竞品），缺少"向内看"的评估能力
   - 新增调研类型：**现状评估 / 差距分析（Gap Analysis）**
   - 适用场景：评估已有系统/产品/流程距离行业标准或最佳实践的差距
   - 方法论流程：
     1. **建立评估基准**：确定"专业标准"是什么（行业最佳实践、竞品水位、规范要求）
     2. **拆解评估维度**：将大命题拆成独立可评的子维度（MECE 原则）
     3. **逐维度现状梳理**：对每个维度描述当前状态，给出定性/定量评价
     4. **差距识别与归因**：每个维度距离基准差多少，根因是什么
     5. **排优先级**：按影响面 × 改进成本给差距项排序
     6. **输出改进路径**：P0/P1/P2 分级的行动建议
   - 报告模板需新增（见下方模板章节）
   - 典型触发词："评估一下 xxx"、"xxx 做得怎么样"、"距离专业还差哪"、"审计"、"诊断"

2. **假设驱动研究流程**（适用于所有调研类型，包括评估型）
   - 搜索前先拆解研究问题（Research Questions），提出初始假设
   - 搜索过程围绕"证实/证伪假设"展开，而非漫无目的地搜集
   - 结论部分回扣假设：哪些被证实、哪些被推翻、哪些证据不足

3. **反面证据强制搜索**
   - 在第 2 层搜索中增加"devil's advocate"环节：对已形成的初步结论，主动搜索反面证据
   - 搜索关键词模板：`"<主题> limitations/problems/criticism/failure"`
   - 报告中必须包含"反面观点与回应"章节

4. **结构图工具统一**
   - deep-research 的结构图弃用 graphviz，改为复用 diagram skill 的 HTML/SVG + ELKjs 方案
   - 统计图仍用 matplotlib（diagram skill 不覆盖此类）
   - 消除两套工具链割裂，提升结构图美观度和一致性

### P1 — 数据质量与报告层次

5. **数据来源分级体系**
   - 一手数据（官方财报、论文实验、官方文档）→ 高可信
   - 二手数据（媒体报道、技术博客、分析师报告）→ 中可信
   - 三手数据（社区讨论、AI 生成摘要、转引）→ 低可信，需交叉验证
   - 正文引用时标注来源级别，关键结论必须基于一手或二手数据

6. **数据标准化规范**
   - 搜集数据时记录口径（地域范围、时间范围、包含/排除条件）
   - 不同来源的数据对比前先统一口径
   - 报告中所有数据标注来源年份

7. **多层次输出**
   - 报告开头增加 **Executive Summary**（500 字以内，只含结论和建议，面向管理层）
   - 正文保持当前深度（面向执行者）
   - 可选：末尾附数据附录（原始数据表、搜索记录）

### P2 — 分析深度增强

8. **行业分析框架适配（互联网 + 金融，两条线）**

   **互联网行业** — 按职能环节提供分析框架：

   | 职能 | 专属分析框架 |
   |------|------------|
   | 产品 | 用户旅程分析、功能价值矩阵（频次×价值）、留存/激活漏斗、PMF 验证框架、需求优先级（RICE/ICE） |
   | 技术 | 技术选型决策矩阵、架构评审维度（可扩展性/可维护性/成本）、技术债评估、性能基准对比 |
   | 设计 | 可用性评估（尼尔森十原则）、设计系统成熟度、竞品 UI/UX 对比、无障碍合规检查 |
   | 测试 | 测试覆盖度分析、质量指标体系（缺陷密度/逃逸率/MTTR）、自动化成熟度评估 |
   | 运营 | 用户生命周期价值（LTV）、获客成本（CAC）、增长飞轮分析、留存曲线、活跃度分层（RFM） |
   | 市场 | 市场规模测算（TAM/SAM/SOM）、竞争格局（CR3/CR5/HHI）、GTM 策略分析、品牌定位图 |

   **互联网行业** — 决策层视角（部门负责人/CEO）：

   | 场景 | 专属分析框架 |
   |------|------------|
   | 创业机会评估 | 赛道筛选（市场规模×增速×竞争格局）、时机判断（技术成熟度曲线/政策窗口）、创始团队-赛道匹配度、冷启动路径设计 |
   | 商业模式分析 | 商业画布（9 要素）、收入模型拆解（交易/订阅/广告/佣金）、单位经济模型（UE）、盈利路径与时间线、定价策略分析 |
   | 战略决策 | 第二曲线识别、Build vs Buy vs Partner 决策框架、护城河评估（网络效应/规模效应/转换成本/数据壁垒）、退出策略分析 |
   | 业务健康度 | 核心指标仪表盘（GMV/收入/利润/现金流）、增长质量诊断（有机增长 vs 烧钱增长）、组织效能（人均产出/人力成本占比） |
   | 融资与估值 | 估值方法选择（PS/PE/DCF/可比公司）、融资节奏规划、投资人视角的业务拆解、BP 核心叙事提炼 |
   | 竞争战略 | 竞争对手动态监控框架、差异化定位（价值曲线/蓝海策略）、攻防策略制定、生态位分析 |

   **金融投资行业** — 按投资领域提供分析框架：

   | 领域 | 专属分析框架 |
   |------|------------|
   | A 股 | 基本面分析（PE/PB/ROE/现金流）、行业轮动、政策面解读、北向资金/融资融券指标、板块热度 |
   | 美股 | 财报季分析框架、DCF 估值、EPS 增速、机构持仓变化、美联储政策影响链 |
   | 港股 | AH 溢价分析、南向资金流向、港股通标的筛选、汇率影响评估 |
   | 加密货币 | 链上数据分析（活跃地址/TVL/Gas）、代币经济模型评估、赛道叙事周期、监管政策跟踪 |
   | 固收/债券 | 收益率曲线分析、信用利差、久期/凸性、宏观利率周期判断 |
   | 基金 | 业绩归因分析（Alpha/Beta 分离）、夏普比率/最大回撤/Calmar、风格漂移检测、费率对比 |

9. **补充图表类型**
   - 统计图新增：瀑布图（成本/收益拆解）、桑基图（流量/转化分析）、树图 Treemap（占比层级）
   - 时间线图：用于技术演进、产品迭代里程碑展示

10. **质量检查升级**
   - 增加：结论可推翻性检查（每个结论能否被反面证据推翻）
   - 增加：数据一致性检查（摘要、正文、图表中的数字是否一致）
   - 增加：逻辑链完整性检查（事实→分析→结论的推理链是否有跳跃）
   - 增加：搜索覆盖度自评（是否遗漏重要视角）

## 验收标准

- **外部调研验证**：用同一个命题（如"2026 AI Coding 工具竞品分析"），对比升级前后：
  - 结论是否更有判断力（不只是罗列信息）
  - 是否包含反面观点和回应
  - 数据来源是否分级标注
  - 图表是否统一使用 diagram skill 风格

- **评估型调研验证**：用"评估 xxx skill 的专业度"类命题，验证：
  - 是否自动识别为评估型调研并选择对应模板
  - 是否建立了清晰的评估基准（而非凭空打分）
  - 维度拆解是否 MECE（不重不漏）
  - 差距分析是否有根因归纳（而非只列现象）
  - 改进建议是否按优先级排序且可执行

## 非目标

- 不做实时数据监控/自动更新
- 不做多人协作编辑报告
- 不做自动化定期产出（可用 schedule skill 手动配置）
