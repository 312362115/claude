# Diagram Skill 能力补齐 — 对标 Mermaid

## 背景

对比 Mermaid（v11.13.0）完整能力，当前 diagram skill 在多个图表类型上存在差距：流程图节点形状/连线不足，类图缺 UML 标准箭头，ER 图缺 crow's foot 符号，状态图缺复合状态，同时缺少 Treemap/Kanban 等图表类型。核心问题是业务场景覆盖不足，不是样式问题（统一设计规范是我们的优势）。

---

## 一、流程图（Flowchart）

### P1 — 高价值，常见场景刚需

| 缺失能力 | 说明 | 场景 |
|---------|------|------|
| **节点形状扩展** | 缺平行四边形（I/O）、梯形、文档形、双圆（双边框）、旗帜、不对称形状等 20+ 种 | 标准流程图规范要求 I/O 用平行四边形，文档用波浪底 |
| **连线类型扩展** | 缺双向箭头、圆头（关联）、叉头（拒绝/终止）、粗线（强调） | 双向数据流、拒绝/终止语义表达 |
| **节点内富文本** | 当前纯文本，缺加粗、多行、关键词高亮 | 节点描述需要突出关键信息 |

### P2 — 中等价值，复杂场景需要

| 缺失能力 | 说明 | 场景 |
|---------|------|------|
| **子图嵌套** | 当前最多 6 个平级分组，不支持嵌套 | 复杂系统架构、多层级组织结构 |
| **BT/RL 布局** | 当前只有 TD 和手动 LR | 数据管道反向追溯、逆向分析 |
| **图标节点** | 当前禁用图标 | 架构图带图标比纯文字专业 |

## 二、类图（Class Diagram）— UML 标准箭头缺失

### P1 — UML 规范刚需

| 缺失能力 | Mermaid 语法 | 说明 |
|---------|-------------|------|
| **聚合** | `o--` 空心菱形+实线 | 弱包含关系（部分可独立存在），当前完全没有 |
| **依赖** | `..>` 普通箭头+虚线 | 临时使用关系，当前完全没有 |
| **基数标记** | `"1" --> "*"` | 关系多重性标注，UML 核心要素，当前完全没有 |
| **继承/实现箭头形状** | `<\|--` 空心三角 / `..\|>` 空心三角+虚线 | 当前用文本标签替代，缺标准 UML 箭头图形 |

### P2

| 缺失能力 | Mermaid 语法 | 说明 |
|---------|-------------|------|
| **棒棒糖接口** | `()--` | 接口的简写表示法，UML 标准但使用频率低 |

---

## 三、ER 图（Entity Relationship）— 关系符号缺失

### P1 — ER 图核心功能

| 缺失能力 | Mermaid 语法 | 说明 |
|---------|-------------|------|
| **Crow's foot 基数符号** | `\|\|--o{` 等图形标记 | 当前只用文本 `1:N` 标注，不够专业。crow's foot 是 ER 图行业标准 |
| **Identifying / Non-identifying 关系** | 实线 `--` / 虚线 `..` | 当前不区分强弱依赖关系 |

### P2

| 缺失能力 | 说明 |
|---------|------|
| **UK（Unique Key）标记** | 当前只有 PK/FK，缺唯一键 |
| **属性注释** | Mermaid 支持属性行内 comment |

---

## 四、状态图（State Diagram）— 复杂状态缺失

### P1

| 缺失能力 | Mermaid 语法 | 说明 |
|---------|-------------|------|
| **复合/嵌套状态** | `state id { ... }` 多层嵌套 | 复杂状态机的核心能力，当前完全没有 |
| **Choice 伪状态** | `<<choice>>` | 条件分支节点，状态机设计常用 |

### P2

| 缺失能力 | Mermaid 语法 | 说明 |
|---------|-------------|------|
| **并发状态** | `--` 分隔并行区域 | 并行执行的状态区域 |
| **Fork/Join** | `<<fork>>` `<<join>>` | 并行路径的分叉与汇合 |
| **Notes** | 左右定位注释 | 状态说明文字 |

---

## 五、新图表类型

### P2 — 有价值的新类型

| 图表类型 | Mermaid 能力 | 价值说明 |
|---------|-------------|---------|
| **Treemap（矩形树图）** | 层级矩形、数值格式化、样式自定义 | 数据占比可视化，调研报告/数据分析刚需，比饼图信息密度高 |
| **柱线混合图** | XY Chart 支持 bar + line 叠加 | 当前柱状图/折线图分开，不支持混合 |

### 第三批

| 图表类型 | 说明 |
|---------|------|
| **Kanban** | 项目管理可视化 |
| **Git Graph** | Git 工作流教学/文档 |

---

## 六、不补 — 与我们定位不符

| 能力 | 不补原因 |
|------|---------|
| 交互功能（click/URL） | 输出 PNG/HTML 静态场景，不需要 |
| 边动画 | 同上 |
| 自定义样式（classDef） | 统一设计规范是我们的优势 |
| 连线长度手动控制 | ELKjs 自动布局已覆盖 |

---

## 实施建议

**第一批（影响面最大）**：
- 类图 UML 标准箭头（聚合、依赖、基数标记、继承/实现箭头图形）
- ER 图 crow's foot 符号 + identifying/non-identifying 区分
- 流程图节点形状先补 5 个最常用（平行四边形、文档形、双圆、梯形、旗帜）

**第二批**：
- 状态图复合状态 + choice
- 流程图连线扩展 + 子图嵌套
- Treemap 新图表类型

**第三批**：
- 剩余流程图形状
- 状态图并发/fork-join
- 柱线混合图
- 图标节点、富文本
- Kanban
- Git Graph
