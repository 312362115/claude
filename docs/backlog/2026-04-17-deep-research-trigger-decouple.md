# deep-research skill 触发失效 + 方法论与样式解耦

## 现象

Opus 4.7 下用户说「深度调研下，现有哪些方式获取抖音视频搜索/下载 API 或爬虫能力，结论输出 HTML 报告」时，**没有触发 deep-research skill**，模型直接用 WebSearch + Write 产出了 `/Users/renlongyu/workspace/douyin/douyin-video-api-research.html`。

意外观察：**模型自由输出的 HTML 样式明显优于 skill 既有模板**（抖音品牌色、hero + TOC + 卡片分区、信息密度高）。

## 根因假设

1. **Opus 4.7 对"方法论型 skill"的主观尊重度下降**——老模型字面匹配触发词就调，4.7 会二次判断"skill 真的比我直接干更好吗"，觉得 HTML 样式自己能做得更好就跳过
2. **skill description 偏描述性**（"覆盖什么场景 / 方法论是什么"），缺少**后果导向**的强制指令（"跳过会漏掉什么"）
3. **skill 把方法论和 HTML 样式绑在一起**——模型隐式权衡"方法论我能替代一部分，样式我比你强"，总账算下来跳过更划算

## 本次输出对比 skill 方法论清单

| 维度 | 本次输出 | skill 应做 | 评价 |
|------|---------|-----------|------|
| 反面证据强制搜索 | ✅ 判例 / 申请失败案例 / 投入产出比警告 | 强制搜索 | 持平 |
| 结构完整性 | ✅ 8 章 + TL;DR + 决策树 | 自适应结构 | 持平 |
| **数据来源三级分类** | ❌ 未分级 | 必做 | **缺失** |
| **结论置信度分级（高/中/低）** | ❌ 未标注 | 必做 | **缺失** |
| **质量自检（Step 6）** | ❌ 未做局限性说明 | 必做 | **缺失** |
| 时效性标注 | ⚠️ 部分（gh api 抓取时间注明） | 强调 | 部分 |
| 图表 | ⚠️ HTML pre 决策树 | Diagram Bridge PNG | skill 更专业 |
| HTML 样式 | ✅✅ 品牌化、信息密度高 | 通用模板 | **模型更强** |

**结论**：模型自由发挥在样式和结构上达到专业水准，但 skill 的三项核心方法论（来源分级 / 置信度 / 自检）被完全漏掉——这些正是 skill 不可替代的价值。

## 待决策方向

### 方向 A：强化 description 指令性

把 description 从"介绍能力"改为"强制指令 + 跳过后果"：
> 研究 / 调研 / 出报告类任务**必须**走此 skill。直接用 WebSearch 出报告会漏掉：反面证据搜索、数据来源三级分类、结论置信度分级、质量自检——这些是此 skill 存在的唯一理由。

触发词块前置并收紧：
> 命中以下任一词立即调用，不做二次判断：调研 / 深度调研 / 研究一下 / 分析报告 / 出一份报告 / 竞品分析 / 技术选型。

### 方向 B：解耦方法论和样式

让 skill 只管"调研质量"：
- skill 输出结构化中间产物（含置信度 / 来源分级的 markdown）
- HTML 渲染这一步独立出去，让模型自由发挥或显式调 frontend-design

这样 skill 的触发理由变得不可替代（方法论严谨性），样式优势保留。

### 方向 C：不改 skill，改用显式 slash command

接受"Opus 4.7 对方法论型 skill 主观阈值变高"的事实，改用 `/deep-research <topic>` 显式调用绕过判断。

## 执行建议

先做方向 A（小成本、可逆、立即见效），观察 1–2 个真实调研请求的触发率。若仍失败，再评估方向 B（架构调整，工作量大）。方向 C 作为日常兜底。
