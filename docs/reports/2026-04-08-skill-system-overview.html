    Claude Code Skill 体系全景

    20 个 skill 如何覆盖从需求到上线的完整开发流程

    日期 2026-04-08 汇报类型 能力体系介绍 技术栈 JS/TS + Python

    核心摘要

    构建了覆盖需求管理 → 方案设计 → 编码执行 → 安全审查 → 发版上线全生命周期的 20 个 skill 体系。采用流程与能力分离架构——5 个流程 skill 负责调度"做什么"，15 个能力 skill 负责"怎么做"。核心闭环：task-start 检索历史经验 → 执行 → task-finish 产出新经验 → docs/decisions/ 沉淀。

    写作模板
    架构理念：流程与能力分离

    流程 skill 负责调度"什么时候做什么"，自己不产出具体内容。
    能力 skill 负责"怎么做好某件事"，被流程 skill 按需调用。
    这样每个 skill 职责单一、可独立使用，也能被组合成完整流程。

    流程 Skill 5 个

    串联整个开发生命周期的骨架，每个 skill 在流程中有明确的位置。

    流程

    task-manager

    需求管理：backlog 梳理、优先级排序、过期清理。管"做不做、做哪个"。

    流程

    task-start

    任务启动：需求对焦 + 历史经验检索 + 调度写方案/选型/调研/原型。

    流程

    task-execute

    持续执行：跨会话大型任务的进度管理和上下文维护。

    流程

    task-finish

    任务收尾：CR 自检 + 强制复盘（轻量/完整）→ docs/decisions/ 沉淀。

    流程

    refactoring

    重构流程：影响分析 → 安全网搭建 → 分步重构 → 回归验证。

    能力 Skill 15 个

    按能力域分为 5 组，覆盖开发者日常工作的各个方面。

    安全与质量

    能力

    能力

    前后端性能分析：Core Web Vitals、API 响应、数据库查询、优化前后对比。

    经验闭环机制

    解决"大任务交付后反复返工但没有复盘记录"的核心痛点。

    task-start 对焦阶段
    自动检索 docs/decisions/ 历史经验，发现相关踩坑记录时主动提醒："上次类似任务踩过这些坑..."
    task-execute 执行阶段
    基于历史经验规避已知风险，减少重复踩坑。
    task-finish 复盘阶段
    强制复盘判断（不可跳过）→ 轻量/完整复盘 → 写入 docs/decisions/ 作为真相源 → 提取关键经验存 memory 作为跨会话索引。
    docs-management 知识库维护
    Ingest 操作将复盘产出整合到相关文档，Lint 操作定期检查文档健康度，确保知识不腐烂。
    闭环效果

    docs/decisions/ 随 git 提交，团队永久可见。每次新任务启动时自动检索，历史经验变成了团队资产而非个人记忆。
