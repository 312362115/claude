# Deep Research Skill — 行业分析框架体系设计

> 基于 4 个方向的系统性调研（通用行业分析、互联网执行层、互联网决策层、金融投资），提炼出 deep-research skill 应内置的分析框架、报告模板、数据采集方法和结论推导方法。

**日期**：2026-03-29
**关联需求**：`docs/backlog/2026-03-29-deep-research-upgrade.md`
**调研参考**：见 `docs/specs/deep-research-frameworks/` 目录下 4 份详细调研

## 一、总体架构：三层框架体系

```
┌─────────────────────────────────────────────┐
│           通用行业分析（底座）                  │
│  PESTEL / 五力 / 价值链 / TAM-SAM-SOM /      │
│  行业生命周期 / 竞争格局 / 费米估算            │
├──────────────────────┬──────────────────────┤
│   互联网行业（特化层） │   金融投资（特化层）    │
│                      │                      │
│  执行层：6 个职能     │  6 个投资领域          │
│  决策层：6 个场景     │  A股/美股/港股/        │
│                      │  加密/债券/基金        │
└──────────────────────┴──────────────────────┘
```

- **底座**：任何行业都适用的通用方法论，所有调研默认先走底座
- **特化层**：针对互联网/金融的专属框架，在底座基础上叠加

## 二、数据来源三层模型

调研 skill 的数据采集应区分三层来源，不同来源的获取方式和可信度不同：

| 层级 | 来源 | 获取方式 | 可信度 | 示例 |
|------|------|---------|--------|------|
| **公开数据** | 行业报告、财报、政府统计、开源数据、新闻 | WebSearch / WebFetch / Playwright | 需交叉验证 | Statista、国家统计局、SEC EDGAR、招股书 |
| **内部数据** | 公司 BI 系统、内部代码库、内部文档、数据库报表 | Read / Grep 读本地文件、用户提供 | 高（一手） | 内部销售数据、埋点数据、内部复盘文档、代码仓库 |
| **半公开数据** | 付费数据库、专家访谈纪要、行业社群 | 用户提供路径或授权访问 | 中-高 | Wind、Bloomberg、GLG 纪要 |

### 关键设计要点

1. **调研开始时主动询问**：是否有内部数据/文档可供参考？路径在哪？
2. **内部数据优先**：有内部数据时以内部数据为主，公开数据做交叉验证
3. **数据来源标注**：报告中每个数据点标注来源层级（公开/内部/半公开）
4. **内部数据的典型场景**：

| 调研类型 | 内部数据用途 |
|---------|------------|
| 技术选型 | 读现有代码了解技术栈、读性能监控数据、读内部 ADR |
| 业务健康度 | 读 BI 报表/数据库产出、内部 OKR 文档 |
| 测试质量评估 | 读测试覆盖率报告、CI/CD 日志、缺陷管理系统数据 |
| 竞品分析 | 读内部竞品跟踪文档、客户反馈记录 |
| 商业模式分析 | 读财务报表、单元经济学数据、用户行为数据 |

## 三、通用行业分析框架（底座）

### 3.1 核心框架清单

| 框架 | 适用场景 | 核心产出 |
|------|---------|---------|
| **PESTEL** | 宏观环境扫描 | 关键驱动因素 + 机会/威胁清单 |
| **波特五力** | 行业结构评估 | 行业吸引力评分 + 利润分配判断 |
| **行业生命周期** | 判断行业阶段 | 所处阶段 + 对应策略 |
| **价值链分析** | 理解行业运作 | 产业链图 + 利润池分布 |
| **TAM/SAM/SOM** | 市场规模测算 | 市场天花板 + 可获取空间 |
| **竞争格局分析** | 了解谁在赢 | CR5/HHI + 战略组图 + 竞品矩阵 |
| **费米估算** | 数据缺失时快速估算 | 量级判断 + 不确定性范围 |

### 3.2 标准分析流程

```
第一步：PESTEL（宏观环境，快速定性扫描）
第二步：行业生命周期（判断阶段，决定后续分析重点）
第三步：价值链分析（理解行业怎么运作、钱怎么流）
第四步：波特五力（判断行业好不好赚钱）
第五步：TAM/SAM/SOM（市场多大）
第六步：竞争格局（谁在玩、谁在赢）
第七步：趋势预判（Gartner Hype Cycle / S 曲线 / 创新扩散）
```

### 3.3 行研节奏（三档深度）

| 深度 | 时间 | 产出 | 适用场景 |
|------|------|------|---------|
| **快速摸底** | 4-8 小时 | 3-5 页摸底报告 | 初步了解、快速判断 |
| **标准调研** | 1 周 | 10-15 页标准报告 | 技术选型、竞品分析 |
| **深度研究** | 2-4 周 | 20-50 页深度报告 | 行业进入决策、投资决策 |

### 3.4 通用报告模板（快速版 3-5 页）

```
1. 行业概览（定义 + 产业链图 + 生命周期阶段）
2. 市场规模与增长（规模 + 增速 + 驱动因素）
3. 竞争格局（CR5 + 主要玩家对比表）
4. 盈利模式与关键指标（商业模式 + 核心财务指标 + KSF）
5. 趋势与关键变量（正在发生的变化 + 风险）
6. 初步判断与待深入问题
```

### 3.5 通用报告模板（深度版 20-50 页）

```
Executive Summary（1-2 页）
第一部分：行业全景（定义/产业链/发展历程/宏观环境）
第二部分：市场分析（规模测算/供需/价值链/利润池）
第三部分：竞争格局（集中度/五力/战略组图/头部对比）
第四部分：趋势与前瞻（技术/商业模式/监管/拐点判断）
第五部分：判断与建议（吸引力评分/KSF/风险/结论）
附录（数据汇编/术语/参考资料）
```

## 四、互联网行业特化框架

### 4.1 执行层（6 个职能）

| 职能 | 核心框架 | 典型报告 | 关键数据源 |
|------|---------|---------|-----------|
| **产品** | KANO 模型、RICE 评分、北极星指标、用户故事地图 | PRD 扩展版 | 埋点数据（内部）、用户调研、竞品分析 |
| **技术** | ATAM、C4 模型、技术选型矩阵、ADR、USE 方法 | 技术方案评审文档 | 压测数据（内部）、监控数据（内部）、官方文档 |
| **设计** | HEART 框架、Nielsen 十大启发式、Design Sprint | UX 审计报告 | 热力图（内部）、可用性测试、竞品截图 |
| **测试** | 测试金字塔、敏捷测试象限、质量度量指标体系 | 测试分析报告 | CI/CD 数据（内部）、缺陷系统（内部）、覆盖率报告（内部） |
| **运营** | AARRR、RFM 模型、用户生命周期、北极星+OSM | 运营分析报告 | 埋点数据（内部）、BI 报表（内部）、行业基准 |
| **市场** | STP、4P、PESO 媒体模型、品牌健康度追踪 | 市场分析报告 | 投放数据（内部）、品牌调研、社交媒体监测 |

### 4.2 决策层（6 个场景）

| 场景 | 核心框架 | 典型报告 | 关键数据源 |
|------|---------|---------|-----------|
| **创业机会评估** | TAM-SAM-SOM、Thiel 七问、Lean Validation、PMF 测试 | Pitch Deck（投资人版）/ 内部立项报告 | 行业报告、投融资数据、竞品数据 |
| **商业模式分析** | 商业模式画布、单元经济学（LTV/CAC）、收入模型类型学 | 商业模式分析报告 | 财务数据（内部）、对标公司财报 |
| **战略决策** | 波特五力、McKinsey 7S、Ansoff 矩阵、BCG 矩阵 | 战略规划文档 | 内部经营数据、竞品情报、专家访谈 |
| **业务健康度** | a16z 16 指标、Sacks SaaS 指标体系、AARRR、Cohort 分析 | 业务健康度仪表盘 | BI 系统（内部）、产品数据（内部） |
| **融资与估值** | VC 估值法、可比公司分析、Berkus 方法、记分卡法 | BP / 融资文档 | 融资数据库、可比公司财报 |
| **竞争战略** | 护城河分析、蓝海战略、颠覆式创新、飞轮效应 | 竞争战略报告 | 竞品数据、用户调研、专利数据 |

## 五、金融投资特化框架

### 5.1 六大投资领域

| 领域 | 核心分析框架 | 核心估值方法 | 关键数据平台 |
|------|------------|------------|------------|
| **A 股** | 自上而下宏观驱动、景气度投资、行业轮动 | PE/PB/PEG/DCF（按行业选） | Wind/Choice/东方财富 |
| **美股** | 基本面分析、DCF、可比公司分析（Comps） | DCF + Comps（Football Field） | Bloomberg/SEC EDGAR/Finviz |
| **港股** | AH 溢价分析、流动性双驱动、老千股识别 | 通用方法 + NAV 折让 + 股息率 | 港交所披露易/AAStocks |
| **加密货币** | 链上分析（MVRV/NVT）、叙事周期、代币经济学 | NVT/MVRV/S2F/协议 P/S | Glassnode/Dune/DeFiLlama |
| **固收/债券** | 利率分析、信用分析五维度、久期-凸性管理 | YTM/信用利差/OAS | 中债信息网/Wind |
| **基金** | Brinson 归因、因子归因、晨星风格箱、T-M 择时模型 | Sharpe/IR/Calmar/最大回撤 | 天天基金/晨星/Wind |

### 5.2 金融研报的通用结构

```
卖方深度研报：
  封面（评级+目标价） → 核心摘要 → 公司/行业概况 → 核心逻辑
  → 盈利预测（三表） → 估值（多方法交叉） → 风险提示

买方投资备忘录：
  投资论点 → Bull/Base/Bear 三情景 → 催化剂 → 风险
  → 仓位建议 → 退出条件
```

## 六、评估型调研框架（新增类型）

适用场景：评估已有事物距离专业标准的差距。

### 标准流程

```
1. 建立评估基准（行业最佳实践 / 标准规范 / 竞品水位）
2. 拆解评估维度（MECE 原则，互斥且穷尽）
3. 逐维度现状梳理（定性描述 + 定量评价）
4. 差距识别与归因（差多少 + 根因是什么）
5. 排优先级（影响面 × 改进成本）
6. 输出改进路径（P0/P1/P2 分级行动建议）
```

### 报告模板

```
1. 评估概述（评估对象 + 基准来源 + 方法说明）
2. 评估维度与基准（维度清单 + 每个维度的"专业标准"）
3. 现状评估（逐维度：当前状态 + 做得好的 + 差距）
4. 差距分析（差距汇总表 + 根因归类 + 优先级排序）
5. 改进路径（P0/P1/P2 行动建议 + 验收标准）
```

## 七、三种视角的切换逻辑

调研 skill 应根据用户角色和目的自动选择分析视角：

| 视角 | 核心问题 | 适用用户 | 时间尺度 |
|------|---------|---------|---------|
| **券商/分析师** | "值多少钱？" | 投资者 | 6-12 个月 |
| **咨询/战略** | "怎么赢？" | 企业管理层 | 3-5 年 |
| **VC/创业** | "能不能 10x？" | 创业者/投资人 | 5-10 年 |

触发规则：
- 用户问"估值"、"值不值得买"→ 券商视角
- 用户问"怎么做"、"战略"、"进入"→ 咨询视角
- 用户问"赛道"、"创业"、"融资"→ VC 视角
- 不确定时默认咨询视角（最通用）

## 八、结论推导方法论（通用）

### 假设驱动研究法

```
1. 提出假设（"我认为 X 行业正在经历 Y 变化，因为 Z"）
2. 拆解验证点（MECE 拆解为可验证的子假设）
3. 搜集数据（围绕子假设搜索，而非漫无目的）
4. 验证/证伪（每个子假设标注：证实/证伪/证据不足）
5. 更新假设（基于证据修正初始假设）
6. 形成结论（回扣假设，写出推理链）
```

### 反面证据强制搜索

在第 2 层搜索中增加 "devil's advocate" 环节：
- 对已形成的初步结论，主动搜索反面证据
- 搜索关键词：`"<主题> limitations/problems/criticism/failure/risk"`
- 报告中必须包含"反面观点与回应"章节

### 结论质量检查

- [ ] 每个核心结论都有数据或事实支撑？
- [ ] 结论经得起"如果反过来呢？"的追问？
- [ ] 数据来源分级标注？（公开/内部/半公开）
- [ ] 存在不确定性的地方标注了置信度？
- [ ] 报告中的数字前后一致？
- [ ] 推理链完整，没有逻辑跳跃？
