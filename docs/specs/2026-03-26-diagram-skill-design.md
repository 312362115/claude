# 画图 Skill 方案设计

## 背景与动机

日常开发中经常需要画各种图：架构图、流程图、时序图、统计图表等。手动用 draw.io 等工具画图效率低，且风格不统一。希望通过一个 Claude skill，用自然语言描述需求即可自动生成专业的图表 PNG，直接嵌入 Markdown 文档。

## 现状分析

- 当前画图靠手动操作 draw.io、ProcessOn 等 GUI 工具，耗时且风格难统一
- Claude 擅长生成文本 DSL（Mermaid、D2）和结构化代码（HTML/JS），适合自动化画图
- 市面上没有一个工具能覆盖所有图表类型，需要组合方案

## 调研与备选方案

### 方案 A：draw.io XML 生成
- **调研内容**：draw.io 支持 36+ 种图表，覆盖最广
- **验证结果**：XML 格式冗长复杂，一个中等图需 200+ 行 XML，Claude 生成容易出错
- **结论**：覆盖广但生成成本太高

### 方案 B：单一工具（Mermaid 或 D2）
- **调研内容**：Mermaid 覆盖 23/36 种（64%），D2 覆盖 14/36 种（39%）
- **验证结果**：单一工具覆盖率不够，且各有独占优势区
- **结论**：需要组合使用

### 方案 C：D2 + Mermaid + ECharts + HTML/SVG 四层组合
- **调研内容**：每种工具负责最擅长的图表类型
- **验证结果**：覆盖率 ~95%，每种图都用最合适的工具
- **结论**：最优方案

## 决策与取舍

采用方案 C（四层组合）。

- **核心理由**：覆盖率最高，且每种图都能用最擅长的工具画出最好效果
- **取舍说明**：工具数量多（4 种），skill 实现复杂度高，但图表质量有保障
- **风险**：需要统一设计规范确保视觉一致性

## 技术方案

### 工具分层

| 层 | 工具 | 覆盖类型 | PNG 生成方式 |
|---|------|---------|-------------|
| 结构图（专业美观） | **D2** | 流程图、序列图、状态图、ER图、网络拓扑、决策树、数据流图 | `d2 input.d2 output.png` |
| 结构图（D2不支持的） | **Mermaid** | 类图、C4模型、思维导图、甘特图、时间线 | `mmdc -i input.mmd -o output.png` |
| 统计图表 | **ECharts** | 柱状图、折线图、饼图、雷达图、桑基图、热力图等 36+ 种 | HTML → Playwright 截图 |
| 特殊图表 | **自定义 HTML/SVG** | 架构图（分层式）、SWOT、鱼骨图、组织结构图、文氏图、旅程图、泳道图 | HTML → Playwright 截图 |

### 关键发现

1. **架构图必须用 HTML/CSS**：分层堆叠式架构图是布局问题，D2/Mermaid 的图论布局引擎无法精确控制层级关系
2. **D2 和 Mermaid 互补明显**：
   - D2 独占：架构图、网络拓扑、数据流图（样式更好看）
   - Mermaid 独占：类图、C4 模型、思维导图、甘特图、时间线
3. **统计图表 ECharts 最全**：36+ 种图表类型，交互式渲染后截图为 PNG

### 依赖环境

- D2：`brew install d2`（单二进制，无依赖）
- Mermaid CLI：`npm install -g @mermaid-js/mermaid-cli`
- Playwright：`pip3 install playwright`（用于 ECharts/HTML 截图）

### 输出规范

- 格式：PNG
- 嵌入方式：`![图表名](./images/xxx.png)`
- 存放位置：与 Markdown 文件同级的 `images/` 目录

## 待完成：统一设计规范

当前各图表样式不统一，需要制定一套设计规范，确保所有图的视觉风格一致。

### 需要确定的内容

1. **调色板** — 主色、层级色、语义色，低饱和度专业风格
2. **字体** — 中英文字体栈、字号层级、字重
3. **组件样式** — 圆角、边框、填充、阴影规则
4. **布局规则** — 间距网格、对齐方式
5. **连线规范** — 线宽、颜色、箭头、虚线用法
6. **跨工具一致性** — D2 theme、Mermaid theme、ECharts theme、CSS 变量统一

### 设计原则

- 现代简洁，参考 Stripe/Linear 风格
- 低饱和度配色，不花哨
- 4-6px 圆角，微阴影（0 2px 4px rgba(0,0,0,0.08)）
- 不用粗色条、不用 emoji icon
- 所有图表视觉上统一协调
