# 通用行业分析方法论：从零到专业报告的元能力体系

> 拿到任何一个陌生行业，都能在有限时间内摸清格局、形成判断、产出专业报告的系统性方法论。

**调研日期**：2026-03-29
**调研类型**：概念研究 + 方法论体系构建
**调研深度**：搜索 3 层，参考 30+ 来源

---

## 摘要

本报告系统梳理了"如何调研一个行业"的元能力体系，覆盖七大维度：分析框架、标准流程、报告模板、数据采集、分析方法、可视化标准、业界实践。核心结论是：行业分析是一套可标准化的方法论，关键在于**框架选择 -> 数据采集 -> 逻辑推演 -> 判断输出**四个环节的有序组合。不同机构（券商、咨询、VC）的分析视角不同，但底层方法论相通。掌握本报告所述方法体系，可以在一天内形成行业初印象，一周内产出摸底报告，一个月内输出深度行研报告。

## 1. 行业分析的通用框架体系

### 1.1 宏观环境分析

#### PEST/PESTEL 分析

| 维度 | 说明 |
|------|------|
| **适用场景** | 进入一个新市场/新国家前的环境扫描；评估宏观变量对行业的影响 |
| **核心步骤** | 1) 列出 P（政治）E（经济）S（社会）T（技术）L（法律）E（环境）六个维度的关键因素 -> 2) 评估每个因素的影响方向（利好/利空）和影响程度（高/中/低）-> 3) 识别关键驱动因素和潜在风险 |
| **输入数据** | 政府政策文件、经济统计数据、人口普查、技术趋势报告、法规变化、环保政策 |
| **产出结论** | 宏观环境对行业的总体影响评估；关键机遇与威胁清单；需要持续监测的变量 |
| **局限性** | 偏静态分析，难以捕捉各因素间的动态交互；不同分析师对同一因素的影响判断可能差异很大；容易沦为"列清单"而缺乏深度洞察 |

**实操要点**：
- 不要试图穷举所有因素，每个维度聚焦 2-3 个最关键的变量
- 重点分析"正在变化"的因素，稳定不变的不值得花篇幅
- 必须落到"对具体行业意味着什么"，而非泛泛描述宏观环境

#### 波特钻石模型（Porter Diamond Model）

| 维度 | 说明 |
|------|------|
| **适用场景** | 分析某国/某地区特定行业的竞争优势来源；评估产业集群效应 |
| **核心步骤** | 分析四大要素：1) 要素条件（劳动力、资本、基础设施）-> 2) 需求条件（本土市场需求的特征）-> 3) 相关及支持性产业（供应链生态）-> 4) 企业战略/结构/竞争（行业内竞争强度）；再加两个外生变量：政府、机会事件 |
| **输入数据** | 国家/地区的产业数据、教育/人才储备、基础设施水平、本土消费者特征、产业链上下游布局 |
| **产出结论** | 该行业在特定地区的竞争优势来源；产业集群发展的驱动因素 |
| **局限性** | 主要适用于国家/地区层面的产业分析，对单一企业层面的指导性较弱；对全球化供应链分散的行业适用性下降 |

### 1.2 行业结构分析

#### 波特五力模型（Porter's Five Forces）

| 维度 | 说明 |
|------|------|
| **适用场景** | 评估行业的整体盈利能力和竞争强度；判断一个行业是否"好行业" |
| **核心步骤** | 逐一分析五种力量：1) **现有竞争者之间的竞争强度** -- 玩家数量、行业增速、产品差异化程度、退出壁垒；2) **新进入者的威胁** -- 进入壁垒（规模经济、品牌、资本、牌照、技术）；3) **替代品的威胁** -- 替代产品的性价比、用户转换成本；4) **供应商的议价能力** -- 供应商集中度、转换成本、前向一体化可能性；5) **买方的议价能力** -- 买方集中度、信息透明度、后向一体化可能性 |
| **输入数据** | 行业内企业数量和市占率、行业增长率、产品差异化程度、进入壁垒高度、供应商/客户集中度 |
| **产出结论** | 行业利润池的大小和分配；哪些力量最强（意味着利润被谁拿走）；行业的整体吸引力评分 |
| **局限性** | 偏静态快照，难以反映行业演变趋势；未充分考虑互补品的作用（Brandenburger & Nalebuff 后来补充了"价值网"模型）；对平台型、生态型商业模式的适用性需要调整 |

**五力强弱判断标准**：

| 力量 | 强（不利于行业利润） | 弱（有利于行业利润） |
|------|----------------------|----------------------|
| 现有竞争 | 玩家多、增长慢、产品同质化、固定成本高 | 寡头格局、高增长、强差异化 |
| 新进入者 | 低资本门槛、无品牌壁垒、无规模效应 | 高资本/技术/牌照壁垒 |
| 替代品 | 替代品性价比高、转换成本低 | 无可替代、高转换成本 |
| 供应商 | 供应商集中、产品独特、转换成本高 | 供应商分散、标准化产品 |
| 买方 | 买方集中、标准化采购、信息透明 | 买方分散、定制化产品 |

#### 行业生命周期模型（Industry Life Cycle）

| 阶段 | 特征 | 增长率 | 竞争格局 | 典型策略 |
|------|------|--------|----------|----------|
| **导入期** | 产品刚问世，市场认知低 | 缓慢 | 少数先驱者，无标准 | 教育市场、建立品类认知 |
| **成长期** | 需求快速增长，新进入者涌入 | 高速（20%+） | 大量玩家涌入，快速洗牌 | 抢占市场份额、建立品牌 |
| **震荡期** | 增速放缓，弱势玩家被淘汰 | 减速 | 剧烈洗牌，集中度提升 | 差异化或成本领先 |
| **成熟期** | 增长平稳，格局固化 | 低速（GDP附近） | 寡头稳定，价格竞争 | 效率优化、细分市场 |
| **衰退期** | 需求萎缩，被替代 | 负增长 | 玩家退出，剩者为王 | 收割或转型 |

**实操要点**：
- 判断行业处于哪个阶段是行研的**第一个关键判断**
- 用增长率曲线（近5-10年）+ 玩家数量变化 + 新进入者频率来辅助判断
- 不同阶段的估值逻辑完全不同：成长期看 PS/用户数，成熟期看 PE/现金流，衰退期看重置价值

#### 价值链分析（Value Chain Analysis）

| 维度 | 说明 |
|------|------|
| **适用场景** | 理解行业的运作流程和利润分配；找到高价值环节 |
| **核心步骤** | 1) 梳理从原材料到最终用户的完整链条 -> 2) 识别每个环节的参与者和市场结构 -> 3) 分析每个环节的毛利率/增加值 -> 4) 找出"利润池"最深的环节 -> 5) 分析各环节的壁垒和话语权 |
| **输入数据** | 产业链各环节的企业财务数据（毛利率、净利率）、上下游议价关系、库存周转、应收账期 |
| **产出结论** | 价值链全景图 + 利润池分布图；行业核心环节在哪里；哪些环节有结构性优势 |
| **局限性** | 需要大量的实际财务数据支撑；对平台型/双边市场的价值链分析需要特殊处理 |

**价值链分析的三层递进**：
1. **描述层**：画出完整的产业链图（上游 -> 中游 -> 下游）
2. **定量层**：标注每个环节的市场规模、毛利率、CR5
3. **判断层**：哪个环节最赚钱？为什么？这个格局会变吗？

### 1.3 竞争格局分析

#### 市场集中度指标

| 指标 | 计算方式 | 判断标准 | 适用场景 |
|------|---------|----------|---------|
| **CR3/CR5** | 前3/5名企业市占率之和 | CR5 > 60% 为高集中度 | 快速判断行业格局 |
| **HHI** | 所有企业市占率的平方和 (x10000) | < 1000 分散；1000-1800 中等；> 1800 高集中 | 反垄断审查、精确度量 |

**CR 与 HHI 的区别**：CR 只看头部，忽略尾部分布；HHI 对大企业赋予更大权重，能区分"一家独大"和"双寡头"的区别。

#### 战略组图（Strategic Group Mapping）

**画法**：
1. 选择两个最能区分竞争者的战略维度（如：价格带 vs. 产品线宽度，或 线上 vs. 线下，或 2B vs. 2C）
2. 将所有主要玩家标注在二维坐标上
3. 同一区域的企业构成一个"战略组"，圆圈大小可表示营收规模
4. 分析组间的移动壁垒和竞争动态

**价值**：一张图看清"谁和谁真正在竞争"，而不是笼统地把所有行业参与者混为一谈。

#### 竞争地位矩阵

常用矩阵包括：
- **BCG 矩阵**：市场增长率 x 相对市场份额 -> 明星/金牛/问题/瘦狗
- **GE-麦肯锡矩阵**：行业吸引力 x 竞争地位 -> 9宫格
- **Gartner 魔力象限**：执行力 x 愿景完整性 -> 领导者/挑战者/远见者/利基

### 1.4 市场规模测算

#### TAM/SAM/SOM 模型

| 层级 | 定义 | 举例（中国在线教育 K12） |
|------|------|--------------------------|
| **TAM**（Total Addressable Market） | 全部可触达市场 | 中国 K12 教育市场总规模约 8000 亿 |
| **SAM**（Serviceable Addressable Market） | 可服务的市场 | 其中在线教育渗透率约 15%，约 1200 亿 |
| **SOM**（Serviceable Obtainable Market） | 可获取的市场 | 基于竞争格局和执行力，可拿到 5%，约 60 亿 |

#### 两种测算方法

**自上而下（Top-Down）**：
```
总人口/企业数
x 目标客群占比
x 渗透率
x 客单价/ARPU
= 市场规模
```
- **优点**：快、宏观感知好
- **缺点**：依赖假设，精度低
- **数据来源**：统计局、行业报告、Statista

**自下而上（Bottom-Up）**：
```
目标客户数量（按 ICP 画像逐层筛选）
x 单客户年度支出
x 付费转化率
= 市场规模
```
- **优点**：贴近实际，假设可验证
- **缺点**：慢、数据获取难
- **数据来源**：企业调研、招股书、渠道数据

**最佳实践**：两种方法都做，如果结果在 15% 偏差范围内，说明假设可靠。偏差过大时，检查假设中的薄弱环节。

#### 费米估算（Fermi Estimation）

当缺乏现成数据时的快速估算方法。核心原则是**将大问题拆成小问题，每个小问题做合理假设**。

**标准步骤**：
1. **明确问题边界**：估算什么、时间范围、地理范围
2. **拆解为可估算的子问题**：用乘法链或加法链
3. **为每个子问题找锚点**：已知数据、常识推理、类比参考
4. **计算并做 sanity check**：结果是否在合理数量级？
5. **标注不确定性**：给出范围而非单一数字

**常见锚点数据（中国市场）**：
- 中国总人口约 14 亿，家庭数约 5 亿户
- 城镇化率约 65%，城镇人口约 9 亿
- GDP 总量约 130 万亿元，人均 GDP 约 9 万元
- 劳动年龄人口约 8.8 亿
- 智能手机用户约 10 亿
- 网购用户约 9 亿
- 私家车保有量约 3.4 亿辆
- 餐饮行业规模约 5 万亿，外卖约 1.2 万亿

**费米估算示例**：估算中国宠物食品市场规模
```
养宠家庭数 = 城镇家庭 3.2亿 x 养宠率 25% = 8000万户
宠物均数/户 约 1.2 只
宠物食品月均消费 约 200 元
年市场规模 = 8000万 x 1.2 x 200 x 12 约 2300 亿元
（交叉验证：Euromonitor 2025 数据约 2000-2500 亿，吻合）
```

### 1.5 趋势预判

#### Gartner Hype Cycle（技术成熟度曲线）

| 阶段 | 特征 | 分析要点 |
|------|------|---------|
| **技术触发期** | 概念验证、媒体关注开始 | 识别真正的技术突破 vs. 概念炒作 |
| **期望膨胀期** | 大量投资涌入，预期过高 | 警惕泡沫，不宜追高 |
| **泡沫破裂期** | 落地不及预期，投资冷却 | 关注真正有产品的幸存者 |
| **爬坡期** | 应用场景清晰，商业化开始 | 最佳投资/进入时机 |
| **成熟期** | 大规模采用，ROI 可预测 | 关注效率和成本优化 |

**局限性**：研究表明，只有约 20% 的技术真正走完完整的 Hype Cycle 五阶段。该曲线更多反映市场情绪而非技术本身的成熟度。建议与 S 曲线结合使用。

#### S 曲线（Technology S-Curve）

描述技术性能随累积研发投入的关系：初期进展缓慢 -> 加速突破 -> 逐渐趋于极限。

**分析要点**：
- 识别技术是否接近 S 曲线顶端（意味着渐进式创新回报递减）
- 识别"S 曲线跳跃"：新技术范式取代旧技术（如液晶取代 CRT、锂电取代铅酸）
- 在旧 S 曲线尾部 + 新 S 曲线头部的交叉点，就是行业拐点

#### 创新扩散模型（Innovation Diffusion Model）

Rogers 的创新扩散理论将用户分为五类：

| 用户类型 | 占比 | 特征 |
|---------|------|------|
| 创新者 | 2.5% | 技术极客，容忍不完美 |
| 早期采用者 | 13.5% | 意见领袖，看中先发优势 |
| 早期多数 | 34% | 实用主义者，需要成熟方案 |
| 晚期多数 | 34% | 保守派，需要主流验证 |
| 落后者 | 16% | 最后接受，被迫跟进 |

**关键概念**："鸿沟"（Chasm）位于早期采用者和早期多数之间。许多技术/产品在跨越鸿沟时失败。判断行业是否已跨越鸿沟，是预判增长曲线的关键。

### 框架选择决策树

```
问："这个行业我一无所知"

 第一步：PESTEL（宏观环境，10 分钟定性扫描）
 第二步：行业生命周期（判断阶段，决定后续分析重点）
 第三步：价值链分析（理解行业怎么运作）
 第四步：波特五力（判断行业好不好）
 第五步：市场规模测算（行业多大）
 第六步：竞争格局（谁在玩、谁在赢）
 第七步：趋势预判（接下来会怎样）
```

## 2. 快速行研的标准流程

### 第一天（4-8 小时）：建立初印象

| 时段 | 关键动作 | 产出 |
|------|---------|------|
| **前 2 小时** | 1) 搜索行业概览（维基百科 + 百度百科 + 行业协会官网）；2) 阅读 2-3 篇券商/咨询公司的行业综述报告；3) 扫描行业头部公司的官网和投资者关系页面 | 行业基本定义、边界、主要产品/服务 |
| **第 3-4 小时** | 1) 整理行业产业链图（上游-中游-下游）；2) 列出 Top 10 玩家清单及其基本信息；3) 快速浏览 2-3 家上市公司的年报/招股书摘要 | 产业链草图 + 竞争者初始名单 |
| **第 5-6 小时** | 1) 搜索行业近 1-2 年的重大事件（并购、政策变化、技术突破）；2) 浏览行业媒体/自媒体/论坛，获取一线感知；3) 记录不理解的行业术语和概念 | 行业动态时间线 + 待深入问题清单 |
| **第 7-8 小时** | 1) 初步判断行业所处生命周期阶段；2) 形成 3-5 个关键假设/待验证问题；3) 规划第一周的深入方向 | 初步判断备忘录（1-2 页） |

**第一天的信息来源**：
- 百度/Google 搜索 + Wikipedia/百度百科
- 券商研报摘要（东方财富、Wind 免费部分）
- 行业协会年鉴/年报
- 头部公司官网 + 投资者关系页面
- 36氪/虎嗅/亿欧等行业媒体
- 知乎行业话题下的高赞回答

### 第一周（5 个工作日）：摸底报告

| 天数 | 关键动作 | 产出 |
|------|---------|------|
| **D2-D3** | 1) 系统阅读 3-5 家头部公司招股书/年报的行业部分；2) 做市场规模测算（Top-Down + Bottom-Up 双验证）；3) 建立竞争格局矩阵 | 市场规模数据 + 竞争格局表 |
| **D4** | 1) 做价值链分析，标注各环节的利润率；2) 做波特五力评估；3) 梳理行业关键成功因素 | 价值链图 + 五力分析 + KSF 清单 |
| **D5** | 1) 整合所有发现，撰写摸底报告（3-5 页）；2) 标注"信息充分"和"信息不足"的区域；3) 提出后续深入研究的方向建议 | **摸底行业报告（3-5 页）** |

**第一周可达到的水平**：
- 能回答"这个行业是做什么的、有多大、谁在玩、怎么赚钱、趋势如何"
- 能与行业从业者进行有意义的对话（不至于问外行问题）
- 能识别出行业的 2-3 个核心议题，但尚未形成深度判断

### 深度研究（2-4 周）：专业报告

| 周数 | 关键动作 | 产出 |
|------|---------|------|
| **W2** | 1) 深入财务分析（头部 5-10 家公司的财报对比）；2) 建立行业数据库（收入、利润率、增长率、市占率时间序列）；3) 安排 3-5 场专家访谈 | 财务分析表 + 行业数据库 |

| **W4** | 1) 形成核心投资/战略判断；2) 撰写深度报告（20-50 页）；3) 制作配套图表和数据可视化；4) 内部评审和修改 | **深度行业研究报告（20-50 页）** |

**深度研究的信息来源升级**：
- 付费数据库（Wind/Bloomberg/Statista/IBISWorld）

- 专家网络访谈（GLG/AlphaSights/Guidepoint）
- 行业展会和论坛资料
- 专利数据库（CNKI 专利、Google Patents）
- 另类数据（招聘数据、卫星数据、社交媒体）

- **配套图表**：至少包含产业链图、市场规模趋势图、竞争格局图、利润池分布图

## 4. 数据采集的通用方法论

### 4.1 公开数据源

| 数据源 | 特点 | 获取方式 | 适用场景 |
|--------|------|---------|---------|
| **国家统计局** | 宏观经济数据、行业产值、人口数据，权威但颗粒度粗 | stats.gov.cn 免费查询 | 行业总量估算、宏观趋势验证 |
| **行业协会** | 细分行业数据、会员企业数据，专业但可能有行业美化 | 各行业协会官网 + 年鉴 | 行业规模、产量、进出口数据 |
| **上市公司年报** | 企业层面的精确财务数据，经审计但仅限上市公司 | 巨潮资讯网/SEC EDGAR | 公司分析、行业对比 |
| **招股书** | 行业分析章节通常是高质量的行业综述，有第三方引用 | 证监会/交易所官网 | **快速获取行业概览的最佳来源之一** |
| **行业白皮书** | 行业全景+趋势，但可能有赞助方倾向 | 各机构官网下载 | 行业趋势、市场预测 |
| **政府规划** | 政策方向、产业目标，权威但预测偏乐观 | 发改委/工信部等部委官网 | 政策趋势、补贴方向 |
| **海关数据** | 进出口量价数据，精确到 HS 编码 | 海关总署统计 | 国际贸易分析、进口替代判断 |

**实操技巧**：招股书是快速入门一个行业的最佳信息源。因为招股书的行业分析章节通常引用 Frost & Sullivan、IDC 等第三方数据，等于免费获取了付费报告的精华内容。

### 4.2 付费数据库

| 数据库 | 覆盖范围 | 年费量级 | 适用场景 |
|--------|---------|---------|---------|
| **Wind（万得）** | 中国资本市场数据最全面，覆盖 400+ 行业 | 10-50 万/年 | 中国市场行业研究、财务分析 |

### 4.3 另类数据

截至 2025 年，超过 70% 的对冲基金已将另类数据纳入投资策略。

| 数据类型 | 能推断什么 | 数据来源 | 示例 |
|---------|-----------|---------|------|
| **招聘数据** | 公司扩张/收缩方向、新业务布局 | 猎聘/BOSS直聘/LinkedIn | 某公司大量招聘自动驾驶工程师 -> 布局自动驾驶 |
| **专利数据** | 技术方向、研发投入重点、专利壁垒 | CNKI专利/Google Patents/USPTO | 某技术领域专利申请量爆发 -> 技术临近商业化 |
| **卫星数据** | 工厂开工率、零售客流、农作物产量 | Planet/Orbital Insight | 特斯拉工厂停车场车辆密度 -> 推算产量 |
| **社交媒体** | 品牌声量、消费者情绪、产品口碑 | 微博/抖音/小红书/Twitter | 某品牌讨论量突增 -> 新品成功/危机公关 |
| **App 数据** | DAU/MAU、使用时长、下载量 | 七麦/SensorTower/QuestMobile | 某 App 下载量连续 3 月下降 -> 用户流失 |
| **供应链数据** | 上下游关系、原材料价格趋势 | 海关数据/物流数据 | 锂矿进口量激增 -> 电池产能扩张 |

### 4.4 专家访谈

#### 找专家的渠道

| 渠道 | 成本 | 速度 | 质量 |
|------|------|------|------|
| **专家网络平台**（GLG/AlphaSights/Third Bridge/Guidepoint） | 1000-3000 美元/小时 | 24-48 小时匹配 | 高 |
| **LinkedIn 直接联系** | 免费/低成本 | 不确定 | 取决于个人能力 |
| **行业会议/展会** | 会议费用 | 需等到会议日 | 高，面对面深入 |
| **行业协会** | 会员费 | 一般 | 中等 |
| **个人网络** | 免费 | 快 | 取决于网络质量 |

#### 访谈提纲设计原则

1. **开场（5分钟）**：自我介绍 + 访谈目的 + 保密说明
2. **背景了解（10分钟）**：专家的行业经历、当前角色、视角定位
3. **行业核心问题（30分钟）**：
   - "这个行业赚钱最重要靠什么？"（识别 KSF）
   - "行业最大的变化是什么？什么在驱动变化？"（识别趋势）
   - "哪些公司做得好？好在哪里？"（竞争情报）
   - "如果今天要进入这个行业，您的建议是什么？"（进入壁垒）
   - "行业最大的风险/不确定性是什么？"（风险识别）
4. **验证与追问（10分钟）**：验证自己的关键假设
5. **收尾（5分钟）**：是否有推荐的其他专家或资料来源

**信息提取要点**：
- 区分"事实"和"观点"——事实可直接引用，观点需交叉验证
- 关注专家的"意外之语"——行业内的 common knowledge 往往不在报告里
- 一个问题问 3 个不同背景的专家，取交集作为可靠结论

### 4.5 没有数据时的估算方法

当面对完全空白的领域时，采用**"锚点 + 推理链"**方法：

```
Step 1: 找到最接近的已知数据点（锚点）
        例：不知道中国宠物医疗市场规模
        锚点：知道养宠家庭数约 8000 万

Step 2: 建立推理链
        8000 万养宠家庭
        x 年均医疗消费（看病 + 疫苗 + 体检）估 800 元
        = 640 亿

Step 3: 交叉验证
        方法 A（供给端）：中国宠物医院约 2.5 万家
        x 平均年营收约 250 万
        = 625 亿
        两种方法偏差 < 5%，结论可信

Step 4: 标注不确定性
        中心估计 640 亿，
        保守估计（年均消费 500 元）= 400 亿
        乐观估计（年均消费 1200 元）= 960 亿
```

## 5. 从数据到判断的分析方法

### 5.1 提炼行业核心逻辑——"这个行业赚钱靠什么"

**三问法**：
1. **客户为什么付钱？**（价值主张——功能价值 / 情感价值 / 社交价值）
2. **钱怎么流转？**（客单价 x 频次 x 毛利率 x 运营杠杆）
3. **谁能持续赚到钱？**（壁垒——规模效应 / 网络效应 / 品牌 / 技术 / 牌照 / 数据）

**杜邦分析拆解行业利润**：
```
ROE = 净利率 x 资产周转率 x 杠杆倍数

不同行业赚钱模式的本质差异：
- 奢侈品：高净利率 x 低周转 x 低杠杆 = "赚利润率的钱"
- 零售/物流：低净利率 x 高周转 x 中杠杆 = "赚周转的钱"
- 银行/地产：中净利率 x 低周转 x 高杠杆 = "赚杠杆的钱"
- SaaS：逐步提升净利率 x 中周转 x 低杠杆 = "赚续费的钱"
```

### 5.2 识别行业关键成功因素（KSF）

KSF 是指在某个行业中，决定企业能否成功的 3-5 个最关键因素。

**识别方法**：

| 方法 | 具体操作 |
|------|---------|
| **成功者逆推** | 分析 Top 3 企业的共同特征——它们都做对了什么？ |
| **失败者反推** | 分析退出/倒闭的企业——它们缺了什么？ |
| **客户视角推** | 客户选择供应商的决策因素排序（价格/质量/速度/品牌/服务） |
| **价值链推** | 价值链中利润最高的环节，其壁垒就是 KSF |
| **专家访谈推** | 直接问行业老兵："做好这行最重要的三件事是什么？" |

**常见行业 KSF 模式**：

| 行业类型 | 典型 KSF |
|---------|---------|
| 消费品 | 品牌、渠道、产品力 |
| 互联网平台 | 网络效应、用户规模、数据 |
| 制造业 | 成本控制、良率、产能规模 |
| 医药 | 研发管线、注册审批、销售团队 |
| 金融 | 风控能力、资金成本、牌照 |
| SaaS | 产品力、客户成功、续费率 |

### 5.3 判断行业拐点和变量

**拐点信号识别**：

| 信号类型 | 具体指标 | 说明 |
|---------|---------|------|
| **需求侧拐点** | 渗透率突破 15-20% | Rogers 创新扩散模型的"鸿沟跨越"标志 |
| **供给侧拐点** | 成本下降穿越用户付费意愿 | 如光伏发电度电成本低于火电 |
| **技术拐点** | 核心技术性能指标突破阈值 | 如 AI 大模型准确率超过人类 |
| **政策拐点** | 重大政策出台/行业整顿 | 如双减政策对教培行业 |
| **资本拐点** | 一级市场融资金额/笔数突变 | 资金大量涌入或突然冷却 |
| **竞争拐点** | CR5 快速变化 | 快速集中（洗牌中）或快速分散（新入者涌入） |

**关键变量分析框架**：
1. 列出可能改变行业走向的 5-10 个变量
2. 评估每个变量的**影响程度**（高/中/低）和**发生概率**（高/中/低）
3. 聚焦"高影响 x 中高概率"的变量——这就是需要持续跟踪的"胜负手"
4. 对核心变量做情景分析（乐观/基准/悲观）

### 5.4 评估"这个行业值不值得进入"

**行业吸引力评估矩阵（10 分制打分）**：

| 评估维度 | 权重 | 评分要点 |
|---------|------|---------|
| 市场规模 | 15% | TAM > 1000 亿得高分 |
| 增长速度 | 20% | 增速 > 20% 得高分 |
| 利润水平 | 15% | 行业平均净利率 > 10% 得高分 |
| 竞争强度 | 15% | 格局清晰、壁垒高得高分 |
| 进入壁垒 | 10% | 对在位者有利的壁垒得高分 |
| 政策风险 | 10% | 政策稳定/利好得高分 |
| 技术变革风险 | 10% | 技术路线明确得高分 |
| 客户集中度 | 5% | 客户分散得高分 |

**加权总分解读**：7-10 分高吸引力；5-7 分中等，需差异化切入；3-5 分低吸引力；1-3 分回避。

### 5.5 处理信息不完整和不确定性

| 情况 | 应对策略 |
|------|---------|
| 数据缺失 | 用费米估算 + 多种方法交叉验证 |
| 来源矛盾 | 列出所有说法，分析各自的立场和方法论差异，给出判断 |
| 趋势不明 | 做情景分析（2-3 种情景），标注各情景的概率估计 |
| 专家观点分歧 | 区分事实层面的分歧和判断层面的分歧，前者可验证，后者标注不同观点 |
| 非公开信息 | 通过关联数据间接推算（如通过招聘数据推断业务方向） |

**核心原则**：宁可坦诚说"信息不足，以下是基于有限数据的推测"，也不要假装信息充分做出看似确定的结论。标注不确定性本身就是专业性的体现。

## 6. 行业分析的可视化标准

### 6.1 必备图表清单

| 图表类型 | 适用场景 | 说明 |
|---------|---------|------|
| **产业链全景图** | 每份报告必备 | 上游-中游-下游，标注各环节的主要参与者 |
| **市场规模趋势图** | 每份报告必备 | 柱状图（历史）+ 虚线（预测），标注增长率 |
| **竞争格局图** | 每份报告必备 | 饼图/柱状图展示市占率，或2x2 象限图 |
| **利润池分布图** | 深度报告必备 | 条形图，各环节毛利率对比 |
| **战略组图** | 深度报告必备 | 散点图/气泡图，2 个维度定位竞争者 |
| **五力雷达图** | 按需使用 | 五角雷达图，直观展示五力强弱 |
| **时间线图** | 行业有重大事件时 | 行业发展里程碑 |
| **对比矩阵** | 竞品分析时 | 热力图或打分表，多维度对比 |
| **趋势折线图** | 趋势分析时 | 多条线对比趋势 |
| **桑基图** | 资金/产品流向分析时 | 展示流量分配和转化 |

### 6.2 核心图表的标准画法

#### 产业链图

标准结构：三层横向排列，每层包含环节名称、代表企业（3-5家）、毛利率、集中度（CR5）。箭头方向表示产品/服务流向。可在箭头上标注交易规模。

#### 竞争象限图（2x2 矩阵）

维度选择原则：两个维度必须是独立的（不高度相关）、有区分度的（能将玩家分开）、有意义的（与竞争力相关）。常见维度组合：
- Gartner 魔力象限：执行力 x 愿景完整性
- 通用竞争象限：市场份额 x 增长速度
- 定位图：价格 x 质量/功能

#### 市场规模图

标准样式：柱状图 + 折线图组合。柱状图表示每年市场规模（亿元），历史数据用实心填充、预测数据用斜线填充。折线图叠加年同比增长率。数据标签标注在柱顶，图表底部注明数据来源。

### 6.3 数据可视化最佳实践

| 原则 | 说明 |
|------|------|
| **一图一意** | 每张图表只传递一个核心信息，标题即结论（如"市场规模稳步增长，CAGR 15%"而非"市场规模图"） |
| **色彩克制** | 主色 3-4 种，用对比色强调重点数据，灰色弱化背景数据 |
| **数据标注** | 关键数据直接标注在图表上，减少读者视线在图和图例之间的跳转 |
| **避免 3D** | 永远不用 3D 图表，它只会扭曲数据比例 |
| **排序有意义** | 柱状图按数值大小排序（除非有时间/类别逻辑），而非按字母排序 |
| **来源注脚** | 每张图表底部标注数据来源和时间 |
| **单位统一** | 同一张图表内单位统一，必要时注明汇率或基准年 |

## 7. 业界最佳实践

### 7.1 三大视角对比

| 维度 | 券商研究所 | 管理咨询公司 | VC/PE |
|------|----------|------------|-------|
| **核心问题** | "这个行业/公司值多少钱？" | "在这个行业怎么赢？" | "这个行业能否诞生 10x 回报的公司？" |
| **服务对象** | 基金经理、机构投资者 | 企业高管（CEO/CSO） | 投委会、LP |
| **时间视角** | 6-12 个月（盈利预测） | 3-5 年（战略规划） | 5-10 年（退出周期） |
| **核心方法** | 财务建模 + 估值（DCF/Comps） | 战略框架（五力/价值链/增长矩阵） | 主题驱动 + 创始人评估 + 单位经济模型 |
| **数据重心** | 上市公司财报、行业统计 | 企业内部数据 + 专家访谈 | 另类数据 + 一线调研 |
| **输出物** | 行业深度报告 + 盈利预测 | 战略方案 + 行动计划 | 投资备忘录 + 行业地图 |
| **判断标准** | 盈利预测准确性 | 客户营收/利润改善 | IRR/MOIC |

### 7.2 券商研究所的行研方法论

**标准研究流程**：
1. **行业框架搭建**：用 PEST + 五力 + 生命周期确定行业基本面
2. **市场规模测算**：自上而下 + 自下而上双验证
3. **竞争格局分析**：CR5 + 市占率趋势 + 竞争壁垒
4. **公司筛选与深度分析**：从行业到公司，选择 10-20 家覆盖
5. **财务建模**：三表联动模型，做 3-5 年盈利预测
6. **估值与投资评级**：DCF + 可比公司 + 可比交易

**中金/高盛的特色**：
- 数据基础设施强大：中金覆盖 120,000+ 数据指标和 600+ 专业数据源
- 自上而下与自下而上结合：宏观策略团队 + 行业研究团队 + 公司研究团队三层协同
- 国际化视角：将中国行业放在全球框架下评估
- 产业链调研：定期实地调研（工厂参观、渠道走访、展会参加）

**核心跟踪指标**：行业增速、PE/PB 估值中枢、龙头 ROE/ROIC、毛利率趋势、资本开支/折旧比

### 7.3 咨询公司的行业分析套路

**McKinsey 的结构化问题解决法**：
1. **定义问题**：用一句话描述清楚要解决什么
2. **MECE 拆解**：将问题拆为互不重叠、完全穷尽的子问题
3. **假设驱动**：先形成假设，再收集数据验证/证伪
4. **80/20 法则**：80% 的答案来自 20% 的分析
5. **电梯测试**：能在 30 秒内向 CEO 说清核心结论

**McKinsey 常用框架**：
- **McKinsey 7S**：Strategy/Structure/Systems/Shared Values/Skills/Style/Staff
- **McKinsey Growth Pyramid**：核心增长 -> 新产品 -> 新渠道 -> 新客户 -> 新地理 -> 新行业
- **Profit Pool Analysis**：行业利润在各环节的分配

**BCG 的方法论特色**：
- 更强调迭代和实验，而非严格遵循预定义框架
- BCG 矩阵（增长-份额矩阵）仍是评估业务组合的经典工具
- 注重适应性战略——在不确定环境下，强调快速实验和学习

**Bain 的方法论特色**：
- "结果导向"——更关注帮客户实际提升业绩
- NPS（净推荐值）：Bain 创始人提出的客户忠诚度度量工具
- Value Chain Analysis 和 Customer Segmentation 是 Bain 的强项

### 7.4 VC/PE 的行业研究视角

**VC 看行业的核心问题清单**：
1. **市场够不够大？** TAM > 100 亿美元才值得关注
2. **时机对不对？** Why now？为什么是现在？
3. **有没有结构性变化？** 技术突破/政策变化/消费者行为变化创造的窗口期
4. **能不能 10x？** 能否诞生 10 倍回报的公司？
5. **赢家通吃 or 百花齐放？** 市场集中度会如何演变？
6. **护城河在哪？** 网络效应/规模效应/数据壁垒/品牌/技术/牌照
7. **谁是最佳团队？** 创始人的认知深度和执行力

**红杉资本的投资框架**：
- 专注种子期到成长期的技术公司
- 开发了 Arc Framework，将 PMF 分为三种原型："Hair on Fire"（刚需）、"Hard Fact"（事实驱动）、"Future Vision"（愿景驱动）
- 深度参与投后：产品策略、核心团队招聘、首批客户获取

**高瓴资本**："研究驱动"投资风格，100+ 人投后管理团队，关注"结构性变化"，投资视角 10 年+。

**a16z**："Software is Eating the World"视角，主题驱动研究（Thesis-Driven），近年 15%+ 新基金配置在 AI 基础设施和应用。

### 7.5 三者视角差异的实操启示

- **企业做战略规划** -> 用咨询视角："怎么赢"，输出战略路线图 + 资源配置方案
- **投资人做投资决策** -> 用券商 + VC 混合视角，输出投资备忘录 + 估值模型
- **创业者选赛道** -> 用 VC 视角倒推："VC 会怎么看？市场够大吗？Why now？护城河在哪？"

## 结论与建议

### 核心结论

1. **行业分析是一套可标准化的方法论**，掌握"框架选择 -> 数据采集 -> 逻辑推演 -> 判断输出"四步法，可以快速入门任何陌生行业。专业的方法论可以弥补经验不足。

2. **框架是工具而非目的**。波特五力、PEST、TAM/SAM/SOM 等框架的价值在于提供思考的结构，而非填满每个格子。最终的判断力来自"将框架分析结果串成逻辑链条"的能力——即从零散数据中提炼出"这个行业的核心逻辑是什么、正在发生什么变化、变化带来什么机会或风险"。

3. **三大机构视角互补而非对立**。券商的财务分析深度、咨询的战略思维框架、VC 的终局思维和变量敏感度，是三种不同的"镜头"。最优秀的行业分析师能灵活切换这三种视角。

### 行动建议

1. **建立个人的行业分析 Checklist**：基于本报告的框架体系，定制适合自己工作场景的行研检查清单。
2. **积累"锚点数据库"**：持续收集各行业的关键基准数据，作为费米估算和交叉验证的参照。
3. **练习"一天摸清一个行业"**：选取不熟悉的行业，限时 8 小时做一份摸底报告，是最有效的训练。
4. **重视专家网络**：数据和报告只能提供 60-70% 的行业认知，剩下的 30-40% 来自与行业一线从业者的交流。

### 未覆盖的问题

- 具体行业的分析模板差异（消费品 vs. 制造业 vs. 互联网的侧重点差异）
- 财务建模的具体操作方法（三表联动、DCF 估值等技术细节）
- AI 工具在行研中的应用（AI 辅助搜索、自动化数据采集、报告生成）
- 跨国行业对比的方法论差异
